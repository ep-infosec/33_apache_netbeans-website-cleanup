Guia de configuração da plataforma do emulador CDC do NetBeans

Embora o NetBeans ofereça suporte ao desenvolvimento de Connected Device Configuration (CDC) em Java Micro Edition (ME), o IDE não tem a plataforma CDC incluída no instalador a fim de permitir a criação de aplicativos para dispositivos CDC usando os perfis Personal, Foundation ou AGUI. Este documento fornece informações sobre download, instalação e registro do Sun Java Toolkit para CDC e outros Software Development Kits (SDK) populares para o desenvolvimento em CDC.

Software necessário

Para concluir este tutorial, você precisa dos seguintes recursos e softwares:

Software ou recurso Versão necessária
NetBeans IDE com Java ME Versão 6.9 ou posterior
Java Development Kit (JDK) Versão 6 ou versão 7

Instalando as plataformas do emulador

Nokia Series 80 Platform SDK para SO Symbian, para Java, perfil Personal

Os desenvolvedores dos dispositivos Nokia 80 Series podem instalar o SDK da plataforma Series 80 para Symbian OS, para Java, perfil Personal, seguindo estas instruções:

  1. Baixe o SDK da plataforma Nokia Series 80 -para Symbian OS, para Java, perfil Personal do fórum de desenvolvedor da Nokia (download). O nome do arquivo é S80_DP_2_0_PP_SDK.zip.
  2. Descompacte o arquivo.
  3. Navegue para o local em seu sistema onde você descompactou o arquivo. Clique duas vezes no ícone do arquivo setup.exe. O assistente do instalador do Nokia se abre.
  4. Siga as instruções do assistente para instalar o SDK. O assistente orientará você para escolher o dispositivo que deseja tornar o padrão. Escolha o Series80_DP2_0_PP_SDK.
  5. No menu principal do IDE, escolha Ferramentas > Plataformas Java.
  6. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java se abre.
  7. Selecione Emulador da plataforma Java ME CDC e clique em Próximo.
  8. Selecione a pasta que contém o SDK da plataforma. O diretório padrão é C:\Symbian\S80_DP2_0_PP_SDK. Clique em Próximo.
  9. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.
  10. Clique em Terminar. O IDE registra a plataforma CDC.

CrEme VM para Windows CE

Os desenvolvedores de dispositivos Windows CE podem instalar a NSIcom CrEme VM para Windows CE, seguindo estas instruções:

  1. Faça o download da plataforma CrEme 4.12 do NSIcom (download).
  2. Navegue para o local em seu sistema onde você baixou o arquivo CrEmeDevSup410.exe. Clique duas vezes no arquivo .
  3. Siga as instruções do assistente InstallShield NSI.com para instalar o Developer Kit.
  4. No menu principal do IDE, escolha Ferramentas > Plataformas Java.
  5. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  6. Selecione Emulador da plataforma Java ME CDC e clique em Próximo.
  7. Selecione a pasta que contém o SDK da plataforma. O diretório padrão é C:\Program Files\NSIcom\CrE-ME V4.12.. Clique em Avançar.
  8. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.
  9. Clique em Terminar. O IDE registra a plataforma CDC.

Observação: o JFrame e o GroupLayout não são compatíveis com a licença de avaliação do CrEme, mas o NSI.com oferece um download para compatibilidade com gráficos avançados.
