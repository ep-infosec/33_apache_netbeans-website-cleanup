简介

欢迎使用 NetBeansTM IDE 3.6。它是一种模块化、基于标准的集成开发环境 (IDE)，采用 JavaTM 编程语言编写而成。NetBeans 项目包括：

  • 一个使用 Java 编程语言编写的开放源码 IDE。
  • 一个可用作生成任何种类应用程序的通用框架的应用平台。

阅读有关 NetBeans 项目的更多信息...

更改的内容

自 3.5.1 发行版本以来更改的内容

请参见 NetBeans IDE 3.6 中的新功能，以了解此发行版本中更改内容的完整描述。以下是简短摘要：
  • IDE 的窗口系统及整体外观的增强功能：
    • 窗口的拖放。
    • 使用 Ctrl-` 可以切换打开的 IDE 窗口。
    • 执行某些任务时，使用所显示的与任务相关的窗口替换工作区。
    • 用于 Microsoft Windows 和 Mac OS X 平台的本机外观风格。
    • 重新设计的属性表单。
  • 源编辑器的增强功能：
    • 代码折叠。
    • 标记覆盖方法。
    • “待做事项”窗口。
    • 自动插入后半个方括号、引号和圆括号。
    • 与“源编辑器”窗口集成得更好的“窗体编辑器”。
    • 输出为 HTML 格式。
    • 在包含编译错误的任何文本上移动鼠标时，会出现工具提示错误描述。
    • 通过拖放标签，可以折分“源编辑器”窗口。
  • Web/J2EE 开发的更改内容：
    • 支持 Servlet 2.4 和 JSP 2.0。
    • 支持使用 Tomcat 5 部署和调试两层 J2EE 1.4 和 1.3 应用程序。
    • 区分大小写的 HTML 代码完成。
    • 开放/结束标记匹配。
    • “编译 JSP”命令替换为“验证 JSP”命令。
  • Ant 支持的更改内容：
    • Ant 1.6.1 现已被绑定。
    • 可以轻松管理 Ant 类路径。
    • 可以轻松管理 Ant 版本。
  • 版本控制集成的更改内容：
    • 为了使用相同的向导、属性编辑器和命令对话框，已对内置 CVS 客户机和命令行 CVS 支持进行了统一协调。
    • 改进了 VCS 命令输出的显示。
  • 其它更改：
    • IDE 不再运行于 JavaTM 2 SDK Standard Edition 1.3.1 版上，并且不再于 1.4.0 和 1.4.1 版本上进行测试。建议使用 JDK 1.4.2 版。IDE 仍可以在 1.5 Beta 版上运行。
    • 支持 JUnit 测试创建 (Junit 3.8.1)。
    • 支持使用 Tomcat 5 部署和调试两层 J2EE 1.4 和 1.3 应用程序。
    • 所有模块的帮助现已合并到单个目录和单个索引中。
    • 某些节点的属性表单中增加了“文件大小”和“修改时间”。
    • “启动”文件夹已从“选项”窗口中删除，因此不再支持于用户界面中指定 IDE 启动时在 IDE 中运行的类。
    • Apache 的正规表达式库不再包含于 IDE 中。IDE 现在使用 java.util.regex 包来支持正规表达式。不再支持类似非标准 POSIX 的字符类（[:javastart:] 和 [:javapart:]）。
    • 不再支持于同一用户目录中同时运行多个 IDE 实例。
    • 现在能够以 GNU 样式输入 IDE 命令行开关，也就是说，这些开关的前面有两条短划线（例如，--jdkhome），而不是一条短划线。单字母开关（如 -h）和 JVM 标志仍以一条短划线的形式输入。

3.4.1 发行版本与 3.5.1 发行版本之间更改的内容

NetBeans IDE 3.5 和 3.5.1 发行版本的开发工作主要集中在提高 IDE 的性能。启动时间和一般 UI 响应均有显著的改进。此外，还有如下更改：
  • 以前在“更新中心”上提供的几个模块已被增加至标准发布版本中。这些模块提供许多功能，其中包括支持：
    • 调试使用 JavaServer PagesTM 技术编写的 Servlet 和应用程序。
    • 创建和编辑标记库。
    • 监控 HTTP 请求。
    • WAR 文件封装。
    • 连接并浏览数据库。
  • 某些存在于先前 IDE 发行版本中的功能已从标准发布版本中删除并置于 NetBeans 更新中心中：
    • RMI 支持
    • JNDI 支持
    • XML 树编辑器
    • 脚本编写支持
    • 新建模块向导

系统需求

由于 NetBeans IDE 是用纯 Java 编写的，因此可运行在 Java 2 SDK Standard Edition 1.4 版或更高版本的任何有效实施版本上。如果使用某个 NetBeans IDE 安装程序，则必须安装 J2SDK 的 1.4.1 版或兼容版本。

启动程序和启动参数

可以使用启动程序启动 IDE。多种平台的启动程序均位于安装目录的 bin 子目录中。

对于 UNIX 系统，Bourne shell 脚本 runide.sh 是其启动程序。
对于 Microsoft Windows，请使用 runide.exe 或 runidew.exe 可执行程序。runide.exe 是 Microsoft Windows 控制台应用程序。当您运行 runide.exe 时，桌面上会打开一个控制台，同时 stderr 和 stdout 会从 NetBeans IDE 中输出。您可以按下 Ctrl-Break 获得线程转储，或按下 Ctrl-C 退出整个程序。若要将 NetBeans IDE 作为不带控制台的 Windows 应用程序来运行，请使用 runidew.exe 可执行程序。
对于 OpenVMS，runideopenvms.com 是启动程序。

启动程序加载 JVM，生成 IDE 的类路径，将类路径和某些缺省参数传递给 JVM，然后让 JVM 启动 Java 应用程序。此外，在您使用“更新中心”之后，启动程序会重新启动 IDE。

您可以使用 ${IDE_HOME}/bin/ide.cfg 文件将启动参数传递给启动程序。在开始分析命令行选项之前，启动程序尝试读取此文件。您可以将选项分为多行。

注意：在 NetBeans IDE 3.6 中，可以按照 GNU 样式输入 IDE 命令行开关，也就是说，这些开关的前面有两条短划线（例如，--jdkhome），而不是一条短划线。单字母开关（如 -h）和 JVM 标志仍以一条短划线的形式输入。

以下是可用的选项：

-h
--help
打印常用启动参数的描述。

--jdkhome jdk_home_dir
使用指定的 Java 2 SDK 版本而不使用缺省的 SDK。缺省情况下，在 Windows 系统中，加载器在 Windows 注册表中查找并使用最新的 SDK。

--cp:p additional_classpath
将指定的类路径附加至 IDE 的类路径前面。通常，建议不要使用此选项。

--cp:a additional_classpath
--cp additional_classpath
将指定的类路径附加到 IDE 的类路径后面。通常，建议仅在增加定制外观实现 JAR 时使用此选项。另一方面，您可以将 JAR 增加到 NetBeans IDE lib/ext/ 目录中。请参见联机帮助，以了解关于安装用户开发库的信息。

-Jjvm_flag
将指定的标志直接传递给 JVM。

--laf UI_class_name
使用给定的类作为 IDE 的外观。

--fontsize size
使用以点为单位的给定大小作为 IDE 用户界面的基本字体大小。

--locale language[:country[:variant]]
使用指定的语言环境。

--userdir userdir
显式指定用户目录，该目录是存储用户设置的位置。如果在 UNIX 系统中未使用该选项，则位置为 ${HOME}/.netbeans/3.6。在 Microsoft Windows 系统上，缺省位置为 Windows 配置文件区域（如 c:\Documents and Settings\yourlogin）下的 .netbeans\3.6。

--open file
将包含指定文件的目录安装到“文件系统”窗口中并在“源编辑器”中打开此文件。

--open file:line number
将包含指定文件的目录安装到“文件系统”窗口中并在“源编辑器”中的指定行打开此文件。

兼容性/升级

在您首次运行 IDE 时，您可以导入以前安装 IDE 时使用的设置。这些设置包括项目特定的设置和全局选项。如果您选择不导入以前发行版本中的设置，则 IDE 会以一组缺省的设置运行。
注意：IDE 仅能识别驻留在缺省位置的用户目录的先前安装。如果您使用 --userdir 开关指定用户目录，则安装程序不能识别此类安装。如果您想从安装程序不能识别的 IDE 上导入设置，请下载 IDE 的归档版本，而不是安装程序。

您可以导入 NetBeans IDE 3.4.x 版和 3.5.x 版中的设置，还可以导入 Sun Java Studio Standard Edition 5 中的设置。

虽然可以将以前 IDE 安装中的设置导入到 NetBeans IDE 3.6，但是并不能将 NetBeans IDE 3.6 中的所有设置导入到较早的 IDE 发行版本。

已知问题

以下是本发行版本中尚未解决的一些主要问题：
  • 39780 描述：如果在 Mac OS 中安装了 IDE 的归档发布版本，则通常关闭 IDE 的方法将无效，这可能导致以下问题：
    • 丢失工作（如果在退出 IDE 前未手动保存更改）。
    • 再次启动 IDE 时，收到 IDE 已经运行的警告。

    解决方法：下载并安装 IDE 的 .dmg 图像发布版本，它包含一个修补程序，用于解决 Mac OS 上存在的此问题。

  • 11477 描述：在 Mac OS X 上无法键入需要使用 Alt 键的字符。如果运行 IDE 的归档发布版本，则会出现此问题。

    解决方法：下载并安装 IDE 的 .dmg 图像发布版本，它包含一个修补程序，用于解决 Mac OS X 上存在的此问题。

  • “待做事项扫描”模块（它是“任务列表”模块集的一部分）现在是标准发布版本的一部分。如果在 IDE 的前一版本中安装了此模块，且将您的旧用户目录指定为 NetBeans IDE 3.6 的用户目录，则 IDE 将使用该模块的旧版本。您可以使用“更新中心”将此模块更新为 3.6 版。

  • 如果在亚洲语言环境中运行 Java Desktop System 2.0，则应使用 Java Desktop System (JDS) CD 套装附带的 NetBeans IDE 版本。（此 CD 还包含 Java 2 SDK (J2SDK) 和 Sun Java Studio 软件。）此 JDS CD 包含 J2SDK 的修补版本，该版本用于修复多字节文本的查看和输入问题。

    Java Desktop System 1.0 不支持查看或输入多字节文本。

  • 41639 描述：当对 .properties 文件使用表编辑器时，包含多字节字符的注释无法正确保存。

    解决方法：使用“编辑”命令打开 .properties 文件的文本编辑器。在文本编辑器中，必须将多字节字符输入为 Unicode 值。

  • 40769 描述：在 Mandrake Linux 9.2 中，某些安装程序窗格最初不显示内容。

    解决方法：手动调整安装程序窗格的大小，以便显示文本。

  • 39705 描述：如果安装了 Java Web Services Developer Pack（Java Web 服务开发人员软件包），则 NetBeans IDE 安装程序可能失败并显示以下消息：“由于出现以下错误，导致向导无法继续：无法加载在 /wizard.inf 中指定的向导 (104)。安装无法继续。”

    解决方法：安装 NetBeans IDE 前，移动 J2SDK 安装中的 jre/lib/endorsed/sax.jar 文件。可以在完成 NetBeans IDE 安装后将该文件移回到原来的位置。

  • 描述：在 Java Desktop System for Linux 上安装 NetBeans IDE 后，必须注销并重新登录才能使程序显示在“程序”下的“启动”菜单中。

  • 34834 描述：如果在 IDE 中执行的 ant 生成脚本的 Ant 目标非常复杂，则可能会出现内存不足的错误。这是因为在同一 JVM 中既运行 Ant 又运行 IDE，且缺省的最大堆大小不足以运行生成脚本。

    解决方法：在 IDE 安装的 bin/ 目录的 ide.cfg 配置文件中，指定一个大于缺省值的最大堆大小。例如，-J-Xmx200m。

  • 41012 描述：有时在执行较大的 CVS 签出后，会出现内存不足的错误，这将强制您关闭 IDE。

  • 40783 描述：由于 Tomcat servlet 容器中存在错误，因此部署描述符 (web.xml) 中用于指定页面编码的属性组条目必须使用不包含目录但包含 .jsp 扩展名的 URL 模式。如果只指定目录（如 /jsp_dir/* 或 /jsp_dir/*.jsp），则 IDE 的源编辑器和 Tomcat 服务器不使用指定的编码加载 JSP。以下是如何指定 URL 模式以便 Tomcat 使用正确的编码加载 JSP 的示例。 
    <jsp-config>
<jsp-property-group>
<url-pattern>*.jsp</url-pattern>
<page-encoding>
UTF-8
</page-encoding>
</jsp-property-group>
</jsp-config>

    或者，可以从 http://jakarta.apache.org/tomcat/index.html 下载以下 JAR 的最新版本并替换其中的 JAR 文件 
    <ide-install-dir>\jakarta-tomcat-5.0.19\common\lib

jasper-compiler-5.0.19.jar
jasper-runtime-5.0.19.jar

  • 40765 描述：在 PVCS 和 VSS 文件系统中，在安装文件系统后必须立即从版本控制系统中签出完整的 Web 模块。如果在签出 Web 模块前展开文件系统中的任何节点，那么将无法部署此 Web 模块。

    解决方法：如果在签出 Web 模块前展开了文件系统中的文件夹，请卸载此文件系统。然后，重新安装此文件系统并在随后立即签出 Web 模块。

  • 39817 描述：CVS 文件系统中某些文件的显示状态有时恢复为“本地”。

    解决方法：右键单击该文件系统的节点并选择“CVS”|“递归刷新”。

  • 41109 描述：在 CVS 文件系统的根目录上运行“递归刷新”（使用 IDE 的内置 CVS 客户机）后，最新文件的状态更改为“本地”。

    解决方法：要解决此问题，请执行以下操作之一：
    • 使用 IDE 的命令行支持而不是内置的客户机支持来重新安装 CVS 文件系统。
    • 使用“刷新”命令刷新 CVS 文件系统的根。可以在文件系统的子节点上使用“递归刷新”命令。

  • 25934 描述：在某些系统上，“Goto 声明”命令 (Alt-G) 有时运行得很慢。

  • 11637 描述：对内部类使用导入管理工具时，外部类的名称会从对内部类的引用中删除。

  • 32093 描述：如果您通过代理服务器访问 Web，则使用 Javadoc 管理器将 Javadoc 安装为 HTTP 文件系统的操作可能无法执行。

  • 36781 在“选项”窗口中，当试图在节点的上下文菜单中选择某个项目时，“选项”树中的节点有时会被选中。

  • 40257 在源编辑器中折叠代码时，代码行突出显示内容和注释字型被隐藏或仅部分可见。

  • 40530 调试过程中，会跳过 JSP 文档 (.jspx) 文件中的某些代码行。由于已知的服务器错误，使得并非所有 JSP 行都映射为相应的 servlet 代码。因此，调试器会跳过 JSP 文档文件中的某些行。这些被跳过的行上的断点无效，因此当按 F8 键时，调试器将忽略它们。

  • 41238 描述：联机帮助主题“将 Web 模块文件系统更改为标准文件系统”不正确。

    如果按照该主题中的说明操作，则 IDE 将在您尝试打开要编辑的文件时输出空指针错误 (NPE)。请改用下面的说明执行操作。
    1. 卸载文件系统。
    2. 在操作系统文件管理器中，移动要保存的 WEB-INF 下的任何文件，然后删除此 WEB-INF 目录。对 META-INF 目录执行相同操作。必须重命名或删除这两个目录。
