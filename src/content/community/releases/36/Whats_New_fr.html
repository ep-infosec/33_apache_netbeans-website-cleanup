Quoi de neuf dans NetBeans 3.6?

Un grand merci aux membres de la communauté NetBeans qui ont contribué à cette page .

Navigation Fenêtre

  • Disposer vos fenêtre par tirer-lacher. Tirer n'importe quelle fenêtre n'importe où dans l'EDI (excepté l'Editeur de Source) et lacher. Une boîte rouge vous indique la destination de la fenêtre.
  • Navigation de fenêtre facilitée. Pressez Ctrl-` pour passer d'une fenêtre à l'autre. Cette fonctionnalité est comme l'utilisation d'ALT-Tab sur la plupart des gestionnaires de fenêtre des systèmes d'exploitation.
  • Fenêtre liée à la tâche. Les Espaces de Travail de l'EDI NetBeans 3.5 ont été remplacé avec les fenêtres liées à la tâche, qui s'affichent automatiquement lorsque vous exécutez certaines tâches. Par exemple, la fenêtre du débogueur apparaitra automatiquement lorsque vous démarrez une session de débogage et disparaîtront lorsque vous aurez terminé la session.

Apparence et LookandFeel

  • L'EDI utilise le lookAndFeel natif pour Windows et MacOS. L'EDI utilise Aqua sur MacOS, Classis sous Windows 2000, et le thème de bureau sous Windows XP. sur toutes les autres plateformes, l'EDI utilise le LookandFeel Java (Metal). Vous pouvez définir un LookAndFeel différent en démarrant l'EDI avec le paramètre --laf nom_classe_LaF.
  • Feuille de propriétés retravaillée. La nouvelle feuille de propriété est plus compacte, fonctionne plus rapidement, et permet l'édition en un seul clic de toutes les propriétés. Il y a également une aire de description qui affiche les informations concernant la propriété sélectionnée.

Amélioration de l'Editeur

  • Code folding. Vous pouvez cacher les corps de méthodes, des commentaires et des instructions d'import, et d'autres blocs de code en les repliant dans l'Editeur de Source.
  • Méthodes Surchargées. L'Editeur de Source indique toutes les méthodes héritées dans la marge de gauche. Mettez votre souris sur l'icone pour voir quelle classe ou interface la méthode hérite.
  • Fenêtre To Do. Vous pouvez facilement tracer vos tâches et commentaires TODO en utilisant la fenêtre To Do. Vous pouvez voir toutes les tâches dans un simple fichier ou dans un répertoire sélectionné.
  • Insertion automatique de parenthèses, crochets et guillemets fermant. Par exemple, tapez System.out.println( et l'EDI ajoutera automatiquement la parenthèse fermante ).
  • Intégration Editeur de Forme. L'Editeur de Forme a été intégré en tant qu'onglet dans l'Editeur de Source. L'espace de travail d'édition de Gui a été remplacé par la fenêtre de Palette, Inspecteur et Propriétés apparaissant automatiquement lorsque l'onglet Editeur de Forme est sélectionné.
  • Imprimer en tant qu'HTML. Vous pouvez prendre n'importe quel fichier Java et l'imprimer vers un page HTML, pour pouvoir publier la source sur le web.
  • Description de l'Erreur. Maintener la souris sur le texte contenant des erreurs de compilation pour voir la description de l'erreur dans un tooltip.
  • Scinder Facilement l'Editeur de Source. Déplacer l'onglet de n'importe quel document vers le bas ou vers la marge de gauche de l'Editeur de source pour voir de multiples documents en une fois.

Développement Web/J2EE

  • Servlet 2.4 & JSP 2.0. La base de données du parseur a été mise à jour pour supporter les spécifications du Servlet 2.4 et JSP 2.0.
  • Tomcat 5. vous pouvez déployer et déboguer des applications deux-tiers J2EE 1.4 et 1.3 vers Tomcat 5 dans l'EDI.
  • Case-sensitive HTML code completion. HTML code completion now enters the tags in uppercase or lowercase instead of in uppercase only.
  • Balise d'ouverture/fermeture correspondante. Placer le curseur de texte dans n'importe quelle balise pour indiquer la balise opposée correspondante.
  • Validation JSP. L'EDI fournit une action Validation pour les JSP au lieu de l'action Compilation. L'action Validation détecte tous les problèmes de syntaxe excepté les erreurs de syntaxes qui se produisent au moment de la compilation. Ces types d'erreurs sont habituellement limitées aux fichiers qui contiennent des scriplets.

Support Ant

  • Intégration Ant 1.6.1. L'EDI est maintenant livré avec Ant 1.6.1.
  • Gestion aisée du classpath de Ant. Allez dans les Paramètres Ant de la fenêtre d'Options et ajouter n'importe quel répertoire ou librairies supplémentaires au classpath de Ant en utilisant la propriété Additional Classpath.
  • Gestion aisée des versions de Ant. Allez dans les paramètres de Ant de la fenêtre Options et sélectionnez n'importe quelle installation de Ant dans la propriété Ant Home.

Support JUnit

  • Support JUnit 3.8.1. JUnit 3.8.1 est délivré avec l'EDI NetBeans. Vous pouvez générer automatiquement les tests et test suites aussi bien dans le mode texte que dans le mode GUI.
  

Intégration Contrôle de Version

  • Design revu de CVS. Les deux supports de CVS (ligne de commande et client interne) ont été harmonisés pour utiliser les mêmes assistants, éditeurs de propriétés, et les boîtes de dialogue de commande.
  • Affichage amélioré de l'output de la commande VCS. L'output de toutes les commandes VCS sont maintenant affichés en tant qu'onglet dans la fenêtre de résultat de VCS. Vous pouvez voir la progression de la commande et arrêter n'importe quelle commande en cours d'exécution.

Système d'Aide Amélioré

  • Système d'Aide unifié. L'EDI NetBeans fusionne l'aide de tous les modules dans une seule table de matières et index pour une navigation plus aisée.
  

Autres Modifications

  • L'EDI ne tourne plus sous la version 1.3.1 du JavaTM 2 SDK, Edition Standard, et n'est plus testé sous les versions 1.4.0 et 1.4.1. La version recommandée du JDK est la 1.4.2.
  • Taille du Fichier et Date Modification ajoutés aux feuilles de propriétés pour certains noeuds.
  • Il n'y a plus de barre d'outils de filtre dans la fenêtre de débogage. Vous pouvez accéder à toutes les fenêtres du débogueur dans le menu Windows > Debugger.
  • Le dossier Startup a été enlevé de la fenêtre Options, il n'est dont plus possible de spécifier via l'interface utilisateur des classes qui doivent être exécuté dans l'EDI au moment de son chargement.
  • La Bibliothèque d'expression régulière d'Apche n'est plus comprise dans l'EDI. L'EDI utilise maintenant le paquetage java.util.regex pour le support des expressions régulières. Les classes de caractères non-standard POSIX-Like ([:javastart:] et [:javapart:]) ne sont plus supportées.
  • Il n'est plus possible d'avoir plus d'une instance de l'EDI qui s'exécutent simultanément dans le même répertoire d'utilisateur.
  • Les paramètres de ligne de commande de l'EDI peuvent maintenant être introduit dans le style GNU. Cela signifie qu'ils sont précédés de deux tirets (par exemple, --jdkhome) plutôt que d'un tiret. Les paramètres d'une seule lettre, comme -h et les paramètres de la JVM continuent d'être passé avec un tiret.
