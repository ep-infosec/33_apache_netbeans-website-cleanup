NetBeans 3.6 中的新功能

感谢提供此页的 NetBeans 社区的几位成员。

窗口导航

  • 通过拖放排列窗口。将任何窗口拖至 IDE 中的任何位置（“源编辑器”除外），然后松开鼠标。红色预览框显示窗口的目标位置。
  • 便捷的窗口导航。按 Ctrl-` 可在打开的 IDE 窗口和文档之间切换。此功能与在大多数操作系统窗口管理器中使用 Alt-Tab 类似。
  • 集成的任务相关窗口。NetBeans IDE 3.5 中的工作区已替换为任务相关窗口，这些窗口在执行某些任务时自动显示。例如，“调试器”窗口在调试会话启动时自动出现，并在会话完成后隐藏。

外观和外观风格

  • 在 Windows 和 MacOS 中，IDE 使用本机外观风格。在 MacOS 中，IDE 使用 Aqua；在 Windows 2000 中，IDE 使用 Classic；在 Windows XP 中，IDE 使用桌面主题。在所有其它平台上，IDE 使用 Java (Metal) 外观风格。要设置不同的外观风格，请使用 --laf LaF_class_name 开关启动 IDE。
  • 重新设计的属性表单。新的属性表单更为紧凑，运行更快，并且单击一次即可编辑所有属性。属性表单中还包含一个描述区域，显示有关选定属性的信息。

编辑器改进

  • 代码折叠。可以通过在源编辑器中折叠方法主体、注释、输入语句和其它代码块来将其折叠。
  • 覆盖方法。“源编辑器”在左边距处标记所有已继承的方法。将鼠标停在图标上方，可查看方法是从哪一个类或接口继承的。
  • “待做事项”窗口。使用“待做事项”窗口可以轻松地跟踪任务和 TODO 注释。您可以查看单个文件或选定目录中的所有任务。
  • 自动插入右方括号/右引号/右圆括号。源编辑器现在自动插入与所有左圆括号、左方括号和左引号对应的闭合部分，即右圆括号、右方括号和右引号。例如，键入 System.out.println ( 后 IDE 将自动输入右括号 )。
  • 窗体编辑器集成。窗体编辑器已被集成为源编辑器中的一个标签。“GUI 编辑”工作区已替换为“面板”、“检查器”和“属性”窗口，这些窗口在选择“窗体编辑器”标签时自动出现。
  • 输出为 HTML 格式。您可以使用任何 Java 文件并将其输出为 HTML 格式，以便在 Web 上发布。
  • 工具提示错误描述。将鼠标停在包含编译错误的任何文本上方，以便查看工具提示中的错误。
  • 轻松拆分源编辑器。将任何打开文档的标签拖放到源编辑器的下边距或左边距，以便同时查看多个文档。

Web/J2EE 开发

  • Servlet 2.4 和 JSP 2.0。分析器数据库已得到升级，可以支持 Servlet 2.4 和 JSP 2.0 规范。
  • Tomcat 5。在 IDE 中，您可以将两层 J2EE 1.4 和 1.3 应用程序部署到 Tomcat 5 并进行调试。
  • 区分大小写的 HTML 代码完成。HTML 代码完成现在以大写或小写形式输入标记，而不仅以大写形式输入。
  • 开放/闭合标记匹配。将您的文本插入记号放在任何标记中，以便突出显示其闭合标记。
  • 验证 JSP。IDE 为 JSP 提供“验证”操作，而不是“编译”操作。“验证”操作检测所有语法问题，转换时出现的语法错误除外。这些类型的错误通常仅出现在包含嵌入式 scriptlet 的文件中。

Ant 支持

  • Ant 1.6.1 集成。IDE 现在与 Ant 1.6.1 捆绑在一起。
  • 轻松管理 Ant 类路径。转到“选项”窗口中的“Ant 设置”，并使用“附加的类路径”属性将任何其它目录或库增加到 Ant 的类路径中。
  • 轻松管理 Ant 版本。转到“选项”窗口中的“Ant 设置”，并在“Ant 起始目录”属性中选择任何 Ant 安装。

JUnit 支持

  • JUnit 3.8.1 支持。JUnit 3.8.1 与 NetBeans IDE 捆绑在一起。在文本模式和 GUI 模式中都可以自动生成测试和测试套件。
 

版本控制集成

  • 简化的 CVS 设计。为了使用相同的向导、属性编辑器和命令对话框，已对内置 CVS 客户机和命令行 CVS 进行了统一。
  • 改进了 VCS 命令输出的显示。所有 VCS 命令的输出现在都显示为“VCS 输出”窗口中的标签。您可以查看命令的进度以及停止任何正在运行的命令。

帮助系统的改进

  • 统一帮助系统。NetBeans IDE 将所有模块的帮助合并到单个目录和索引中，以便更轻松地进行导航。
  

其它更改

  • IDE 不再在 JavaTM 2 SDK 标准版 v. 1.3.1 上运行，并且不再在 v. 1.4.0 和 1.4.1 上进行测试。建议使用 JDK 版本 1.4.2。
  • 在某些节点的属性表单中增加了“文件大小”和“修改时间”。
  • 在“调试器”窗口中不再有过滤器工具栏。可以在“窗口”>“调试器”菜单中访问所有调试器窗口。
  • “启动”文件夹从“选项”窗口中删除，因此将无法再在用户界面中指定 IDE 启动时在 IDE 中运行的类。
  • Apache 的正规表达式库不再包含在 IDE 中。IDE 现在使用 java.util.regex 包来支持正规表达式。不再支持类似非标准 POSIX 的字符类（[:javastart:] 和 [:javapart:]）。
  • 无法再在同一用户目录中同时运行多个 IDE 实例。
  • 现在能够以 GNU 样式输入 IDE 命令行开关，也就是说这些开关的前面有两条短划线（例如，--jdkhome），而不是一条短划线。单字母开关（如 -h）和 JVM 标记仍以一个短划线的形式输入。
