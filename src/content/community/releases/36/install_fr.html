Windows

Sous Windows, vous pouvez choisir parmis l'un des téléchargements suivants:

  • Installateur
  • Ditribution Archive

Utilisation de l'Installeur Windows

Une fois que vous avez télécharger le fichier d'installation, localisez l'endroit où vous l'avez sauvegardé sur votre machine locale, et double cliquez dessus pour lancer l'assistant à l'installation. L'installateur vous permet de spécifier :

  • le Java ™ 2 Software Development Kit (J2SDK) qui sera utilisé par l'EDI.
  • le répertoire où doit être installé l'EDI NetBeans.
  • s'il faut associer les fichiers .java et .nbm (NetBeans Module) à l'EDI NetBeans.

Pour Lancer l'EDI NetBeans, vous avez le choix entre :

  • double-cliquez sur l'icone de l'EDI NetBeans, située sur votre bureau.
  • Sélectionnez Démarrer | Programmes | NetBeans IDE | NetBeans IDE 3.6 | NetBeans IDE.

Utilisation des Archives sur les Machines Windows

Bien que la méthode décrite ci-dessus est la méthode préférée pour effectuer l'installation sous Windows, vous pouvez également l'installer en utilisant les distributions archivées. Deux types d'archives sont disponibles - tar.gz, and zip. Les outils courants d'archives, comme Winzip peuvent manipuler les deux types de fichiers - les explications qui suivent assument que vous avez un outil d'archive comme Winzip installé. Localisez l'archive que vous avez téléchargée, et double-cliquez dessus. Votre outil d'archive s'ouvrira, et de là extrairera tous les fichiers, préservant la structure des répertoires, vers un répertoire vide, comme C:\NetBeans 3.4.

To Launch NetBeans : Naviguez jusqu'au sous-répertoire bin/ de votre installation de NetBeans. Dans ce dossier sont situés les lanceurs de NetBeans - runide.exe est le lanceur pour Windows. Double-cliquez dessus pour démarrer NetBeans.

Installation sous Unix


  • Installateur Exécutable : Certains exécutables ".bin" de l'installateur Installshield sont disponibles pour de nombreuses plateformes Unix. Vous avez besoin de les rendre exécutable avant de les exécuter :
    $ chmod +x NetBeans.bin
    $ ./NetBeans.bin

    Bien que l'installateur recherchera les JDKs installé, et vous demandera lequel NetBeans devrait utiliser, vous pouvez accélérer l'installation en spécifiant un JDK sur la ligne de commande :

    $ ./NetBeans.bin -is:javahome <chemin_jusqu'à_votre_jdk>
  • Installations Archives Zip : Utilisez l'outil approprié sur votre plateforme pour décompresser (untar/unzip) l'archive dans un répertoire vierge. Par exemple, depuis la ligne de commande, introduisez :
    $ gzip -d NetBeans.tar.gz
    $ tar xf NetBeans.tar

    Remarque : Les utilisateurs de Solaris devraient utiliser GNU tar pour les fichiers tar.gz. Pour plus d'information, voyez l'élément dans la FAQ.

    Pour Lancer NetBeans : depuis une ligne de commande, exécutez le script de lancement runide.sh présent dans le sous-répertoire bin/ de votre installation.

Macintosh

(Mac OS X uniquement!)

  • Installation Archives Tar : Ouvrez l'application Terminal et faites un chdir vers là où vous désirez installer l'EDI. Sur la ligne de commande, tapez :
    gnutar -xvzf netbeans.tar.gz

    (en y mettant le nom de fichier de la distribution que vous venez de télécharger).

    • Remarque : N'utilisez pas StuffitExpander pour extraire NetBeans - vous aurez des problèmes avec de longs noms de fichiers incorrectement rendus!
    • Remarque : Utilisez gnutar, pas tar pour extraire l'archive.

    Pour Lancer NetBeans : Localisez le lanceur runide.sh dans le répertoire netbeans/bin créé lors de l'installation.

    Pour lancer NetBeans avec :

    • le look & feel standard Metal

      ./runide.sh -jdkhome /Library/Java/Home

    • le look & feel Aqua

      ./runide.sh -jdkhome /Library/Java/Home -ui com.apple.mrj.swing.MacLookAndFeel -fontsize 10

    Pour plus de détail, veuillez consulter le Guide de l'EDI NetBeans sous Macintosh.

Autres Systèmes d'Exploitation

  • Zip/Archive installations: Décompressez votre archive en utilisant les outils appropriés pour votre système.

    Pour Lancer NetBeans : Naviguer jusqu'au sous-répertoire bin/ de votre installation NetBeans. Dans ce dossier se trouvent les lanceurs pour les différents Systèmes d'Exploitation :

    runide.exe - Windows runidew.exe - Windows (pas de console window)
    runide.sh - Unix
    runideos2.cmd - OS/2
    runideopenvms.com - OpenVMS

    S'il n'y a aucun lanceur pour votre système d'exploitation spécifique, vous pourriez essayer d'en créer un en modifiant un des scripts le plus approprié. Si votre machine supporte JDK 1.4 ou plus, vous devriez être capable d'exécuter l'EDI. Remarque : Si vous créez un lanceur, vous devriez le contribuer au projet!

Configurer l'EDI NetBeans

Généralement, vous ne devez pas faire grand chose pour configurer l'EDI NetBeans. Il y a, cependant, quelques paramètres en ligne très pratiques pour faire des choses comme indiquer le look & feel ou la taille de la police. Pour les utiliser, éditer $NETBEANS_HOME$/bin/ide.cfget ajouter les options sur des lignes séparées. Voici quelques unes des plus utilisées:

  • -jdkhome - indique la JVM qui doit être utilisée pour exécuter l'EDI. Par exemple, -jdkhome C:\jdk1.4
  • -ui - set the pluggable look and feel class to be used, e.g. -ui com.sun.java.swing.plaf.windows.WindowsLookAndFeel
  • -fontsize - indique la taille de la police pour les menus, l'explorateur et la plupart du texte utilisé ailleur que par l'éditeur. Par exemple -fontsize=14
  • -J-Dnetbeans.windows.small_main_window=true - cache l'onglets des espaces de travail et les barres d'outils et déplace la barre de status à gauche de la fenêtre principale - pour les inconditionels des écrans.

Pour des arguments de démarrage supplémentaire, exécuter le lanceur pour votre plateforme avec l'argument -help. Par exemple :

runide.exe -help
