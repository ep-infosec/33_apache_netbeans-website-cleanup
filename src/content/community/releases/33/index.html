Key Features and Goals - The main goals of NetBeans release 3.3 are:
  • Performance improvement (most frequent negative feedback). Read more.
  • Stability - The amount of open bugs should be lower than in 3.2, when it was released. The final criteria were discussed on nbdev and they are:
    • Beta quality -
      • All known P1 bug resolved.
      • The backward compatibility of settings/project bugs resolved
    • Final Release Quality -
      • Beta quality criteria.
      • All known P1/P2 bug resolved.
      • The total number of open bugs defects not higher than 400. (This criterion is based on the requirement to release with 20% less open defects than there were at the time of the NB 3.2 release.)
  • MDR (Metadata repository) - as an experimental module
  • Projects implementation (see http://projects.netbeans.org/ps/index.html) and major project will be based on new Projects (Java module, JSP, ...) - this feature was postponed - it will be implemented in NetBeans 4.0
Detailed list of new features - is available here. Features are collected using new infrastructure.
