Notas de versão do NetBeans Mobility Pack 5.5 para CDC

Bem-vindo ao Mobility Pack do NetBeans para CDC, um pacote suplementar que fornece as ferramentas para a criação de aplicativos Java Micro Edition (Java ME) compatíveis com as tecnologias Java ME, Connected Device Configuration (CDC).

Requisitos do sistema

Sistemas operacionais suportados

O Mobility Pack do NetBeans para CDC 5.5 é executado no Java SE JDK versões 5.0 e 6.0, que consiste no Java Runtime Environment além das ferramentas do desenvolvedor para compilar, depurar e executar aplicativos escritos na linguagem Java.

O Mobility Pack do NetBeans para CDC é executado nos sistemas Microsoft Windows compatíveis com Java VM. Veja uma lista das plataformas nas quais o NetBeans IDE já foi testado.

  • Microsoft Windows XP Professional SP2
  • Microsoft Windows 2000 Professional SP4
Configuração mínima de hardware

O Mobility Pack para CDC não requer recursos significativos além daqueles necessários para o NetBeans IDE.

Observação: A resolução de tela mínima para o IDE é de 1.024x768 pixels.

  • Processador: 500 MHz Intel Pentium III ou equivalente
  • Memória: 512 MB
  • Espaço em disco: 850 MB de espaço em disco disponível
Configuração de hardware recomendada
  • Processador: 1,4 GHz Intel Pentium III ou equivalente
  • Memória: 1 GB
  • Espaço em disco: 1 GB de espaço em disco disponível
Software necessário

O NetBeans IDE é executado no J2SE JDK (Java SE Development Kit) 5.0 e 6.0, que é composto pelo Java Runtime Environment e por ferramentas de desenvolvimento para compilar, depurar e executar aplicativos escritos na linguagem Java.

Você pode fazer o download do JDK para a sua plataforma em http://java.sun.com/javase/downloads/index.jsp

Plataformas do emulador

O Sun Java Toolkit para CDC é integrado no Mobility Pack do NetBeans. Entretanto, se você programar aplicativos para outras plataformas compatíveis, será necessário fazer o download e instalar um ou mais dos SDKs a seguir:

  • Ricoh
    • Emulador Embedded Software Architecture (registro necessário para download)
  • Sony Ericsson
    • UIQ 3 SDK (logon necessário para download)
    • Pacote de extensão M600 para UIQ 3 SDK Beta 3 (download)
    • Pacote de extensão P990 para UIQ 3 SDK Beta 3 (download)
  • Nokia
    • Plataforma Nokia Series 80 SDK para sistema operacional Symbian, para Java, perfil Personal (download)
  • SavaJe
    • O sistema operacional SavaJe (registro necessário para download)

 

As instruções de instalação de cada emulador estão disponíveis no Guia de instalação do Mobility Pack NetBeans para CDC 5.5

Compatibilidade

Ao executar pela primeira vez o IDE com o Mobility Pack do NetBeans, você pode importar um subconjunto das configurações usadas no Mobility Pack do NetBeans para CDC 5.5 Beta 2. Se você optar por não importar as configurações dessa versão, o IDE será aberto usando as configurações padrão.

Observação: O IDE só reconhece instalações anteriores se o diretório de usuário estiver no local padrão. O IDE não reconhece instalações em que o diretório de usuário foi especificado em --userdir. Se quiser importar configurações de um IDE não reconhecido pelo instalador, faça o download de uma versão arquivada do IDE em vez de substituir o instalador.

Embora seja possível importar as configurações do Mobility Pack do NetBeans para CDC 5.0 para o NetBeans 5.5, não é possível importar as configurações do Mobility Pack do NetBeans para CDC 5.5 para a versão anterior do IDE.

Problemas conhecidos

Os problemas não solucionados dessa versão são os seguintes:

  • Erro #86566: Erro ao salvar as configurações da paleta.

    Descrição: As configurações da Paleta do GUI Designer não são salvas, caso você altere a plataforma do emulador do projeto.

  • Erro #88709: A compilação falha ao alternar entre as plataformas do emulador.

    Descrição: Se você alterar a plataforma de Nokia para Sony Ericsson, a compilação falha.

    Solução: Altere a UID clicando com o botão direito do mouse no projeto e escolhendo Propriedades do projeto > Geral > Sony Ericsson.

  • Erro #77083: Link do caminho absoluto dos ícones do arquivo bundle.jnlp

    Descrição: icon.small/icon.focused possui um caminho absoluto após a implantação

    Solução: O caminho pode ser especificado para os dispositivos de acordo com a documentação dos mesmos.

  • Erro #88913: Não é possível localizar o erro nbproject/build-impl.xml erro

    Descrição: A versão internacionalizada do IDE receberá uma mensagem de erro se você construir, executar e renomear um projeto.

  • Erro #90047: Problemas ao depurar projetos CDC

    Descrição: java.lang.NoClassDefFoundError é lançado durante a tentativa de depurar um projeto CDC

    Solução: A depuração não funciona se você carregar um projeto do CDC Toolkit para o Mobility Pack para CDC. Você deve atualizar o projeto para o Mobility Pack para CDC indo até as Propriedades do projeto, clicando em OK e seguindo as etapas que o IDE fornece.

  • Erro 90647: Não é possível alterar o dispositivo da plataforma Sony Ericsson UIQ no IDE

    Descrição: O dispositivo de destino da plataforma não pode ser alterado no IDE

    Solução: Execute o utilitário SDKConfig.exe localizado no <install_dir>\UIQ3SDK\epoc32\tools\distrib\ onde <install_dir> é o diretório em que o emulador do Sony Ericsson UIQ3 foi instalado no computador.

  • Erro #90654: Projetos construídos no Mobility Pack para CDC 5.5. atualização 1 falham após a atualização para a versão FCS

    Descrição: O problema é causado porque o método de execução está incorreto para a versão FCS do CDC pack

    Solução: Altere as Propriedades do projeto para o método de execução Xlet ao atualizar os projetos usando AGUI que foram criados com o Mobility Pack para CDC 5.5.1 atualização 1.
