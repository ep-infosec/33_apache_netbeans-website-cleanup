适用于 Mac OS 平台的 NetBeans 5.5 UML 建模模块发行说明

最新更新日期：2007 年 3 月 23 日

Macintosh OS 平台目前不支持 NetBeans™ 5.5 UML 建模模块发行版本。如果您要在 Macintosh OS X 平台上使用 UML 建模模块，则可以通过本发行说明文档了解有关已知问题和解决方法的重要信息。要获取有关支持平台的已知问题和解决方法的详细信息，请参见 NetBeans 5.5 UML 建模模块发行说明。

目录

  • 系统要求
  • 已知问题和解决方法

系统要求

如果系统满足 NetBeans IDE 5.5 发行说明“系统要求”部分指定的要求，则该系统可以支持 NetBeans™ 5.5 UML 建模模块，但是 Macintosh OS X 平台目前不支持 UML 建模模块。


已知问题和解决方法

如果您在 Mac OS X 平台上运行的 IDE 中使用 UML 建模模块，则会出现以下尚未解决的问题。

  • 在运行 JDK 1.5 的 Mac OS 平台上，如果图编辑器窗口中出现滚动条，则 UML 建模的性能会降低。

    描述：

      问题 #78346：在运行 JDK 1.5 的 Mac OS 平台上，如果因为手动移动图元素而在图编辑器窗口中出现滚动条，则 UML 建模的性能会降低。当出现滚动条后，在图编辑器中执行任何操作都需要很长的时间才能完成。

    解决方法：

      单击位于图编辑器工具栏上的任一布局按钮，重新设置图布局。如果图形较大，请将 -J-Dapple.awt.graphics.UseQuartz=false 命令行开关添加到
      nb5.5-installation-directory/Contents/Resources/NetBeans/etc/netbeans.conf 文件的 netbeans_default_options 语句中，然后重新启动 IDE。

      注意：使用上述命令行开关可能会对 NetBeans 5.5 IDE 和其他模块的行为产生不利的影响。在以下问题描述中也会出现这些行为：
      • 问题 #73623：文件资源管理器随机更新文件树结构中的文本，并将该文本置于其他文本的顶部。强制执行刷新命令时可恢复文件树结构的外观，但这种文件结构的混乱却时有发生。
      • 问题 #83271：将组件面板从某一位置移到另一位置后，屏幕的某些区域无法得到及时更新，会出现信息重叠的现象。
      • 问题 #88271：关闭 Quartz 呈现会导致其他问题，例如“帮助”窗口中出现的呈现问题。
      如果您在使用 -J-Dapple.awt.graphics.UseQuartz 开关时遇到了上述任何问题或其他问题，请从 netbeans.conf 文件中删除该开关，然后重新启动 IDE。
  • 在运行 JDK 1.6 的 Mac OS X 平台上，启动 IDE 会影响 UML 建模的性能。

    描述：

      问题 #82943：对于 Intel 平台上运行 JDK 1.6 的 Mac OS X 系统，当启动 IDE 时，前面部分描述的 UML 性能问题可以得到解决。然而，Power PC 平台上运行 JDK 1.6 的 Mac OS X 系统在启动 IDE 时，UML 建模模块的性能则会显著下降。

      在使用 JDK 1.6 启动 IDE 时，还会在 IDE 中出现其他问题（请参见问题 #86850）。使用 JDK 1.6 时，IDE 会出现常见的刷新问题。

    解决方法：

      使用 JDK 1.5 来启动 IDE。
  • 在运行 JDK 1.6 的 Mac OS 上，如果通过选定的元素来创建类图，则会抛出异常。
    描述：

      问题 #82831：在 Mac OS 平台上，如果通过选定的元素来创建类图，则会抛出 StackOverflowError 异常。这是一个已知的 Macintosh 错误 (ID #4647508)

    解决方法：

      使用 JDK 1.5 来启动 IDE。

     

  • 在中文和日文平台上，UML模块不能被正常安装。
    描述：

      问题 #100045：在 Mac OS 平台上，完成从更新中心对UML模块的下载和安装后，需要选择“重新启动IDE”以完成最后的安装。但是在IDE重新启动之后，UML模块并没有被安装，所以UML项目也不能被创建。

    解决方法：

      手动关闭IDE，这会导致IDE重新启动并正确安装UML模块。

     


NetBeans 5.5 UML 建模模块发行版本允许您在 IDE 中对 UML 功能进行浏览和评估。我们希望您积极地参与这项活动，将遇到的任何问题记录在问题跟踪系统 (https://netbeans.org/community/issues.html) 中，从而为我们提供有价值的建议。
