安裝 NetBeans IDE（獨立安裝程式）

以下是在每個已支援的平台上安裝獨立版本 NetBeans IDE 5.5 的說明。

Windows

在 Microsoft Windows 作業系統的電腦上，最好使用自行解壓的安裝程式來安裝 NetBeans IDE。

安裝 NetBeans IDE：

  1. 在下載安裝程式之後，雙擊安裝程式的圖示以啟動該安裝程式。

  2. 在安裝精靈中，同意“使用者授權合約同意書”，然後點選“下一步”。

  3. 指定一個空目錄用來安裝 NetBeans IDE 。

    注意：此 NetBeans IDE 將不會破壞其他 NetBeans 安裝的設定，因為它在啟動時會自動建立新的用戶目錄 (${HOME}/.netbeans/5.5)。


  4. 從JDK選項的列表中選擇希望 IDE 使用的 JDK，然後單擊“下一步”。

  5. 檢查安裝位置是否正確，以及系統上是否有足夠的空間可用於安裝。

  6. 點選“下一步”開始安裝。

注意：如果您遇到問題而無法成功完成軟體安裝，請參閱下面的疑難解答部分，以獲取有關可能影響安裝過程的未解決問題的完整描述以及建議的解決方法。


Solaris 作業系統 (Solaris OS)

在 Solaris 作業系統的電腦上，最好使用二進位安裝程式來安裝 NetBeans IDE。

安裝 NetBeans IDE：

  1. 找到含有安裝程式的目錄。

  2. 如有必要，請在命令提示符下鍵入以下內容來更改安裝程式檔的權限，以便執行該二進位檔：
    $ chmod +x your_binary_executable

    注意：請確保將 your_binary_executable 替換為您下載的二進位檔的實際檔案名。


  3. 在命令提示符下鍵入以下內容來啟動安裝程式：
    $ ./your_binary_executable

    注意：再次提醒您，請確保將 your_binary_executable 替換為您下載的二進位檔的實際檔案名。


  4. 在安裝精靈中，同意“使用者授權合約同意書”，然後點選“下一步”。

  5. 指定一個空目錄用來安裝 NetBeans IDE ，然後點選“下一步”。

  6. 從JDK選項的列表中選擇希望 IDE 使用的 JDK，然後點選“下一步”。

    安裝程式將搜索系統上已安裝的 JDK，並提示您指定它應該使用的 JDK。您也可以通過命令列指定 IDE 應該使用的 JDK。例如：

    $ ./your_binary_executable -is:javahome path_to_your_jdk

  7. 檢查安裝位置是否正確，以及系統上是否有足夠的空間可用於安裝。

  8. 點選“下一步”開始安裝。

注意：如果您遇到問題而無法成功完成軟體安裝，請參閱下面的疑難解答部分，以獲取有關可能影響安裝過程的未解決問題的完整描述以及建議的解決方法。

註冊並啟動應用伺服器

如果使用獨立版本的 NetBeans IDE 安裝程式，則需要先註冊應用伺服器，然後才能使用它。

註冊 Sun Java System Application Server

註冊 Sun Java System Application Server：

  1. 轉至“工具”>“伺服器管理器”。

  2. 在“伺服器管理器”中，點選“添加伺服器”。

  3. 在對話方塊中，選擇 "Sun Java System Application Server 9"，然後點選“下一步”。

  4. 為伺服器實體輸入名稱。此名稱用於 IDE 中辨認伺服器。

  5. 輸入位置、使用者名稱和密碼。如果要註冊遠端實體的伺服器，則無需輸入安裝位置和網域。您必須先註冊應用伺服器的本地安裝，然後才能註冊遠端實體。

  6. 點選“完成”。

注意：如果使用套裝版本安裝程式，則會自動安裝和配置 Sun Java System Application Server，因此在這種情況下不必執行上述註冊過程。


啟動 Sun Java System Application Server

啟動應用伺服器：

  1. 在“運行環境”視窗中，右鍵點選新建立的 localhost 節點，然後從浮動右鍵選單中選擇“啟動/停止伺服器”。

  2. 在“伺服器狀態”對話方塊中，點選“啟動伺服器”。

  3. 點選“關閉”以關閉該對話方塊。

注意：預設情況下，NetBeans IDE 和 Sun Java System Application Server 套裝版本將建立具有 adminadmin 密碼的 admin 使用者。使用者應使用這些設定透過 Web 控制台進行登錄。

已壓縮的發行版本

如果沒有針對您所使用的作業系統的安裝程式，您可以通過下載並安裝已壓縮的發行版本來使用 NetBeans IDE。如果要測試內部開發版本，也可以安裝 IDE 已壓縮的發行版本。

安裝 NetBeans IDE 已壓縮的發行版本：

  1. 找到含有壓縮檔的目錄。

  2. 使用適於系統的工具程式來解壓縮壓縮檔。

啟動 NetBeans IDE 已壓縮的發行版本：

  1. 找到 NetBeans IDE 安裝的 bin 子目錄。

  2. 執行適於系統的啟動程式。

已壓縮的發行版本中包含以下啟動程式：

  • netbeans.exe - Windows
  • nb.exe - Windows（透過控制台視窗）
  • netbeans - UNIX、Linux
  • NetBeans - Mac OS X
  • netbeans.cmd - OS/2

如果沒有針對您的作業系統的特定啟動程式，您可能需要透過修改對應的腳本來建立一個啟動程式。但是，如果您的電腦支援 JDK 1.5.0 或更高版本，則應該能夠執行 IDE。

注意：如果建立了啟動程式，您可以將其提供給 NetBeans 項目。有關詳細資訊，請參閱向 netbeans.org 投稿。

移除 NetBeans IDE

如有必要，您可以使用以下步驟移除 NetBeans IDE。

從 Windows 中移除 NetBeans IDE 5.5

移除 NetBeans IDE：

  1. 從系統的“開始”功能表中選擇“設定”>“控制台”。

  2. 雙擊“新增或移除程式”圖示。

  3. 從列表中選擇 "NetBeans IDE 5.5"，然後點選“移除”。

移除 Sun Java System Application Server Platform Edition 9：

  1. 從系統的“開始”功能表中選擇“設定”>“控制台”。

  2. 雙擊“新增或移除程式”圖示。

  3. 從列表中選擇 "Sun Java System Application Server Platform Edition 9"，然後點選“移除”。

從 Solaris（SPARC 和 x86 Platform Editions）中移除 NetBeans IDE 5.5

移除 NetBeans IDE：

  1. 找到 NetBeans IDE 安裝目錄的 _uninst 子目錄。

  2. 鍵入 ./uninstaller 以執行 InstallShield 移除程式。

移除 Sun Java System Application Server：

  1. 找到 Sun Java System Application Server Platform Edition 9 的安裝目錄。

  2. 在終端視窗中，鍵入 ./uninstall 以執行移除程式。

從 Linux 中移除 NetBeans IDE 5.5

移除 NetBeans IDE：

  1. 找到 NetBeans IDE 安裝目錄的 _uninst 子目錄。

  2. 在終端視窗中，鍵入 ./uninstaller 以執行 InstallShield 移除程式。

移除 Sun Java System Application Server：

  1. 找到 Sun Java System Application Server Platform Edition 9 的安裝目錄。

  2. 在終端視窗中，鍵入 ./uninstall 以執行移除程式。

從 Macintosh OS X 中移除 NetBeans IDE 5.5

移除 NetBeans IDE：

  1. 找到 NetBeans IDE 的安裝位置。預設情況下，NetBeans 安裝在“應用程式”目錄中。

  2. 右鍵點選 NetBeans 圖示，然後從彈出式功能表中選擇“顯示包內容”。

  3. 在出現的視窗中，找到 _uninst 子目錄。

  4. 在終端視窗中，鍵入 ./uninstaller 以執行 InstallShield 移除程式。

移除 Sun Java System Application Server：

  1. 找到 Sun Java System Application Server Platform Edition 9 的安裝目錄。預設情況下，應用伺服器安裝在 home 目錄的 SUNWappserver 子目錄中。

  2. 在終端視窗中，鍵入 ./uninstall 以運行移除程式。

  • 問題 #26965

    描述：

    如果 JDK 安裝中存在不相容的 sax.jar 版本，則 IDE 安裝程式將會出現錯誤，並顯示以下訊息：“由於出現以下錯誤，導致安裝精靈無法繼續執行：無法裝入在 /wizard.inf 中指定的精靈 (104)”

    解決方法：

    在 IDE 安裝過程中，暫時將 jre/lib/endorsed/sax.jar 從 JDK 目錄中移除。


  • 問題 #26965

    描述：

  • 問題 #65482（僅限 Linux）

    描述：

    在某些 Mandrake/Mandriva Linux 發行版本上無法安裝 NetBeans。初始化期間，安裝程式會顯示以下消息： 

    Initializing InstallShield Wizard........
    ./netbeans-5_0-bin-linux.bin: line 189: bc: command not found
    ./netbeans-5_0-bin-linux.bin: line 404: [: : integer expression expected

    解決方法：

    NetBeans 安裝程式使用 GNU 的 bc 應用程式，預設情況下該實用程式似乎並未安裝在 Mandriva 中。如果遇到此問題，請先手動安裝 GNU 的 bc 應用程式，然後再安裝 NetBeans。如果存在有效的 Internet 連接，則可以通過在命令行中運行 urpmi bc 完成此操作。請注意，也可以下載合適的 RPM，然後使用 rpm -i <bc_package>.rpm 命令進行安裝。


  • 問題 #67530

    描述：

    啟動安裝程式後，出現消息“歡迎使用 ERROR: null 安裝精靈”。這可能是由於同時運行多個安裝程式或者在一個安裝程式退出失敗後運行另一安裝程式而導致的。

    解決方法：

    刪除 \InstallShield\Universal\Common 目錄的內容，然後重新啟動安裝程式。

    在 Windows 作業系統上，此目錄位於 C:\Program Files\Common files\InstallShield\Universal\Common。

    在 Solaris 和 Linux 作業系統上，此目錄位於 /$HOME/InstallShield/Universal/Common。

  • 問題 #52012（僅限 Windows XP 系統上的土耳其語言環境）

    描述：

    對於設置為使用土耳其語言環境的系統，無法成功完成安裝程式。

    解決方法：

    取消安裝程式。在您的系統上，切換到“英語（美國）”語言環境，然後重新啟動安裝程式。

    在 Windows XP 系統上，您可以通過執行以下操作來切換語言環境：選擇“開始”>“控制台”，雙擊“地區及語言選項”，然後從對話方塊中“標準及格式”部分的下拉清單中選擇語言環境。
