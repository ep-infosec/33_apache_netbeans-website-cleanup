Guia de instalação do NetBeans Mobility Pack para CDC 5.5.1

O NetBeans Mobility Pack para CDC 5.5.1 fornece um ambiente para construir rapidamente aplicativos do Java Micro Edition (Java ME) que dão suporte a dispositivos usando a Connected Device Configuration (CDC) e os perfis Pessoal, Fundação e AGUI.

Conteúdo

  • Pré-requisitos
  • Instalando o software
  • Instalando plataformas de emulador
  • Desinstalando o software
  • Solução de problemas

 

Observação: Para obter informações sobre os requisitos do sistema para o software incluído neste pacote, consulte as Notas da versão do NetBeans Mobility Pack para CDC 5.5.1.

Pré-requisitos

Este tópico especifica os pré-requisitos que devem ser cumpridos antes de você iniciar a instalação do NetBeans Mobility Pack para CDC 5.5.1.

Observe que o NetBeans Mobility Pack para CDC não pode ser executado no NetBeans IDE 5.5. Você deve usar o NetBeans IDE 5.5.1.

Faça o download do NetBeans Mobility Pack para CDC 5.5.1 na Página de downloads netbeans.org.

Software necessário

Informações detalhadas sobre hardware, software e requisitos do sistema operacional encontram-se nas Notas da versão do NetBeans Mobility Pack para CDC 5.5.1.

É necessário que os softwares abaixo estejam instalados antes da instalação do NetBeans Mobility Pack para CDC 5.5.1.

  • Java SE Development Kit (JDK) 5.0 ou 6.0
  • NetBeans IDE 5.5.1

O Sun Java Toolkit para CDC está integrado ao NetBeans Mobility Pack. No entanto, se desejar programar aplicativos para outras plataformas suportadas, é necessário fazer o download de um ou mais dos seguintes SDKs e instalá-los:

  • Ricoh
    • Embedded Software Architecture Emulator (Registro necessário para o download)
  • Sony Ericsson
    • UIQ 3 SDK (logon necessário para o download)
    • M600 Extension Package para o UIQ 3 SDK Beta 3 (download)
    • P990 Extension Package para o UIQ 3 SDK Beta 3 (download)
  • Nokia
    • Nokia Series 80 Platform SDK para Symbian OS, para Java, perfil Pessoal (download)
  • NSIcom (CrEme para Windows CE)
    • CrEme 4.10 Developer Support (download)
    • Microsoft Device Emulator 1.0 (download)
    • Virtual Machine Network Driver para Microsoft Device Emulator (download)
    • ActiveSync (download)
  • SavaJe
    • A plataforma SavaJe Mobile (não mais disponível)

Instalação do software

Seguem-se instruções para instalar o Mobility Pack do NetBeans para CDC 5.5.1. Estas instruções pressupõem que você já tenha instalado o Java SE Development Kit e o NetBeans IDE 5.5.1.

Para instalar o Mobility Pack NetBeans para CDC 5.5.1:

  1. Faça o download do arquivo instalador em http://services.netbeans.org/downloads/index.php?p=10.
  2. Navegue o sistema até o local do arquivo instalador e clique duas vezes no ícone do instalador para iniciar o instalador.
  3. Na página de boas-vindas do assistente para instalação, clique em Próximo.
  4. Aceite o contrato de licença e clique em Próximo.
  5. Na página do diretório de instalação do assistente para instalação, especifique o local de instalação do NetBeans IDE 5.5.1.
  6. O assistente para instalação procura as instalações compatíveis do JDK. Se o JDK não estiver no local padrão, será necessário especificar o local do Java JDK 1.5.0 ou superior. Você também pode digitar o caminho de outro JDK instalado ou usar o botão Explorar para ir até outra instalação e selecioná-la.
  7. Quando a instalação estiver concluída, clique em Finalizar para sair do assistente.

O diretório de projetos padrão é C:\Documentos e configurações\nome de usuário. Se você desinstalar o Mobility Pack do NetBeans para CDC, este diretório não será excluído e será identificado na próxima vez que você instalar o Mobility Pack para CDC.

Observação: Se encontrar problemas durante a instalação do software, consulte Solução de problemas para obter descrições completas e soluções sugeridas para problemas não resolvidos que poderiam afetar a instalação.

 

Instalando o Mobility Pack do NetBeans para CDC 5.5.1 em plataformas não-Windows

Atualmente o Mobility Pack para CDC não está disponível para plataformas não-Windows devido à inexistência de suporte de emuladores de terceiros para outras plataformas. No entanto, oferecemos um zip binário não suportado do pacote para quem desejar instalar o pacote em outros sistemas operacionais. Observe que não há documentação ou suporte para o zip binário. Para obter informações sobre o download e a instalação do zip binário, consulte este tópico na Página de perguntas freqüentes. A Lista de usuários do Mobility também está disponível para a solução de outras dúvidas relacionadas a este tópico.

Instalando plataformas de emulador

Para desenvolver aplicativos CDC, é necessário ter uma plataforma CDC registrada no IDE. As instruções a seguir descrevem o processo para instalar cada plataforma de emulador que atualmente tem suporte do IDE.

Sun Java Toolkit para CDC

O Sun Java Toolkit para CDC está integrado ao NetBeans Mobility Pack, por isso não é necessário fazer o download deste emulador e instalá-lo.

No entanto, se desejar ter uma instalação separada do kit de ferramentas, siga estas instruções:

  1. Faça o download do Sun Java Toolkit para CDC 1.0 na Sun Developer Network (download).
  2. Instale o emulador de plataforma de acordo com as instruções fornecidas.
  3. No Menu principal, escolha Ferramentas > Gerenciador de plataformas Java.
  4. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  5. Selecione Emulador da plataforma CDC Java Micro Edition e clique em Próximo.
  6. Selecione a pasta que contém a plataforma SDK. O diretório padrão é C:\CDCTK10\. Clique em Próximo.
  7. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.
  8. Clique em Finalizar. O IDE registra a plataforma CDC.

Ricoh Embedded Software Architecture

O Mobility Pack para CDC oferece suporte a Ricoh Embedded Software Architecture. Para integrar suporte para esta arquitetura, siga estas instruções:

  1. Faça o download dos seguintes arquivos no site Ricoh Developer Program Site:
    • Embedded Software Architecture Emulator (logon necessário para o download)
  2. Descompacte o arquivo Embedded_Software_Architecture_Emulator_1.14c.zip.
    Os arquivos serão extraídos em uma pasta <raiz>\Embedded_Software_Architecture_Emulator_1.14c\.
  3. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  4. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  5. Selecione Emulador da plataforma CDC Java Micro Edition e clique em Próximo.
  6. Selecione a pasta que contém a plataforma SDK. O diretório padrão é C:\Embedded_Software_Architecture_Emulator_1.14c\. Clique em Próximo.
  7. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.
  8. Clique em Finalizar. O IDE registra a plataforma CDC.

Sony Ericsson CDC Platform para o UIQ SDK 3

O Mobility Pack para CDC dá suporte a Sony Ericsson CDC Platform 1. Para integrar o suporte para os dispositivos das séries M600 e P990, siga estas instruções:

  1. Faça o download dos seguintes arquivos no site developer.sonyericsson.com:
    • UIQ 3 SDK (logon necessário para o download)
    • M600 Extension Package para o UIQ 3 SDK (download)
    • P990 Extension Package para o UIQ 3 SDK (download)
  2. Descompacte o arquivo UIQ3SDK_final.zip.
  3. Navegue até o local no sistema em que os arquivos foram extraídos e clique duas vezes em UIQS3SDK.exe.
    É aberto o assistente para instalação de UIQ 3 SDK.
  4. Siga as instruções do assistente para instalar o SDK.
    O diretório padrão para a instalação é C:\Symbian\UIQ3SDK\
  5. Para instalar os pacotes de extensão M600 e P990, navegue até os locais de download no sistema e clique duas vezes nos ícones do instalador para iniciar os assistentes de instalação.
  6. Siga as instruções do assistente para instalar cada SDK.
  7. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  8. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  9. Selecione Emulador da plataforma CDC Java Micro Edition e clique em Próximo.
  10. Selecione a pasta que contém a plataforma SDK. O diretório padrão é C:\Symbian\UIQ3SDK\. Clique em Próximo.
  11. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.

    Observação: Se vir a mensagem "A platforma não está configurada corretamente. Use as ferramentas de configuração do SDK para instalar as ferramentas obrigatórias", um estilo de emulador não foi selecionado.

    Para corrigir isso:
    1. Clique na guia Configurações e clique no botão Ferramenta de configuração do SDK.
    2. Em Configuração do SDK, clique na guia Estilo e escolha um estilo na lista Estilos de emulador. Escolha Aplicar estilos.
    3. Feche o Configurador do SDK.
  12. Clique em Finalizar. O IDE registra as plataformas CDC.

Nokia Series 80 Platform SDK para SO Symbian, para Java, perfil Pessoal

O Mobility Pack para CDC dá suporte a Series 80 Platform SDK para Symbian OS, para Java, perfil Pessoal. Para integrar suporte para os dispositivos Nokia Series 80, siga estas instruções:

  1. Faça o download de Nokia Series 80 Platform SDK para Symbian OS, para Java, perfil Pessoal, na Sun Developer Network (download) O nome do arquivo é S80_DP_2_0_PP_SDK.zip.
  2. Descompacte o arquivo.
  3. Navegue até o local no sistema em que o arquivo foi descompactado. Clique duas vezes no ícone do arquivo setup.exe.
    O assistente do instalador de Nokia é aberto.
  4. Siga as instruções do assistente para instalar o SDK.
    O assistente irá solicitar a você que escolha o dispositivo que será o dispositivo padrão. Escolha o Series80_DP2_0_PP_SDK.
  5. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  6. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  7. Selecione Emulador da plataforma CDC Java Micro Edition e clique em Próximo.
  8. Selecione a pasta que contém a plataforma SDK. O diretório padrão é C:\Symbian\S80_DP2_0_PP_SDK. Clique em Próximo.
  9. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.
  10. Clique em Finalizar. O IDE registra a plataforma CDC.

CrEme VM para Windows CE

O Mobility Pack para CDC oferece suporte ao NSIcom CrEme VM para Windows CE. Para integrar suporte para dispositivos do Windows CE, siga estas instruções:

  1. Faça o download da plataforma CrEme 4.10 em NSIcom (download)
  2. Navegue até o local no sistema em que você fez o download do arquivo CrEmeDevSup410.exe. Clique duas vezes no arquivo.
  3. Siga as instruções do assistente NSI.com InstallShield para instalar o Developer Kit.
  4. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  5. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  6. Selecione Emulador da plataforma CDC Java Micro Edition e clique em Próximo.
  7. Selecione a pasta que contém a plataforma SDK. O diretório padrão é C:\Arquivos de programas\NSIcom\CrE-ME V4.10. Clique em Próximo.
  8. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover bibliotecas de recursos adicionais.
  9. Clique em Finalizar. O IDE registra a plataforma CDC.

  3. Faça o download de efp.msi (download).O assistente de instalação do Emulador de dispositivo irá orientar você no processo de instalação.
  4. Talvez também seja necessário instalar o Virtual Machine Network Driver para Microsoft Device Emulator (download) Faça o download do arquivo e clique duas vezes no arquivo netsvwrap.msi. O assistente de instalação irá orientar você no processo de instalação.

Plataforma SavaJe Mobile

O Mobility Pack para CDC dá suporte à criação de Xlets usando o sistema operacional SavaJe. O suporte para o SavaJe está incorporado ao Sun Java Wireless Toolkit para CDC. Uma vez que o Jasper S20 está disponível atualmente apenas como uma edição de visualização, não há uma cobertura de emulador disponível para o Jasper S20. Em vez disso, você pode usar o dispositivo DefaultColorPhone para simular o comportamento do Jasper S20.

Se não tiver um Jasper S20, estas instruções mostram como implantar o aplicativo para o dispositivo. Eis o que é necessário:

  • O telefone de Jasper S20.
  • O sistema operacional SavaJe (OS) (não mais disponível)
  • O software GSPDA Mobile Device (drivers USB MCCI).

Configurando o fone

As etapas a seguir descrevem o processo de instalação do software necessário para o tutorial sobre o telefone do Jasper S20.

Instalando o sistema operacional SavaJE

OBSERVAÇÃO: Lamentavelmente, o SavaJe OS não está mais disponível para download. Estas instruções estão disponíveis para quem tem acesso ao software SavaJe Mobile Platform através de CD ou outras fontes. Este procedimento foi escrito para o fone Jasper S20 que teve uma versão anterior do SavaJe OS instalada e permite que você retenha preferências existentes e configurações de fábrica.

Três compilações do SavaJe OS foram produzidas:

  • JasperS20.zip
    A compilação comercial entregue no telefone.
  • JasperS20_EA_Developer.zip
    Compilação do desenvolvedor JavaOne Early Access sem ativação do Java Debug Wire Protocol (JDWP).
  • JasperS20_EA_Developer_debug.zip
    Compilação do desenvolvedor de acesso JavaOne com ativação do JDWP. Para executar um pacote com a depuração do JDWP, é necessário carregar esta compilação.

 

Siga estas etapas para instalar uma compilação atualizada do SavaJe OS no telefone.

  1. Extraia (descompacte) o arquivo zip que contém a compilação desejada na raiz da placa mini-SD.
  2. Conecte um cabo de força/USB ao fone.
  3. Ligue o fone enquanto pressiona a tecla Câmera.
  4. Na tela de menu do atualizador, selecione "Factory Reset" [restaurar configurações de fábrica] movendo o joystick para baixo até "Factory Reset" ser realçado. Incie a reconfiguração de fábrica movendo o joystick para a direita. Use o joystick para navegar até "Yes" e, em seguida, selecione "Yes" movendo o joystick para a direita.
    O fone deve começar a carregar a nova compilação. Após a conclusão do carregamento, você poderá ser solicitado a pressionar qualquer tecla para reiniciar o telefone.
  5. O fone é reiniciado e exibe a tela Xplore. Se a tela de carregamento da bateria for exibida durante a reinicialização, pressione o botão de energia para continuar a reinicializar o telefone.

A primeira vez que o fone for reiniciado após a reconfiguração de fábrica, será exibido um menu de seleção de Idioma/Localização. Faça as seleções navegando com o joystick e pressionando o botão do joystick. Você também será solicitado a definir a hora.

Desinstalando o software

Esta seção contém informações para a desinstalação do NetBeans Mobility Pack para CDC 5.5.1 do sistema.

  1. Feche o IDE.
  2. No Painel de controle, inicie Adicionar ou remover programas, selecione NetBeans Mobility Pack para CDC 5.5.1 e clique em Alterar/Remover.
  3. Siga as instruções do desinstalador.

Observação: Você também pode desinstalar quaisquer plataformas do Emulador através do Painel de controle do Windows.

Solução de problemas

Se tiver problemas ao instalar ou executar o software, consulte a lista de problemas abaixo para ver as soluções. Para obter uma lista completa dos problemas conhecidos, dos requisitos do sistema e de informações de configuração, consulte as Notas da versão.

Se encontrar um problema que não esteja listado nas notas da versão ou quiser solicitar uma melhoria, registre o problema no sistema de controle de erros do NetBeans, IssueZilla, em https://netbeans.org/community/issues.html. Para problemas do NetBeans Mobility Pack, selecione o componente "mobility", selecione a versão "5.5.1" e, em seguida, selecione o subcomponente "cdc pack". Se não tiver certeza do nome correto do subcomponente, escolha "other".

Se já estiver conectado a netbeans.org, vá diretamente para a tela de entrada de problemas do Mobility Pack: https://netbeans.org/bugzilla/enter_bug.cgi?component=mobility&version=5.5.1.
