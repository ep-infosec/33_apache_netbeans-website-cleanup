Instalação do NetBeans IDE (Instalador independente)

Seguem as instruções para a instalação do NetBeans IDE independente nas plataformas compatíveis.

Windows

Em computadores com o Microsoft Windows, usar o instalador compactado auto-extraível apresenta um método preferencial de instalar o NetBeans IDE.

Para instalar o NetBeans IDE:

  1. Depois de fazer o download do arquivo instalador, clique duas vezes no ícone para abrir o instalador.

  2. No assistente de instalação, escolha uma resposta para o Contrato de licença e clique em Próximo.

  3. Especifique um diretório vazio dentro do qual será instalado o NetBeans IDE.

    Observação: Esta instalação do NetBeans IDE não desorganizará as configurações de outras instalações do NetBeans, porque o IDE cria automaticamente um novo diretório de usuário quando é aberto (${HOME}/.netbeans/5.5).


  4. Escolha o JDK que o IDE usará na lista de JDKs compatíveis e clique em Próximo.

  5. Verifique se o local de instalação está correto e se há espaço disponível no sistema para a instalação.

  6. Clique em Próximo para iniciar a instalação.

Observação: Caso haja problemas durante a instalação do software, consulte a seção Solução de problemas abaixo para obter as descrições completas e as soluções alternativas sugeridas para problemas não resolvidos que podem afetar o processo de instalação.


Sistema operacional Solaris (Solaris OS)

Em computadores com o sistema operacional Solaris, o método de preferência para instalar o NetBeans IDE é usar o instalador binário.

Para instalar o NetBeans IDE:

  1. Vá até o diretório que contém o instalador.

  2. Se necessário, altere as permissões do arquivo instalador para tornar o binário executável. Para tanto, digite em um prompt de comando:
    $ chmod +x seu_binário_executável

    Observação: Certifique-se de substituir seu_binário_executável pelo nome real do arquivo do binário que foi baixado.


  3. Abra o instalador executando em um prompt de comando:
    $ ./seu_binário_executável

    Observação: Verifique novamente se substituiu seu_binário_executável pelo nome real do arquivo do binário que foi baixado.


  4. No assistente de instalação, escolha uma resposta para o Contrato de licença e clique em Próximo.

  5. Especifique um diretório vazio dentro do qual será instalado o NetBeans IDE e clique em Próximo.

  6. Escolha o JDK que o IDE usará na lista de JDKs compatíveis e clique em Próximo.

    O instalador procura os JDKs instalados no sistema e pergunta se deseja especificar qual será utilizado. É possível fazer isso na linha de comando. Por exemplo:

    $ ./seu_binário_executável -is:javahome caminho_para_seu_jdk

  7. Verifique se o local de instalação está correto e se há espaço disponível no sistema para a instalação.

  8. Clique em Próximo para iniciar a instalação.

Observação: Caso haja problemas durante a instalação do software, consulte a seção Solução de problemas abaixo para obter as descrições completas e as soluções alternativas sugeridas para problemas não resolvidos que podem afetar o processo de instalação.

Registrando e iniciando o servidor de aplicativos

Se você usou o instalador independente do NetBeans IDE, é preciso registrar o servidor de aplicativos antes de utilizá-lo.

Registrando o servidor de aplicativos Sun Java System

Para registrar o servidor de aplicativos Sun Java System:

  1. Vá a Ferramentas > Gerenciador de servidores.

  2. No Gerenciador de servidores, clique em Adicionar servidor.

  3. Na caixa de diálogo, selecione o servidor de aplicativos Sun Java System 9 e clique em Próximo.

  4. Insira um nome para a instância do servidor. Esse nome é usado para identificar o servidor no IDE.

  5. Insira o local, o nome de usuário e a senha. Você não precisa inserir o local de instalação e o domínio se estiver sendo feito o registro de uma instância remota do servidor. Além disso, é necessário registrar uma instalação local do servidor de aplicativos antes de registrar instâncias remotas.

  6. Clique em Finalizar.

Observação: Caso o instalador de pacote tenha sido usado, o servidor de aplicativos Sun Java System foi instalado e configurado automaticamente, portanto não é necessário realizar o processo de registro descrito acima.


Iniciando o servidor de aplicativos Sun Java System

Para iniciar o servidor de aplicativos:

  1. Na janela Tempo de execução, clique com o botão direito do mouse no nó de host local recém-criado e escolha Iniciar/Parar servidor no menu de contexto.

  2. No diálogo Status do servidor, clique em Iniciar servidor.

  3. Clique em Fechar para sair da caixa de diálogo.

Observação: O pacote NetBeans IDE e servidor de aplicativos Sun Java System cria o usuário admin e a senha adminadmin por padrão. Os usuários devem usá-los para fazer logon pelo console da Web.

Distribuição de arquivos

Caso esteja usando um sistema operacional para o qual não há instalador, é possível usar o NetBeans IDE fazendo o download e instalando distribuições de arquivo. Também é possível instalar distribuições de arquivo do IDE caso pretenda testar construções de desenvolvimento.

Para instalar distribuições de arquivo do NetBeans IDE:

  1. Vá até o diretório que contém o arquivo.

  2. Descompacte o arquivo utilizando os utilitários apropriados para o sistema.

Para abrir distribuições de arquivo do NetBeans IDE:

  1. Vá até o subdiretório bin da instalação do NetBeans IDE.

  2. Execute o iniciador apropriado para o sistema.

Os seguintes iniciadores estão incluídos nas distribuições de arquivo:

  • netbeans.exe - Windows
  • nb.exe - Windows (dentro de uma janela de console)
  • netbeans - UNIX, Linux
  • NetBeans - Mac OS X
  • netbeans.cmd - OS/2

Caso não haja um iniciador específico para seu sistema, é possível criar um modificando o script adequado. No entanto, se o equipamento for compatível com JDK 1.5.0 ou superior, é provável que consiga executar o IDE.

Observação: Você pode contribuir para o projeto NetBeans caso crie um iniciador. Para obter mais informações, consulte Contribuindo para o netbeans.org.

Desinstalando o NetBeans IDE

Se necessário, é possível desinstalar o NetBeans IDE por meio dos procedimentos a seguir.

Desinstalando o NetBeans IDE 5.5 do Windows

Para desinstalar o NetBeans IDE:

  1. No menu Iniciar do sistema, escolha Painel de controle.

  2. Clique duas vezes no ícone Adicionar ou remover programas.

  3. Selecione o NetBeans IDE 5.5 na lista e clique em Remover.

Para desinstalar o Sun Java System Application Server Platform Edition 9:

  1. No menu Iniciar do sistema, abra o Painel de controle.

  2. Clique duas vezes no ícone Adicionar ou remover programas.

  3. Selecione Sun Java System Application Server Platform Edition 9 na lista e clique em Remover.

Desinstalando o NetBeans IDE 5.5 do Solaris (Edições das plataformas SPARC e x86)

Para desinstalar o NetBeans IDE:

  1. Vá até o subdiretório _uninst da pasta de instalação do NetBeans IDE.

  2. Digite ./uninstaller para executar o desinstalador InstallShield.

Para desinstalar o servidor de aplicativos Sun Java System:

  1. Vá até a pasta de instalação do Sun Java System Application Server Platform Edition 9.

  2. Em uma janela de terminal, digite./uninstall para executar o desinstalador.

Desinstalando o NetBeans IDE 5.5 do Linux

Para desinstalar o NetBeans IDE:

  1. Vá até o subdiretório _uninst da pasta de instalação do NetBeans IDE.

  2. Em uma janela de terminal, digite./uninstall para executar o desinstalador InstallShield.

Para desinstalar o servidor de aplicativos Sun Java System:

  1. Vá até a pasta de instalação do Sun Java System Application Server Platform Edition 9.

  2. Em uma janela de terminal, digite./uninstall para executar o desinstalador.

Desinstalando o NetBeans IDE 5.5 do Macintosh OS X

Para desinstalar o NetBeans IDE:

  1. Vá até o local de instalação do NetBeans IDE. Por padrão, o NetBeans é instalado no diretório Aplicativos.

  2. Clique com o botão direito do mouse no ícone do NetBeans e escolha no menu pop-up Mostrar conteúdo do pacote.

  3. Na janela que aparece, vá até o subdiretório_uninst.

  4. Em uma janela de terminal, digite./uninstall para executar o desinstalador InstallShield.

Para desinstalar o servidor de aplicativos Sun Java System:

  1. Vá até a pasta de instalação do Sun Java System Application Server Platform Edition 9. Por padrão, o servidor de aplicativos é instalado no subdiretório SUNWappserver do seu diretório-base.

  2. Em uma janela de terminal, digite./uninstall para executar o desinstalador.

Solução de problemas

Se tiver problemas ao instalar ou executar o software, consulte a lista de problemas abaixo para ver as soluções. Para obter uma lista completa dos problemas conhecidos, dos requisitos do sistema e outras informações úteis, consulte Notas da versão.

Encontram-se abaixo os problemas não solucionados desta versão que podem afetar ou interromper o processo de instalação:

  • Erro #26965

    Descrição:

    Ao usar as instalações do JDK que incluem uma versão incompatível do sax.jar, o instalador do IDE trava e aparece a seguinte mensagem: "O assistente não pode prosseguir por causa do seguinte erro: não é possível carregar o assistente especificado em /wizard.inf (104)"

    Solução:

    Remover temporariamente jre/lib/endorsed/sax.jar do diretório do JDK durante a instalação do IDE.


  • Erro #65482 (apenas no Linux)

    Descrição:

    Não é possível instalar o NetBeans em algumas distribuições do Mandrake/Mandriva Linux. Durante a instalação, o instalador exibe a seguinte mensagem: 

    Initializing InstallShield Wizard........
    ./netbeans-5_0-bin-linux.bin: line 189: bc: command not found
    ./netbeans-5_0-bin-linux.bin: line 404: [: : integer expression expected

    Solução:

    O instalador do NetBeans utiliza o utilitário bc do GNU, que pode não ser instalado por padrão no Mandriva. Caso tenha esse problema, instale o utilitário bc do GNU manualmente antes de instalar o NetBeans. Se tiver conexão ativa à Internet, é possível executar urpmi bc na linha de comando para fazer a instalação. Também é possível fazer o download do RPM adequado e instalá-lo por meio do comando rpm -i <bc_package>.rpm.


  • Erro #67530

    Descrição:

    Depois de iniciar o instalador, aparece a mensagem "Bem-vindo ao assistente de instalação para ERRO: nulo". Esse problema pode ser causado pela execução simultânea de vários instaladores ou pela execução do instalador depois que outro não foi fechado de forma correta.

    Solução:

    Exclua os conteúdos do diretório \InstallShield\Universal\Common e reinicie o computador.

    Em sistemas Windows, esse diretório está localizado em C:\Arquivos de programas\Arquivos comuns\InstallShield\Universal\Common.

    Em sistemas Solaris e Linux, esse diretório está localizado em /$HOME/InstallShield/Universal/Common.

  • Erro #52012 (somente em sistemas Windows XP em turco)

    Descrição:

    Para sistemas configurados no idioma turco, o instalador não é concluído com êxito.

    Solução:

    Cancele o instalador. No sistema, altere o idioma para inglês (Estados Unidos) e reinicie o instalador.

    Em sistemas Windows XP, é possível fazer essa alteração clicando em Iniciar, Painel de controle, Opções regionais e de idiomas. Em seguida, selecione o idioma em uma lista suspensa na seção Padrões e formatos da caixa de diálogo.
