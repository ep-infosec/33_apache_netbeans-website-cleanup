NetBeans IDE 5.5 發行說明

歡迎使用 NetBeans IDE 5.5。NetBeans IDE 是一個基於標準的模組化集成開發環境 (Integrated Development Environment, IDE)，它是使用 Java 程式語言編寫的。NetBeans 項目包括：

  • 開放原始碼 IDE，是使用 Java 程式語言編寫的。
  • 應用程式平臺，可以作為一個構建任意應用程式類型的通用框架。
  • 附加套裝軟體，例如 Mobility 和 Profiler，可用於擴展 NetBeans IDE 的功能。

有關 NetBeans 附加套裝軟體的發行資訊，請參閱：

  • Mobility Pack 發行說明

支援的技術


NetBeans IDE 5.5 支援以下技術：

  • Enterprise JavaBeans (EJB) 3.0
  • JAX-WS 2.0
  • Java Persistence (JSR-220)
  • JavaServer Faces 1.2 (JSR-127)
  • Java Servlet 2.5
  • JavaServer Pages (JSP) 2.1
  • Struts 1.2.9
  • Java API for XML-based Remote Procedure Calls (JAX-RPC) 1.6
  • Java API for XML Registries (JAXR) 1.0
  • Java API for XML Processing (JAXP) 1.2
  • JavaServer Pages Standard Tag Library 1.1

注意：如果使用明確列出的版本之外的其他 JBoss、WebLogic 或 Struts 版本，則可能會導致無法預料的 IDE 行為，因為它們屬於外部介面並且不受本專案團隊的控制。

已使用下列應用伺服器對 NetBeans IDE 5.5 進行了測試：

  • Sun Java System Application Server PE 9.0 Update Release 1
  • Sun Java System Application Server PE 8.2
  • Tomcat 5.5.17
  • JBoss 4.0.4
  • WebLogic 9.1

請注意：如果使用 GlassFish 應用伺服器，建議將 GlassFish V1 UR1 的最終版本 (build 14) 與 NetBeans IDE 5.5 整合使用。

系統要求

支援的作業系統

NetBeans IDE 在支援 Java VM 的作業系統上執行。下面列出了已通過 NetBeans IDE 測試的平台。

  • Microsoft Windows XP Professional SP2
  • Solaris 作業系統（SPARC Platform Edition 和 x64 Platform Edition）版本 10
  • Mac OS X 10.4.5 PPC
  • Red Hat Fedora Core 3（簡體中文以及其他亞洲語言環境的作業系統除外）

NetBeans IDE 還可以在以下平臺上運行：

  • Java Desktop System 2
  • Microsoft Windows 2000 Professional SP4
  • Mac OS X 10.4.5 Intel
  • 其他各種 Linux 發行版本，例如 Red Hat Enterprise Linux

相容性

首次運行 IDE 時，可以導入在 NetBeans IDE 5.0 中使用的一部分設定。如果您選擇不導入先前發行版本中的設定，則系統將以預設設定打開 IDE。

注意：IDE 只能識別那些使用者資料夾位於預設位置的先前安裝。它不能辨別您使用 --userdir 來指定用戶資料夾的安裝。如果您要從安裝程式無法識別的 IDE 中導入設定，請下載 IDE 已壓縮的版本（而不是安裝程式）。

雖然可以將 IDE 先前安裝中的設定導入 NetBeans IDE 5.5，但是無法將 NetBeans IDE 5.5 中的設定導入 IDE 的早期版本。

注意：如果在 NetBeans IDE 5.5 中為 NetBeans IDE 4.1 或更早版本建立的專案編輯專案屬性，則早期的 NetBeans IDE 版本將無法再使用該專案。

已知問題


注意：預設情況下，NetBeans IDE 和 Sun Java System Application Server 包將建立具有 adminadmin 密碼的 admin 使用者。使用者應使用這些設定通過 Web 控制台進行登入。


以下是本發行版本中存在的一些待解決問題：

  • 問題 #26965：IDE 安裝程式錯誤。

    描述：如果 JDK 安裝中存在不相容的 sax.jar 版本，則 IDE 安裝程式將會出現錯誤，並顯示以下消息：“由於出現以下錯誤，導致嚮導無法繼續執行：無法裝入在 /wizard.inf 中指定的精靈 (104)”。

    解決方法：在 IDE 安裝過程中，暫時將 jre/lib/endorsed/sax.jar 從 JDK 資料夾中移除。

  • 問題 #64771：在啟用了 SELinux (安全增強) 選項的 Linux 上啟動失敗。

    描述：在啟用了 SELinux（安全增強）Linux 選項的情況下，啟動 NetBeans 失敗（僅限 Linux）。

    解決方法：如果出現此問題，請通過在控制列中鍵入 # setenforce 0 暫時禁用 SELinux。請注意，這需要具有超級用戶許可權。然後，將 selinux=0 添加到核心命令行以便在啟動時停用 SELinux。

  • 問題 #64840：在啟用了 IPv6 的 Linux 上，網路通信中斷。

    描述：在啟用 IPv6 的 Linux 上，網路通信中斷，進而阻止了 IDE 與相連的伺服器進行連接、執行自動更新等操作。

    解決辦法：如果出現此問題，請使用 -J-Djava.net.preferIPv4Stack=true 開關執行 IDE。或者，禁用 IPv6（在 /etc/modules.conf 或 /etc/modprobe.conf 中）。

    有關詳細資訊，請參閱：問題 #5087907。

  • 問題 #75967：使用大型 WSDL 檔時出現記憶體不足的錯誤。

    描述：通過 WSDL 建立 Web 服務用戶端或 Web 服務時，IDE 返回 OutOfMemoryError 並且有可能暫時無法使用。

    解決方法：如果出現此問題，請將 $NB_HOME/etc/netbeans.conf 檔中的 -J-Xmx128m 啟動參數編輯為 -J-Xmx256m。

  • 問題 #71959：對於不包含在專案中的原始檔案，無法執行程式碼輔助功能。

    描述：如果打開一個不包含在專案中的 Java 原始檔，並嘗試使用標準 Java 類的程式碼附註功能時，程式碼輔助框沒有任何回應。

    解決方法：對於要在 IDE 中編輯的原始檔案，請為其創建一個專案。

  • 在某些亞洲語言環境中，字元不能正常顯示。

    描述：IDE 在 zh_CN（簡體中文）及其他某些亞洲語言環境的 Red Hat Fedora Core 3 上執行時，文字由於呈現問題而不能正常顯示。這是由於 Fedora Core 發行版本上 JDK 的字體支援問題所致，它也可能會發生在 Fedora Core 上運作的其他 Java 應用程式中。

    解決方法：對於 Red Hat Fedora Core 3 上的呈現問題，沒有安全的解決方法，因此最好使用這些語言環境的其他 Linux 發行版本或作業系統。我們已驗證該問題不會出現在 Red Hat Enterprise Linux 3 中。

  • 問題 #75562：無法從 NetBeans IDE 5.0 中導入快捷鍵映射。

    描述：IDE 無法正確導入使用者建立的所有快捷鍵映射設定檔，或者導入對標準快捷鍵映射設定檔所做的更改。

    解決方法：無。

  • 問題 #77129：Alt 快捷方式組合鍵在 Macintosh 平臺上不起作用。

    描述：所有使用 Alt 的快捷方式組合鍵在 Macintosh 上都不起作用。

    解決方法：請使用功能表單啟動這些功能。

  • 問題 #85361：NetBeans 5.5 安裝程式無法在 Windows Vista 上啟動。

    描述：在執行 Windows Vista 的系統上，NetBeans 5.5 安裝程式無法啟動，並顯示一條建議採用 "silent" 或 "console" 標誌執行的消息。

    解決方法：要更正此問題，請在 JDK 6.0 build 96 或更高版本上執行安裝程式。

    如果仍出現其他問題，請手動禁用相容模式。具體方法是：右鍵點選 .exe 安裝程式檔，從彈出式功能表中選擇“屬性”，然後取消選擇“檔案屬性”對話方塊中“相容性”標籤上的“相容模式”。請注意，如果自動啟用了相容模式，則需要為當前使用者以及所有使用者禁用該模式。具體方法是：點選“顯示所有使用者的設定”按鈕，然後取消選擇“所有使用者的相容性”對話方塊中的“相容模式”核取方塊。

  • 問題 #86858：在使用 JDK 1.6 的 Mac OS 上出現重繪問題。

    描述：拖動分頁時，會出現一個黑色的方形區域。放下標籤後，出現重繪“歡迎”螢幕問題。

    解決方法：設置 -J-Dapple.awt.graphics.UseQuartz=true 或使用 Java 1.5 執行 IDE。

  • 問題 #87247：複製類別時未添加所需的 import 語句。

    描述：從一個套件複製類別並貼到另一個套件時，未自動添加所需的 import 語句。

    解決方法：如果出現此問題，請對類別使用“修復導入”操作。

  • 問題 #6986：重命名 bean 屬性時未正確更新程式碼。

    描述：使用讀取和寫入方法重命名 bean 屬性時，未正確更新程式碼。

    解決方法：確保 bean 存在完整的元模型，然後查找並更改 setter/getter 模式。請注意，如果已編輯了這些方法的程式碼，則此解決方法將無效。

  • 問題 #83712：含非ASCII字元的專案路徑的專案，無法使用JDK6建置。

    描述：在 JDK 1.6 上執行IDE時(不論您將專案設定編譯成任何Java版本)，且專案名稱含有任何其他非ASCII字元的其他國家文字時，當您在建置或執行專案時可能會遇到問題。這是因為Ant 1.6.5中使用新的XML剖析器所造成

    解決方法：遇到上述問題時，可透過以下方法來解決:

    1. 將專案資料夾名稱設定成不含造成錯誤的字元，可以透過專案的浮動右鍵選單修改專案名稱。
    2. 在 JDK 5 上執行您的IDE。
    3. 下載 Ant 1.7.0. 並切換 Ant Home 參數到 1.7.0 Ant 的安裝資料夾，在 工具 > 選項 > 雜項 > Ant。
    4. 添加 xercesImpl.jar 到 IDE 的類別路徑，到 工具 > 選項 > 雜項 > Ant 並點選管理類別路徑。點選添加 JAR/ZIP 後，從NetBeans安裝資料夾下的ide<>/modules/ext資料夾中選擇要添加的 xerces-<>.jar。
