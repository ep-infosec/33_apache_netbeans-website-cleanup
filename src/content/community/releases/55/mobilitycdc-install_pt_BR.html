Guia de instalação do Mobility Pack do NetBeans para CDC 5.5

O Mobility Pack do NetBeans para CDC 5.5 oferece um ambiente para a construção rápida de aplicativos Java Micro Edition (Java ME) compatíveis com dispositivos que usam Connected Device Configuration (CDC) e os perfis Personal, Foundation e AGUI.

Observação: Para obter informações sobre os requisitos do sistema do software incluído no pacote, consulte as Notas de versão do Mobility Pack do NetBeans para CDC 5.5.1.

Pré-requisitos

Este tópico especifica os pré-requisitos que devem ser atendidos antes de você instalar o Mobility Pack do NetBeans para CDC 5.5.

Software necessário

Os requisitos de hardware, software e sistema operacional estão detalhados nas Notas de versão do Mobility Pack do NetBeans para CDC 5.5.

Você precisará ter os softwares a seguir instalados antes de instalar o Mobility Pack do NetBeans para CDC 5.5.

  • Java SE Development Kit (JDK) 5.0
  • NetBeans IDE 5.5

O Sun Java Toolkit para CDC é integrado no Mobility Pack do NetBeans. Entretanto, se você programar aplicativos para outras plataformas compatíveis, será necessário fazer o download e instalar um ou mais dos SDKs a seguir:

  • Ricoh
    • Emulador de arquitetura de software incorporado (registro necessário para download)
  • Sony Ericsson
    • UIQ 3 SDK (logon necessário para download)
    • Pacote de extensão M600 para UIQ 3 SDK Beta 3 (download)
    • Pacote de extensão P990 para UIQ 3 SDK Beta 3 (download)
  • Nokia
    • Plataforma Nokia Series 80 SDK para sistema operacional Symbian, para Java, perfil Personal (download)
  • SavaJe
    • O sistema operacional SavaJe (registro necessário para download)

Instalação do software

A seguir há instruções para a instalação do Mobility Pack do NetBeans para CDC 5.5. Estas instruções presumem que você já instalou o Java SE Development Kit e o NetBeans IDE 5.5.

Para instalar o Mobility Pack do NetBeans para CDC 5.5:

  1. Faça o download do arquivo do instalador no endereço http://services.netbeans.org/downloads/index.php?rs=11&p=10
  2. Navegue em seu sistema até o local do arquivo do instalador e clique duas vezes no ícone do instalador para iniciá-lo.
  3. Na página de boas-vindas do assistente para instalação, clique em Próximo.
  4. Aceite o contrato de licença e clique em Próximo.
  5. Na página do diretório de instalação do assistente para instalação, especifique o local de instalação do NetBeans IDE 5.5.
  6. O assistente para instalação procura as instalações compatíveis do JDK. Se o JDK não estiver no local padrão, você precisará especificar o local do Java JDK 1.5.0 ou superior. Você também pode digitar o caminho de outro JDK instalado ou usar o botão Explorar para ir até outra instalação e selecioná-la.
  7. Quando a instalação estiver concluída, clique em Finalizar para sair do assistente.

O diretório de projeto padrão é C:\Documentos e configurações\nome de usuário. Se você desinstalar o Mobility Pack do NetBeans para CDC, esse diretório não será excluído, e será identificado da próxima vez que você instalar o Mobility Pack do NetBeans para CDC.

Observação: Se você encontrar problemas durante a instalação do software, consulte Solução de problemas para obter descrições completas e soluções sugeridas para problemas não resolvidos que poderiam afetar a instalação.

 

Instalando o Mobility Pack do NetBeans para CDC 5.5 em plataformas que não são do Windows

Atualmente, o Mobility Pack para CDC não está disponível em plataformas que não são do Windows porque não há suporte a emuladores de terceiros para outras plataformas. Entretanto, oferecemos um zip binário, sem suporte, do pacote para aqueles que quiserem instalá-lo em outros sistemas operacionais. Observe que não há documentação nem suporte ao zip binário. Para obter informações sobre como fazer o download e instalar o zip binário, visite a página de perguntas freqüentes deste tópico. A lista de usuários do Mobility também está disponível caso haja mais perguntas sobre este tópico.

Instalando as plataformas do emulador

Para desenvolver aplicativos CDC, você precisa ter a plataforma CDC registrada no IDE. As instruções a seguir descrevem o processo para a instalação de cada plataforma de emulador aceitas atualmente pelo IDE.

Sun Java Toolkit para CDC

O Sun Java Toolkit para CDC vem integrado ao Mobility Pack do NetBeans, sendo assim, não é necessário fazer o download e instalar esse emulador.

Entretanto, se você quiser ter uma instalação separada do Toolkit, siga estas instruções:

  1. Faça o download do Sun Java Toolkit para CDC 1.0 da rede de desenvolvedores da Sun (download).
  2. Instale o emulador da plataforma de acordo com as instruções fornecidas.
  3. No Menu principal, escolha Ferramentas > Gerenciador de plataformas Java.
  4. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  5. Selecione Emulador de plataforma Java Micro Edition CDC e clique em Próximo.
  6. Selecione a pasta que contém o SDK da plataforma. O diretório padrão é C:\CDCTK10\. Clique em Próximo.
  7. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover as bibliotecas de recursos adicionais.
  8. Clique em Finalizar. O IDE registra a plataforma CDC.

Ricoh Embedded Software Architecture

O Mobility Pack para CDC é compatível com a Ricoh Embedded Software Architecture. Para integrar o suporte a essa arquitetura, siga estas instruções:

  1. Faça o download dos arquivos a seguir no site do programa do desenolvedor do Ricoh:
    • Emulador da Embedded Software Architecture (logon necessário para download)
  2. Descompacte o arquivo Embedded_Software_Architecture_Emulator_1.14c.zip.
    Os arquivos são extraídos para uma pasta <raiz>\Embedded_Software_Architecture_Emulator_1.14c\.
  3. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  4. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  5. Selecione Emulador de plataforma Java Micro Edition CDC e clique em Próximo.
  6. Selecione a pasta que contém o SDK da plataforma. O diretório padrão é C:\Embedded_Software_Architecture_Emulator_1.14c\. Clique em Próximo.
  7. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover as bibliotecas de recursos adicionais.
  8. Clique em Finalizar. O IDE registra a plataforma CDC.

Plataforma CDC Sony Ericsson para o UIQ SDK 3

O Mobility Pack para CDC é compatível com a plataforma CDC 1 Sony Ericsson. Para integrar o suporte aos dispositivos das séries M600 e P990, siga estas instruções:

  1. Faça o download dos arquivos a seguir no site da Web developer.sonyericsson.com:
    • UIQ 3 SDK (logon necessário para download)
    • Pacote de extensão M600 para UIQ 3 SDK (download)
    • Pacote de extensão P990 para UIQ 3 SDK (download)
  2. Descompacte o arquivo UIQ3SDK_final.zip.
  3. Navegue até o local em seu sistema onde os arquivos foram extraídos e clique duas vezes em UIQS3SDK.exe.
    O Assistente para instalação do UIQ 3 SDK é aberto.
  4. Siga as instruções do assistente para instalar o SDK.
    O diretório padrão da instalação é C:\Symbian\UIQ3SDK\
  5. Para instalar os pacotes de extensão M600 e P990, navegue até os locais de download em seu sistema e clique duas vezes nos ícones do instalador para iniciar os assistentes para instalação.
  6. Siga as instruções do assistente para instalar cada SDK.
  7. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  8. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  9. Selecione Emulador de plataforma Java Micro Edition CDC e clique em Próximo.
  10. Selecione a pasta que contém o SDK da plataforma. O diretório padrão é C:\Symbian\UIQ3SDK\. Clique em Próximo.
  11. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover as bibliotecas de recursos adicionais.

    Observação: Se você vir a mensagem, "A plataforma não está configurada corretamente. Use as ferramentas de configuração do SDK para instalar as ferramentas necessárias", é porque não foi selecionado um estilo de emulador.

    Para corrigir o problema:
    1. Clique na guia Configurações e clique no botão Ferramenta de configuração do SDK.
    2. Na Configuração do SDK, clique na guia Estilo e escolha um estilo em Estilos do emulador. Escolha Aplicar estilos.
    3. Feche o Configurador do SDK.
  12. Clique em Finalizar. O IDE registra as plataformas CDC.

Plataforma Nokia Series 80 SDK para sistema operacional Symbian, para Java, perfil Personal

O Mobility Pack para CDC é compatível com a plataforma Series 80 SDK para sistema operacional Symbian, para Java, perfil Personal. Para integrar o suporte aos dispositivos Nokia Series 80, siga estas instruções:

  1. Faça download da plataforma Nokia Series 80 SDK para sistema operacional Symbian, para Java, perfil Personal da rede de desenvolvedores da Sun (download). O nome do arquivo é S80_DP_2_0_PP_SDK.zip.
  2. Descompacte o arquivo.
  3. Navegue para o local em seu sistema em que você descompactou o arquivo. Clique duas vezes no ícone do arquivo setup.exe.
    O assistente do instalador da Nokia é aberto.
  4. Siga as instruções do assistente para instalar o SDK.
    O assistente solicitará que você escolha o dispositivo que deseja definir como padrão. Escolha o Series80_DP2_0_PP_SDK.
  5. No Menu principal do NetBeans IDE, escolha Ferramentas > Gerenciador de plataformas Java.
  6. Clique em Adicionar plataforma. A caixa de diálogo Adicionar plataforma Java é aberta.
  7. Selecione Emulador de plataforma Java Micro Edition CDC e clique em Próximo.
  8. Selecione a pasta que contém o SDK da plataforma. O diretório padrão é C:\Symbian\S80_DP2_0_PP_SDK. Clique em Próximo.
  9. Se desejar, use a caixa de diálogo Nome da plataforma para adicionar ou remover as bibliotecas de recursos adicionais.
  10. Clique em Finalizar. O IDE registra a plataforma CDC.

Plataforma móvel SavaJe

O Mobility Pack para CDC aceita a criação de Xlets usando o sistema operacional SavaJe. O suporte ao sistema operacional SavaJe é incorporado no Sun Java Wireless Toolkit para CDC. Como o Jasper S20 está disponível atualmente apenas como uma edição de visualização, não há skin do emulador para o Jasper S20. Em vez disso, você pode usar o dispositivo DefaultColorPhone para simular o comportamento do Jasper S20.

Se você possui o Jasper S20, estas instruções mostram como implantar o aplicativo em seu dispositivo. Eis aqui o que você precisa:

  • O telefone Jasper S20.
  • O sistema operacional SavaJe (registro necessário)
  • Software de dispositivo móvel GSPDA (drivers USB MCCI).

Configurando o aparelho

As etapas a seguir descrevem o processo de instalação do software necessário para o tutorial do telefone Jasper S20.

Instalando o sistema operacional SavaJE

OBSERVAÇÃO: Este procedimento é para os aparelhos Jasper S20 que possuem uma versão anterior do sistema operacional SavaJe instalado e você deseja manter a calibragem e as configurações de fábrica existentes.

Existem três compilações do sistema operacional SavaJe que devem estar disponíveis em www.SavaJe.com:

  • JasperS20.zip
    A compilação comercial que vem no telefone.
  • JasperS20_EA_Developer.zip
    A compilação do desenvolvedor JavaOne Early Access sem o Java Debug Wire Protocol (JDWP) ativado.
  • JasperS20_EA_Developer_debug.zip
    A compilação do desenvolvedor JavaOne Early Access com o JDWP ativado. Para executar um pacote com a depuração JDWP, será necessário carregar essa compilação.

Siga estas etapas para instalar uma compilação atualizada do sistema operacional SavaJE no telefone.

  1. Extraia (descompacte) o arquivo zip que contém a compilação desejada para a raiz do cartão mini-SD.
  2. Conecte um cabo USB ou de força ao aparelho.
  3. Ligue o aparelho enquanto mantém pressionada a tecla da câmera.
  4. Na tela do menu do atualizador, selecione Redefinição de fábrica, movendo o controle para baixo até que a opção "Redefinição de fábrica" esteja realçada. Inicie a redefinição de fábrica movendo o controle para a direita. Use o controle para navegar até Sim e selecione Sim movendo o controle para a direita.
    O telefone deve começar a carregar a nova compilação. Depois que a carga estiver completa, talvez você seja solicitado a pressionar qualquer tecla para reinicializar o telefone.
  5. O aparelho é reinicializado e exibe a tela Xplore. Se a tela de carregamento da bateria for exibida durante a reinicialização, pressione o botão de força para continuar a reinicializar o telefone.

Na primeira vez em que o aparelho é inicializado depois de uma redefinição de fábrica, é exibido um menu de seleção Idioma/Localidade. Faça as suas seleções, navegando com o controle e pressionando o botão do controle. Você também será solicitado a definir a hora.

Registrando a plataforma CDC

Siga as instruções fornecidas para o Sun Java Wireless Toolkit para CDC 1.0.

Desinstalação do software

Esta seção contém informações sobre a desinstalação do Mobility Pack do NetBeans para CDC 5.5 do seu sistema.

  1. Feche o IDE.
  2. No Painel de controle, inicie Adicionar ou remover programas, selecione Mobility Pack do NetBeans para CDC 5.5 e clique em Alterar/Remover.
  3. Siga as instruções do desinstalador.

Observação: Você também pode desinstalar qualquer uma ou todas as plataformas do emulador através do Painel de controle do Windows.

Solução de problemas

Se tiver problemas ao instalar ou executar o software, consulte a lista de problemas abaixo para ver as soluções. Para obter uma lista completa dos problemas conhecidos, dos requisitos do sistema e de informações de configuração, consulte as Notas da versão.

Se encontrar um problema que não esteja listado nas notas da versão ou quiser solicitar uma melhoria, registre o problema no sistema de controle de erros do NetBeans, IssueZilla, em https://netbeans.org/community/issues.html. Para problemas do Mobility Pack do NetBeans, selecione o componente "mobility", selecione a versão "5.5" e, em seguida, selecione o subcomponente "cdc pack". Se não tiver certeza do nome correto do subcomponente, escolha "other".

Se já estiver conectado ao netbeans.org, vá diretamente para a tela de entrada de problemas do Mobility Pack: https://netbeans.org/bugzilla/enter_bug.cgi?component=mobility&version=5.5.
