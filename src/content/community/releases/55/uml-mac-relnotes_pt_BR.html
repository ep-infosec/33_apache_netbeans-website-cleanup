NetBeans 5.5, Módulo de modelagem UML - Notas de versão da plataforma Mac OS

Última atualização: 23 de março de 2007

A plataforma Macintosh OS atualmente não é compatível com a versão do NetBeans™ 5.5, Módulo de modelagem UML. Este documento de notas de versão fornece informações importantes sobre os problemas conhecidos e as soluções no uso do módulo de modelagem UML em uma plataforma Macintosh OS X. Consulte as Notas de versão do NetBeans 5.5, Módulo de modelagem UML para obter mais informações sobre os problemas conhecidos e as soluções das plataformas aceitas.

Requisitos do sistema

O NetBeans™ 5.5, Módulo de modelagem UML é compatível com todos os sistemas que atendam aos requisitos especificados na seção Requisitos do sistema das Notas de versão do NetBeans IDE 5.5, com a exceção do Macintosh OS. O módulo de modelagem UML atualmente não é compatível com a plataforma Macintosh OS X.


Problemas conhecidos e soluções

Os problemas não resolvidos descritos abaixo ocorrem durante o uso do módulo de modelagem UML no IDE executado na plataforma Macintosh OS X.

  • No Mac OS com JDK 1.5, o desempenho da modelagem UML se torna lento quando as barras de rolagem aparecem na janela do Editor de diagramas.

    Descrição:

    Erro #78346: O desempenho da modelagem UML é mais lento em uma plataforma Mac OS que executa o JDK 1.5 quando as barras de rolagem são exibidas na janela do Editor de diagramas, como um resultado da movimentação manual dos elementos do diagrama. Depois que as barras de rolagem forem exibidas, qualquer ação realizada nos diagramas leva um longo tempo para ser concluída.

    Solução:

    Clique em um dos botões de layout na barra de ferramentas do Editor de diagramas para refazer o layout do diagrama. Se estiver trabalhando com um diagrama grande, adicione a opção da linha de comando
    -J-Dapple.awt.graphics.UseQuartz=false     à declaração netbeans_default_options no diretório_de_instalação_do_nb5.5/Contents/Resources/NetBeans/etc/arquivo/arquivo netbeans.conf. Reinicie o IDE.

    Observação: O uso da opção de linha de comando mencionada acima pode afetar negativamente o comportamento do NetBeans 5.5 IDE e de outros módulos. Esses comportamentos são descritos nos problemas seguintes:
    • Erro #73623: O explorador de arquivos atualiza aleatoriamente e embaralha o texto que fica em cima de outro texto na árvore de arquivos. Um comando de atualização forçada restaura a aparência da árvore de arquivos, mas o embaralhamento ocorre com freqüência.
    • Erro #83271: Depois de mover uma paleta de um local para outro, algumas áreas da tela parece que não são atualizadas corretamente e as informações ficam sobrepostas.
    • Erro #88271: Desativar a renderização de Quartz causa outros problemas, tais como problemas de renderização na janela da ajuda.
    Se tiver qualquer um dos problemas citados acima ou outros problemas ao usar a opção -J-Dapple.awt.graphics.UseQuartz, remova a opção do arquivo netbeans.conf e reinicie o IDE.
  • Iniciar o IDE com o JDK 1.6 em uma plataforma Mac OS X afeta o desempenho da modelagem UML.

    Descrição:

    Erro #82943: O problema de desempenho do UML descrito na seção anterior é corrigido quando o IDE é iniciado com o JDK 1.6 em um Mac OS X em uma plataforma Intel. Entretanto, o desempenho do módulo de modelagem UML é altamente prejudicado quando o IDE é iniciado com o JDK 1.6 em um Mac OS X em uma plataforma Power PC.

    Existem também outros problemas que ocorrem no IDE quando ele é iniciado com o JDK 1.6 (consulte o Problema #86850). O IDE possui problemas gerais que ocorrem com o uso do JDK 1.6.

    Solução: Use o JDK 1.5 ao iniciar o IDE.

  • Criar um diagrama de classe a partir de elementos selecionados retorna uma exceção em um Mac OS usando JDK 1.6.
    Descrição:
    Erro #82831: Em uma plataforma Mac OS, criar um diagrama de classe a partir de elementos selecionados retorna uma exceção StackOverflowError. Trata-se de um erro conhecido do Macintosh (ID #4647508)

    Solução: Use o JDK 1.5 ao iniciar o IDE.
