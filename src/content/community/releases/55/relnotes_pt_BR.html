Notas da versão do NetBeans IDE 5.5

Bem-vindo ao NetBeans IDE 5.5. O NetBeans IDE é um ambiente de desenvolvimento integrado, baseado em padrões e em módulos e escrito na linguagem de programação Java. O projeto NetBeans consiste em:

  • Um IDE de código aberto escrito na linguagem de programação Java.
  • Uma plataforma de aplicativo, que pode ser utilizada como uma estrutura genérica para construir qualquer tipo de aplicativo.
  • Pacotes suplementares, como Mobility e Profiler, que permitem ampliar a funcionalidade do NetBeans IDE.

Para obter informações sobre a versão de pacotes suplementares do NetBeans, consulte:

  • Notas da versão do Mobility Pack
  • Notas da versão do Profiler
  • Notas da versão do Visual Web Pack
  • Notas da versão do Enterprise Pack

Tecnologias compatíveis


O NetBeans IDE 5.5 é compatível com as seguintes tecnologias:

  • Enterprise JavaBeans (EJB) 3.0
  • JAX-WS 2.0
  • Java Persistence (JSR-220)
  • JavaServer Faces 1.2 (JSR-127)
  • Java Servlet 2.5
  • JavaServer Pages (JSP) 2.1
  • Struts 1.2.9
  • API de Java para Chamadas de Procedimento Remoto (RPCs) baseadas em XML (JAX-RPC) 1.6.
  • APIs de Java para registros de XML (JAXR) 1.0
  • API de Java para processamento de XML (JAXR) 1.2
  • JavaServer Pages Standard Tag Library 1.1

Observação: o uso de versões do JBoss, WebLogic ou Struts que não as especificadas pode alterar o funcionamento do IDE de forma imprevisível, porque as interfaces externas não são controladas pela equipe do projeto.

O NetBeans IDE 5.5 foi testado com os seguintes servidores de aplicativo:

  • Servidor de aplicativos Sun Java System PE 9.0, versão de atualização 1
  • Servidor de aplicativos Sun Java System PE 8.2
  • Tomcat 5.5.17
  • JBoss 4.0.4
  • WebLogic 9.1

Se estiver usando o servidor de aplicativos GlassFish, recomenda-se executar a construção final do GlassFish V1 UR1 (build 14) para trabalhar com o NetBeans IDE 5.5.

Software necessário

O NetBeans IDE é executado no J2SE JDK 5.0 (Java SE Development Kit), composto pelo Java Runtime Environment e por ferramentas de desenvolvimento para compilar, depurar e executar aplicativos escritos na linguagem Java.

É possível fazer o download do JDK para sua plataforma em um destes sites:

  • Solaris: http://java.sun.com/j2se/1.5.0/
  • Windows: http://java.sun.com/j2se/1.5.0/
  • Linux: http://java.sun.com/j2se/1.5.0/
  • Mac OS X: http://www.apple.com/support/downloads/javaupdate142.html (Mac OS X 10.3.4 ou superior). As atualizações do Mac OS também estão disponíveis pelo mecanismo de atualização de software da Apple.
  • Open VMS: http://h18012.www1.hp.com/java/download/index.html

    • Observação: É possível fazer o download de versões arquivadas do IDE em http://java.sun.com/products/archive/index.html.

Compatibilidade

Ao executar o IDE pela primeira vez, é possível importar um subconjunto de configurações usadas no NetBeans IDE 5.0. Se escolher não importar as configurações usadas na versão anterior, o IDE é aberto com as configurações padrão.

Observação: O IDE só reconhece instalações anteriores se o diretório de usuário estiver no local padrão. O IDE não reconhece instalações em que o diretório de usuário foi especificado em --userdir. Se quiser importar configurações de um IDE não reconhecido pelo instalador, faça o download de uma versão arquivada do IDE em vez de substituir o instalador.

Embora seja possível importar configurações de uma instalação anterior do IDE para o NetBeans IDE 5.5, não é possível importar as configurações do NetBeans IDE 5.5 para para uma versão anterior.

Observação:Se você editar no NetBeans IDE 5.5 as propriedades de projeto de um projeto criado no NetBeans IDE 4.1 ou anterior, não será mais possível trabalhar com esse projeto em versões anteriores do NetBeans IDE.

Problemas conhecidos


Observação: O pacote NetBeans IDE e servidor de aplicativos Sun Java System cria o usuário admin e a senha adminadmin por padrão. Os usuários devem usá-los para fazer logon pelo console da Web.


Os problemas não solucionados dessa versão são os seguintes:

  • Erro #26965: O instalador do IDE trava.

    Descrição: Caso tenha uma versão incompatível de sax.jar na sua instalação do JDK, o instalador do IDE trava e é exibida a seguinte mensagem: "O assistente não pode prosseguir por causa do seguinte erro: não é possível carregar o assistente especificado em /wizard.inf (104)"

    Solução: Remover temporariamente jre/lib/endorsed/sax.jar do diretório do JDK durante a instalação do IDE.

  • Erro #64771: Falha na inicialização com a opção de segurança elevada SELinux do Linux habilitada.

    Descrição: Ao iniciar o NetBeans com a opção de segurança elevada SELinux habilitada, ocorre uma falha na inicialização (somente no Linux).

    Solução: Se esse problema acontecer, desabilite o SELinux temporariamente digitando #setenforce 0 na linha de comando. Observe que essa operação requer privilégios de raiz. Em seguida, insira selinux=0 à linha de comando do agente para desabilitar o SELinux na inicialização.

  • Erro #64840: Comunicação em rede corrompida no Linux com o IPv6 habilitado.

    Descrição: Comunicação em rede corrompida no Linux com o IPv6 habilitado, impedindo que o IDE se conecte a servidores agrupados, atualizações automáticas, etc.

    Solução: Se isso acontecer, execute o IDE usando -J-Djava.net.preferIPv4Stack=true'. Outra possibilidade é desabilitar o IPv6 (em /etc/modules.conf ou /etc/modprobe.conf).

    Para mais informações, consulte: Erro #5087907.

  • Erro #75967: Erro de memória insuficiente ao trabalhar com arquivos WSDL grandes.

    Descrição: Ao criar um cliente de serviço da Web a partir do WSDL, o IDE retorna a mensagem OutOfMemoryError e temporariamente pode não estar disponível.

    Solução: Se tiver esse problema, altere o parâmetro de inicialização -J-Xmx128m, em $NB_HOME/etc/netbeans.conf, para -J-Xmx256m.

  • Erro #71959:Falha no auto-completar de código porque os códigos-fonte estão fora do projeto.

    Descrição: Ao abrir um arquivo fonte Java que não está em um projeto e tentar executar o auto-completar de código para classes Java padrão, a caixa de auto-completar de código não oferece nenhuma possibilidade.

    Solução: Crie um projeto para os códigos-fonte que você pretende editar no IDE.

  • Erro #75562:Importação de mapa de teclas do NetBeans IDE 5.0 não funciona.

    Descrição: O IDE não importa corretamente todos os perfis de mapa de teclas criados pelo usuário ou as alterações para os perfis padrão de mapa de teclas.

    Solução: Nenhuma.

  • Erro #77129: As combinações da tecla Alt e o teclado não funcionam em plataformas Macintosh.

    Descrição: Nenhuma das combinações do teclado que usam a tecla Alt funciona no Macintosh.

    Solução: Usar itens de menu que executem os comandos.

  • Erro #85361: O instalador do NetBeans 5.5 não é inicializado no Windows Vista.

    Descrição: Em sistemas com o Windows Vista, a inicialização do instalador do NetBeans 5.5 falha e aparece uma mensagem sugerindo a execução com um sinalizador "silencioso" ou de "console".

    Solução: Para corrigir esse problema, execute o instalador no JDK 6.0, construção 96 ou mais recente.

    Se tiver outras dificuldades, desabilite o Modo de compatibilidade manualmente. Para fazer isso, clique com o botão direito do mouse no arquivo instalador .exe e escolha Propriedades no menu pop-up. Em seguida, desmarque o Modo de compatibilidade na guia Compatibilidade na caixa de diálogo Propriedades do arquivo. O Modo de compatibilidade é ativado automaticamente, portanto é preciso desativá-lo para o usuário atual e todos os usuários. Para tanto, clique no botão "Exibir configurações para todos os usuários", depois desmarque a caixa de seleção do Modo de compatibilidade na caixa de diálogo "Compatibilidade para todos os usuários".

  • Erro #86858: Problemas em redesenhar no MacOS com JDK 1.6.

    Descrição: Ao arrastar a guia, aparece um quadrado preto. Depois de soltar a guia, ocorrem problemas de redesenho na tela de boas-vindas.

    Solução: Defina -J-Dapple.awt.graphics.UseQuartz=true ou use Java 1.5 para executar o IDE.

  • Erro #87247: Copiar classes não insere as instruções de importação necessárias.

    Descrição: Ao copiar e colar uma classe de um pacote para outro, as instruções de importação não são adicionadas.

    Solução: Se ocorrer esse problema, execute a ação Corrigir importações na classe.

  • Erro #6986: Código não foi atualizado corretamente quando a propriedade de Bean foi renomeada.

    Descrição: Ao renomear a propriedade de Bean com métodos de leitura e escritura, o código-fonte não foi atualizado corretamente.

    Solução: Certifique-se de que haja um metamodelo completo para o Bean, depois localize o padrão setter/getter e altere-o. Observe que essa solução alternativa não funcionará se os códigos do método já tiverem sido alterados.
