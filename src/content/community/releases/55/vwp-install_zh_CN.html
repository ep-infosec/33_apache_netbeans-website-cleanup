NetBeans Visual Web Pack 5.5 安装说明

NetBeans Visual Web Pack 5.5 为快速构建 Web 应用程序（具有 AJAX 的 JavaServer Faces 组件以及拖放式数据源绑定支持等）提供了一个环境平台。

重要说明

应用服务器/Servlet 容器

Visual Web Pack 可使用 NetBeans IDE 支持的所有应用服务器和 Servlet 容器。

Java EE 5/JavaServer Faces 1.2 限制

截止到目前为止，仅 Sun Java System Application Server 9 U1 (build 14) 或更高版本支持使用 Java EE 5 平台的 Visual Web 应用程序。如果您要使用其他服务器，则必须在创建项目时选择 J2EE 1.4 平台。

捆绑的 Travel 数据库

许多教程和样例应用程序都需要使用 Travel 数据库。如果在安装 Visual Web Pack 之前为 IDE 提供了可用的 Sun Java System Application Server，则 Visual Web Pack 安装会将 Travel 数据库添加到与 Sun Java System Application Server 捆绑在一起的 Java DB (Derby) 服务器中。如果在安装 Visual Web Pack 之后为 IDE 提供了可用的 Sun Java System Application Server，则必须重新启动 IDE 才能启用 Travel 数据库。有关将 Travel 数据库安装到您自己安装的 Java DB 或 Apache Derby 数据库服务器的说明，请参见“配置信息”部分的 Visual Web Pack 样例数据库。

技术预览

如果您已安装了 NetBeans Visual Web Pack 5.5 的技术预览版本，则在安装 Visual Web Pack 的产品发行版本之前，您必须按照卸载软件中所述卸载它。

如果已将 Blueprints Ajax 组件安装到 NetBeans Visual Web Pack 5.5 技术预览版本，则在安装 Visual Web Pack 的产品发行版本之前，必须手动卸载这些组件。从 IDE 中删除组件库：

  1. 选择“工具”>“组件库管理器”。

  2. 在组件库管理器中，选择要删除的库，然后单击该窗口底部的“删除”按钮。

    您可能会看到一条消息，指出在目前打开的项目中存在使用了该库的项目。如果出现这种情况，请在消息中单击“确定”并关闭组件库管理器，然后关闭这些项目。接下来，您便可以打开组件库管理器，重新执行上述操作。

  3. 确认您要删除该库。

    该库将从用户目录中删除。对于任何使用该库的项目来说，仍可以继续使用它，因为这些项目的项目目录中都包含自己的库副本。

先决条件

本主题提出了在您开始安装 NetBeans Visual Web Pack 5.5 之前必须满足的一些先决条件。

请从 netbeans.org 下载页中下载 NetBeans Visual Web Pack 5.5。

所需的软件

在安装 NetBeans Visual Web Pack 5.5 之前，必须先安装以下软件：

  • NetBeans IDE 5.5。
  • Java SE Development Kit (JDK) 5.0 Update 6（版本 1.5.0_06）或更高版本。

    如果未安装 JDK 5.0 Update 6 或更高版本，则无法执行此安装。您可以从 Java SE 下载页中下载此 JDK。

    请注意，Mac OS X 安装需要 Java 2 Standard Edition (J2SE) 5.0 Release 4。您可以从 Apple 下载页中下载此 JDK。请搜索 "J2SE" 以查找此发行版本。
  • Netbeans IDE 支持的某个应用服务器或 Servlet 容器。

    注意：要使用 Java EE 5 平台开发 Visual Web 应用程序，您必须使用 Sun Java System Application Server 9。有关详细信息，请参见 Java EE 下载 Web 站点。

安装顺序

如果要在 NetBeans IDE 中安装 NetBeans Visual Web Pack，但是单独安装了 Sun Java System Application Server，则在安装 Visual Web Pack 之前最好先配置 Sun Java System Application Server，这样才能立即使用捆绑的 Travel 数据库。如果在安装 Visual Web Pack 之后配置了应用服务器，则必须重新启动 IDE，然后才能在“运行环境”窗口中使用 Travel 数据库。

防火墙应用程序

启用防火墙应用程序可能无法在安装过程中正确配置某些组件。根据您使用的防火墙，您可能需要在安装之前禁用它。请参阅防火墙文档。

Microsoft Windows

以下是在 Microsoft Windows 平台上安装 NetBeans Visual Web Pack 5.5 的说明。

开始之前的准备

以下限制适用于 Microsoft Windows 平台上的安装：

安装程序首先会将所需的文件解压缩到 TEMP 环境变量定义的缺省临时空间目录中。如果临时目录中的硬盘空间小于所需的空间，则安装会失败或无法正常执行。

如果出现这种问题，则应当增加分配给 TEMP 环境变量的空间，或者指定其他具有足够空间的临时目录。要为安装指定临时目录，请键入以下命令：

> cd location-of-installer
>  netbeans-visualweb-5_5-windows.exe -is:tempdir temporary-directory

安装

  1. 下载了安装文件 netbeans-visualweb-5_5-windows.exe 后，请双击安装程序图标以启动该安装程序。
  2. 在安装向导的“欢迎”页中，单击“下一步”。
  3. 接受许可证协议，然后单击“下一步”。
  4. 在安装向导的安装目录页中，指定 NetBeans IDE 5.5 安装的位置。
  5. 安装向导将搜索兼容的 JDK 安装。如果 JDK 不在缺省位置，则需要指定 Java JDK 1.5.0_06 或更高版本的位置。您也可以键入其他 JDK 的安装路径，或者使用“浏览”按钮导航并选择其他安装。
  6. 安装完成后，单击“完成”退出向导。

缺省项目目录为 C:\Documents and Settings\username。如果要卸载 NetBeans Visual Web Pack，则系统不会删除此目录，会在下次安装 Visual Web Pack 时识别该目录。

  3. 单击“卸载”，然后单击“关闭”。

配置 Sun Java System Application Server

如果安装的 NetBeans IDE 与 Sun Java System Application Server 捆绑在一起，则已在 IDE 中注册了该应用服务器，您不需要对其进行配置了。如果单独下载并安装了 Sun Java System Application Server，则必须为 IDE 配置可用的应用服务器。最好在安装 Visual Web Pack 之前配置该服务器。

配置单独安装的服务器：

  1. 安装 NetBeans IDE 和 Sun Java System Application Server 后，启动 IDE。
  2. 在“运行环境”窗口中，右键单击“服务器”节点，然后从弹出式菜单中选择“添加服务器”。
  3. 在“添加服务器实例”向导中，从“服务器”下拉列表中选择 "Sun Java System Application Server"，然后单击“下一步”。
  4. 输入应用服务器的安装位置，然后单击“下一步”。
  5. 缺省应用服务器域 domain1 的缺省用户名和口令如下：
    • 用户名：admin
    • 口令：adminadmin
    请注意，如果您在此处指定了一个口令，则系统会将其存储在用户目录中，这将具有一定的安全风险。如果未在此处指定口令，则会在需要时提示您输入口令值。
  6. 单击“完成”。

Visual Web Pack 样例数据库

在安装 Visual Web Pack 的过程中，将为与 Sun Java System Application Server 捆绑在一起的 Java DB 数据库提供样例 Travel 数据库，如下表所示。某些样例应用程序和教程需要使用 Travel 数据库。如果在安装 Visual Web Pack 之前为 IDE 配置使用了 Sun Java System Application Server，则可以直接在“运行环境”窗口中使用 Travel 数据库。如果在安装 Visual Web Pack 之后为 IDE 提供了可用的 Sun Java System Application Server，则必须重新启动 IDE，然后才会在“运行环境”窗口中显示 Travel 数据库。

样例数据库 连接详细信息 适用对象
Travel DB URL：jdbc:derby://localhost:1527/travel
驱动程序：org.apache.derby.jdbc.ClientDriver
用户名：travel
口令：travel
 某些教程和样例应用程序需要使用该数据库

注意：要将 Travel 数据库用于 Sun Java System Application Server 以外的服务器，您必须为该服务器提供 derbyclient.jar（如以下部分所述）。

如果不希望安装 Sun Java System Application Server，请完成以下步骤为 IDE 提供 Travel 数据库。

  1. 下载并安装 Java DB 数据库或 Apache Derby 数据库。

  2. 完成以下步骤为 IDE 提供该数据库：

    1. 从“工具”菜单中选择“选项”。
    2. 单击“高级选项”。
    3. 展开“服务器和外部工具设置”，然后选择“Java DB 数据库”。
    4. 将“Java DB 位置”设置为安装该数据库服务器的文件夹。
    5. 将“数据库位置”设置为存储数据库的文件夹。
    6. 单击“关闭”。
  3. 如果尚未启动数据库服务器，请选择“工具”>“Java DB 数据库”>“启动 Java DB 服务器”。
  4. 在“运行环境”窗口中，右键单击“数据库”，然后从弹出式菜单中选择“新建连接”以打开“新建数据库连接”对话框。
  5. 从“名称”下拉列表中选择 "Java DB (Network)"。
  6. 在“数据库 URL”文本框中，键入 jdbc:derby://localhost:1527/travel;create=true;。
  7. 将“用户名”设置为 travel，将“口令”设置为 travel，然后单击“确定”。
  8. 在建立连接后，请单击“确定”以关闭对话框。
  9. 将为此连接显示一个新节点。右键单击该节点，然后从弹出式菜单中选择“断开连接”。
  10. 如果您不再需要该连接来创建数据库。请右键单击该节点，然后从弹出式菜单中选择“删除”。
  11. 右键单击“数据库”，然后从弹出式菜单中选择“新建连接”以打开“新建数据库连接”对话框。
  12. 从“名称”下拉列表中选择 "Java DB (Network)"。
  13. 在“数据库 URL”文本框中，键入 jdbc:derby://localhost:1527/travel。
  14. 将“口令”设置为 travel，然后单击“确定”。

  15. 在建立连接后，请单击“确定”以关闭对话框。

    从现在起，您将使用此节点来连接 Travel 数据库。
  16. 右键单击该连接节点，然后选择“执行命令”以打开“SQL 命令”查询编辑器。
  17. 在主菜单中，选择“文件”>“打开文件”。
  18. 浏览并选择 "$NetBeansInstallDir/rave2.0/startup/samples/create-Travel-derby.sql"，然后单击“打开”。
  19. 复制 create-Travel-derby.sql 文件的内容，并将其粘贴到“SQL 命令”查询编辑器中。
  20. 单击编辑工具栏中的“运行 SQL”以在 Travel 数据库中创建并填充表。
  21. 关闭“SQL 命令”查询编辑器和 create-Travel-derby.sql。
  22. 展开 Travel 数据库连接节点以查看表。

有关详细信息，请参见：

卸载软件

本部分包含了从以下系统卸载 NetBeans Visual Web Pack 5.5 的信息。

  • Microsoft Windows
  • Solaris OS 和 Linux
  • Mac OS X

    • Microsoft Windows

    1. 关闭 IDE。
    2. 从“控制面板”中启动“添加或删除程序”，选择 "NetBeans Visual Web Pack"，然后单击“更改/删除”。
    3. 按照卸载程序中的说明执行操作。
    4. 如果 $NB_installdir/rave2.0 目录仍然存在，请手动将其删除。

    Solaris OS 和 Linux

    1. 关闭 IDE。

    2. 导航至 NetBeans Visual Web Pack 5.5 安装的 uninstall 子目录：

      $NetBeansInstallDir/rave2.0/uninst
    3. 执行 uninstaller 脚本。
    4. 如果 $NB_installdir/rave2.0 目录仍然存在，请手动将其删除。

    Mac OS X

    1. 关闭 IDE。

    2. 导航至 NetBeans Visual Web Pack 5.5 安装的 Visual Web Pack _uninstall 子目录：

      $NetBeansInstallDir/NetBeans.app/Contents/Resources/NetBeans/rave2.0/uninstall

    3. 执行以下命令：

      java -jar uninstall.jar。
    4. 如果 $NB_installdir/rave2.0 目录仍然存在，请手动将其删除。

    疑难解答

    如果您在安装或运行软件时遇到问题，请参见下面的问题列表以获取解决方法。有关已知问题的完整列表、系统要求以及配置信息，请参见发行说明。

    如果您遇到发行说明中未列出的问题，或者要求提供改进的解决方法，请在 NetBeans 错误跟踪系统 IssueZilla (https://netbeans.org/issues) 中记录这些问题。对于 Visual Web Pack 问题，请选择 "visualweb" 组件，然后选择相应的子组件。如果您不知道相应的子组件名称，请选择“其他”。

    如果您已登录到 netbeans.org，则可以直接进入 visualweb 的输入问题屏幕：https://netbeans.org/bugzilla/enter_bug.cgi?component=visualweb&version=5.5。

    以下是本发行版本中一些可能影响或破坏安装过程的未解决问题：

    • Solaris 安装因“找不到 JRE”错误而终止，即使您已安装并指定了 JRE。

      描述：此应用程序要求运行 Java Runtime Environment (JRE) 实现，但安装程序在计算机上找不到该实现。请使用命令行开关 -is:javahome 指定有效的 JRE。要获得更多帮助，请使用选项 -is:help。

      解决方法：使用 -is:tempdir custom-temp-dir 命令行参数来启动安装程序。请注意，custom-temp-dir 必须是绝对路径，并且至少应包含 1 GB 的可用硬盘空间。
