Instruções de instalação do IDE NetBeans 6.0

Este documento descreve como instalar o IDE NetBeans 6.0 em seu sistema. Software necessário

Antes de instalar o IDE, você deverá instalar o kit Java SE Development (JDK) 5 Atualização 14 (versão 1.5.0_14) ou mais recente (inclusive o JDK 6 Atualização 3) em seu sistema. Se não tiver o JDK 5 Atualização 14 ou uma versão mais recente instalada, não será possível prosseguir com a instalação. Para obter mais informações sobre a versão necessária do JDK, consulte a seção Software necessário em Notas de versão do IDE NetBeans 6.0.

Opções de download do instalador

Na página Download do IDE NetBeans 6.0, você pode fazer o download de um dos diversos instaladores, cada um contendo o IDE básico e as ferramentas adicionais.

Os seguintes instaladores estão disponíveis:

  • Web e Java EE. Fornece ferramentas para o desenvolvimento de Java SE, Java EE e aplicações Web. Esta opção de download também inclui o servidor de aplicação GlassFish V2 e o software Apache Tomcat.
  • Mobility. Oferece suporte ao desenvolvimento de aplicativos Java SE e inclui o pacote Mobility para criação de aplicativos MIDP em dispositivos móveis. Observe que alguns recursos Mobility, como o Assistente de Conexão sem fio, exigem recursos que estão disponíveis no pacote Web e Java EE. Se você planeja usar esses recursos, baixe a opção de instalação "Completa".
  • Java SE. Fornece todos os recursos padrão para o desenvolvimento de Java SE. O suporte ao NetBeans Plugin Development também está incluído.
  • Ruby. Fornece ferramentas para desenvolvimento Ruby e oferece suporte para Rails e JRuby.
  • C/C++. Oferece suporte para o desenvolvimento na linguagem C/C++.
  • Todos. Esta é uma opção de download completo do IDE NetBeans 6.0. Ela contém todos os tempos de execução e pacotes disponíveis para o IDE. Ela contém ferramentas para o desenvolvimento de arquitetura orientada a serviços e construção de diagramas de linguagem de modelagem unificada.

Personalizando sua instalação

Se você baixou a opção Web e Java EE ou a opção "completa", pode escolher exatamente as ferramentas e os tempos de execução para a instalação. Para tanto, você precisa clicar em Personalizar na página de boas-vindas e selecionar os recursos que deseja usar. Para obter mais informações, consulteInstalando o software.

Se você deseja adicionar funcionalidade a sua configuração do IDE NetBeans, use o NetBeans Plugin Manager. Por exemplo, suponhamos que você iniciou baixando e instalando o pacote Ruby. Em seguida, você decidiu que deseja experimentar a funcionalidade C/C++. Para fazer isso, acesse o NetBeans Plugin Manager no IDE (selecione Ferramentas > Plug-ins) e adicione o pacote C/C++ à instalação existente.

Há diversas notas importantes sobre a instalação e a configuração de alguns recursos do IDE NetBeans:

  1. O Apache Tomcat 6.0.14, que está incluído nas opções de download "Web e Java EE" e "Completa", não é instalado por padrão. Se você deseja instalar o Apache Tomcat a partir dos downloads Web e Java EE ou "Completa", inicie o instalador e selecione Apache Tomcat 6.0.14 na caixa de diálogo Personalizar a instalação.
  2. Para o desenvolvimento de C/C++ (aplicável para os downloads C/C++ e "Completa"), além disso é necessário instalar os compiladores e as ferramentas. Para obter mais informações, consulte Instalando e configurando o suporte a C/C++.
  3. Se você deseja usar o Sun Java System Access Manager, além da opção de download "Completa" do IDE, precisará instalar a Java Application Platform SDK Atualização 3 ou superior a partir de http://java.sun.com/javaee/downloads/index.jsp. Certifique-se de fazer o download da versão usando os botões Download ou Download com JDK. Para obter mais informações sobre as etapas necessárias para instalar o Access Manager, consulte Instalando o Sun Java System Access Manager.
  4. Para desenvolver aplicações Mobility em dispositivos CDC é necessário baixar e instalar o Sun Java Toolkit 1.0 for CDC . Após a instalação você deve registrar a plataforma CDC no IDE.
  5. Para Mac OS X, não é possível personalizar a instalação do IDE NetBeans.

Iniciando o download

  1. Acesse a página Download do IDE NetBeans 6.0.
  2. Na parte superior direita da página, selecione a plataforma na lista suspensa.
  3. Clique no botão Download para selecionar a opção de download que deseja instalar.
  4. Salve o arquivo do instalador no sistema.

Instalando o software

Siga as instruções nesta seção para instalar o IDE no sistema. Estas instruções de instalação aplicam-se a todas as plataformas compatíveis. Para obter uma lista das plataformas compatíveis, consulte as Notas de versão do IDE NetBeans 6.0.

Microsoft Windows, Solaris OS e Linux

Para instalar o software:

  1. Após concluir o download, execute o instalador.
    • Para Windows, o arquivo executável do instalador tem a extensão .exe. Clique duas vezes no arquivo do instalador para executá-lo.
    • Nas plataformas Solaris e Linux, o arquivo do instalador tem a extensão .sh. Nessas plataformas, você precisa executar os arquivos do instalador com o seguinte comando:
      chmod +x ./<installer-file-name>
  2. Para selecionar as ferramentas e os tempos de execução a serem instalados, execute as seguintes etapas na página de boas-vindas do assistente de instalação:
    1. Clique em Personalizar.
    2. Na caixa de diálogo Personalizar a instalação, faça suas seleções:
    3. Clique em OK.
  3. Na página de boas-vindas do assistente de instalação, clique em Avançar.
  4. Na página Contrato de licença, leia o contrato, selecione a caixa de seleção para aceitar e clique em Avançar.
  5. Na página de instalação do IDE NetBeans, proceda da seguinte maneira:
    1. Aceite o diretório padrão de instalação do IDE NetBeans ou especifique outro diretório.
      Nota: o diretório de instalação deverá estar vazio e o perfil de usuário que você está usando para executar o instalador deverá ter permissões de leitura/gravação para esse diretório.
    2. Aceite a instalação padrão do JDK para usar com o IDE NetBeans ou selecione uma instalação diferente na lista suspensa.
      Se o assistente de instalação não encontrou uma instalação JDK compatível para ser usada com o IDE NetBeans, o JDK não está instalado no local padrão. Nesse caso, especifique o caminho para um JDK instalado ou cancele a instalação atual, instale a versão necessária do JDK e reinicie esta instalação.
    3. Clique em Avançar.
  6. Se a página de instalação do servidor de aplicações GlassFish abrir, proceda da seguinte maneira:
    1. Aceite o diretório padrão de instalação do servidor ou especifique outro diretório de instalação.
      Nota: o diretório de instalação que você especificar deverá estar vazio e o perfil de usuário que você está usando para executar o instalador deverá ter permissões de leitura/ gravação para este diretório.
    2. Na lista de instalações compatíveis do JDK, selecione o JDK que você deseja usar no servidor de aplicações ou clique no botão Procurar para especificar outra instalação de JDK.
    3. Altere o nome de usuário e a senha para o domínio padrão do servidor ou aceite os padrões e clique em Avançar.
      O nome de usuário e a senha padrão são admin e adminadmin.
    4. Verifique os valores-padrão das portas (HTTP, HTTPS e Admin) para o servidor ou altere-os, se necessário.
    5. Clique em Avançar.
  7. Se a página de instalação do Apache Tomcat abrir, aceite o diretório padrão de instalação ou especifique outro local de instalação.
  8. Na página Resumo, verifique se a lista de componentes a serem instalados está correta e se você tem espaço suficiente no sistema para a instalação.
  9. Clique em Instalar para iniciar a instalação.
  10. Quando a instalação estiver concluída, clique em Concluir para sair do assistente.

Nota: se você encontrar problemas após concluir instalação do software com êxito, consulte Solução de problemas para obter descrições e soluções para as questões não-resolvidas que poderão afetar o processo de instalação.

Mac OS X

Para instalar o software:

  1. Após concluir o download, execute o instalador. O arquivo do instalador tem a extensão .dmg.

    Nota: a opção de download "Completa" contém os quatro pacotes a seguir: IDE NetBeans, servidor de aplicações GlassFish V2, Apache Tomcat e o software OpenESB. Você precisa executá-los sucessivamente para instalar o IDE NetBeans completo no Mac OS X. O OpenESB deve ser instalado após a instalação do software de servidor de aplicações GlassFish.
  2. Na página de boas-vindas do assistente de instalação, clique em Continuar.
  3. Leia o contrato de licença e clique em Continuar. Clique em Aceitar na janela pop-up para aceitar a licença.
  4. Na página Selecionar um destino. selecione o drive e clique em Continuar.
  5. Digite o nome do administrador e a senha no sistema e clique em OK para iniciar a instalação.

Desinstalando o software

Microsoft Windows

Se necessário, você pode desinstalar o IDE NetBeans, o servidor de aplicações GlassFish e o Apache Tomcat usando os seguintes procedimentos.

Para desinstalar o IDE:

  1. Feche o IDE.
  2. No Painel de controle, selecione Adicionar ou remover programas e então selecione IDE NetBeans 6.0 e clique em Remover.

Para desinstalar o servidor de aplicações GlassFish:

  1. Feche o IDE.
  2. No Painel de controle, selecione Adicionar ou remover programas e então selecione GlassFish V2 e clique em Remover.

Nota o servidor de aplicações GlassFish é necessário para a funcionalidade SOA. Quando você desinstalar o servidor de aplicações, a funcionalidade SOA será removida automaticamente da instalação do IDE NetBeans.

Para desinstalar o Apache Tomcat:

  1. Feche o IDE.
  2. No Painel de controle, selecione Adicionar ou remover programas e então selecione Apache Tomcat 6.0.14 e clique em Remover.

Solaris OS, Linux

Se necessário, você pode desinstalar o IDE NetBeans, o servidor de aplicações GlassFish e o Apache Tomcat usando os seguintes procedimentos.

Para desinstalar o IDE:

  1. Feche o IDE.
  2. No diretório de instalação do IDE, execute o arquivo uninstall.sh:
  3. Na página Resumo, clique em Desinstalar.
  4. Quando a desinstalação for concluída, clique em Concluir.

Para desinstalar o servidor de aplicações GlassFish:

  1. Feche o IDE.
  2. Localize o diretório de instalação do servidor de aplicações.

    Nota: o local padrão de instalação será diferente, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório padrão de instalação do servidor de aplicações.

    Plataforma Instalando como... Instalação da SOA

    		 raiz /opt/glassfish-v2
    Solaris OS
    Linux    		 usuário ~/glassfish-v2
    Linux raiz /usr/local/glassfish-v2
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação for concluída, clique em Concluir.

Nota o servidor de aplicações GlassFish é necessário para a funcionalidade SOA. Quando você desinstalar o servidor de aplicações, a funcionalidade SOA será removida automaticamente da instalação do IDE NetBeans.

Para desinstalar o Apache Tomcat:

  1. Feche o IDE.
  2. Localize o diretório de instalação do Apache Tomcat e execute o arquivo uninstall.sh.
  3. Na página Resumo, clique em Desinstalar.
  4. Quando a desinstalação for concluída, clique em Concluir

Mac OS X

Para desinstalar o IDE e o servidor de aplicações:

  1. Feche o software.
  2. Arraste as pastas ou arquivos de programas para a Lixeira.
    Observe que o local padrão do IDE e do servidor de aplicações é /Applications/NetBeans/.

Atualizando e coexistindo com versões anteriores

Se você tiver versões anteriores do IDE NetBeans 6.0 (M9 Preview, Beta 1, Beta 2, RC1 ou RC2) e deseja fazer o upgrade para a versão mais recente do FCS, recomendamos que você desinstale a versão anterior. Para tanto, execute as seguintes etapas:

  1. Desinstale a versão 6.0 anterior do sistema.
    Siga o procedimento de desinstalação, conforme descrito neste documento. Para obter detalhes, consulte Desinstalando o software.
  2. Execute o instalador do IDE NetBeans 6.0 e instale o IDE, conforme descrito em Instalando o software.

Os IDE NetBeans 5.5 ou 5.5.1 podem coexistir com o IDE NetBeans 6.0 no mesmo sistema. Você não precisa instalar as versões 5.5 ou 5.5.1 para instalar ou executar o IDE NetBeans 6.0. Durante a primeira execução do IDE NetBeans 6.0, você poderá optar se deseja importar as configurações de usuário do diretório do usuário anterior.

Solução de problemas

Sobre o diretório de trabalho do instalador

O instalador armazena dados sobre as sessões de instalação no diretório de trabalho do instalador. Exemplos desses dados são a lista de componentes instalados no sistema, as mensagens do log de instalação e os dados de registro.

O local padrão do diretório de trabalho do instalador é %USERPROFILE%\.nbi no Windows e ~/.nbi em outras plataformas. O subdiretório do log contém os arquivos de log de instalação.

Limpeza após instalações incompletas

Após instalações incompletas ou malsucedidas, talvez seja necessário limpar o sistema das instalações anteriores.

Para Microsoft Windows, Solaris OS e Linux:

  1. Exclua o diretório de trabalho do instalador.
  2. Exclua os diretórios de instalação do IDE e do servidor de aplicações GlassFish.
  3. Na plataforma Windows, exclua todas as entradas de registro nbi* nas seguintes chaves de registro:
    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall
    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Para Mac OS X:

  • Arraste as pastas ou arquivos de programas para o Lixeira.

Desabilitando alguns produtos de software antes da instalação

É melhor desabilitar o firewall, o antivírus e os aplicativos de monitoramento antes de iniciar a instalação. Alguns desses aplicativos podem evitar que os componentes do IDE sejam configurados corretamente durante o processo de instalação.

Por exemplo, se você possuir o aplicativo Logitech Process Monitor (LVPrcSrv.exe) no sistema, desative-o antes da instalação.
