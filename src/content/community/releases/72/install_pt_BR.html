Instruções de Instalação do NetBeans IDE 7.2

Última atualização: October 31, 2013

Este documento descreve como instalar o NetBeans IDE 7.2 no seu sistema. Consulte as Notas da Release do NetBeans IDE 7.2 para obter informações sobre os sistemas operacionais e configurações de hardware compatíveis para o IDE. Para saber mais sobre as novas funcionalidades incluídas nesta release do IDE, consulte a página Informações da Release do NetBeans IDE 7.2.

Software Necessário

É necessário ter o JDK (Java SE Development Kit) 6, Atualização 26, ou posterior, para instalar o NetBeans IDE. Você pode fazer download da última atualização de JDK 6 ou 7 em http://www.oracle.com/technetwork/java/javase/downloads.

Observação: A versão 7.2 do IDE não pode ser instalada ou executada utilizando o JDK 5.0.

Os pacotes PHP e C/C++ NetBeans requerem apenas que o JRE (Java Runtime Environment) 6 seja instalado e esteja em execução. No entanto, se você tiver a intenção de utilizar qualquer funcionalidade Java, será necessário o JDK 6 ou 7.

O JavaFX 2.1.1 SDK é necessário para usar os recursos do JavaFX 2.1.1 no Netbeans IDE 7.2. Você pode fazer download do JavaFX 2.1.1 SDK do web site de download do JavaFX.

Observação: dependendo de seu sistema operacional, você pode instalar o JavaFX 2.1.1 empacotado com o Java JDK. Caso contrário, será necessário instalar o JavaFX 2.1.1 separadamente.

  • Para o Windows, o JavaFX 2.1.1 é empacotado com o JDK 1.7 Atualização 4. Se você instalar o Java FX 2.1.1 SDK e Runtime empacotados nas localizações default, o NetBeans IDE pode criar uma plataforma Java ativada por JavaFX automaticamente.
  • Para OS X, o JavaFX 2.1.1 é incorporado no Mac JDK 1.7 Atualização 4. O NetBeans IDE pode criar uma plataforma Java ativada por JavaFX automaticamente.
  • O JavaFX não é executado em todos os sistemas operacionais. Verifique o documento Requisitos do Sistema JavaFX 2.1.1 para ver quais plataformas são suportadas por JavaFX. Os requisitos do sistema para JavaFX 2.1.1 também permitem que você instale o NetBeans IDE.

Opções de Download do Instalador

Na página Download do NetBeans IDE, você pode adquirir um dos diversos instaladores, cada um contendo o IDE básico e as ferramentas adicionais.

Os seguintes instaladores estão disponíveis:

  • Java SE. Oferece suporte para todos os recursos de desenvolvimento padrão do Java SE e também para desenvolvimento da Plataforma NetBeans e para o JavaFX 2.1.1 SDK.
  • Java EE. Fornece ferramentas para o desenvolvimento de aplicações Java SE e Java EE, assim como para a Plataforma de desenvolvimento NetBeans e para o JavaFX 2.1.1 SDK. Esta opção de download inclui também o GlassFish Server Open Source Edition 3.1.2.2 e o software Apache Tomcat 7.0.27.
  • C/C++. Oferece suporte para desenvolvimento nas linguagens C, C++, Fortran e Assembly.
  • PHP. Fornece ferramentas para o desenvolvimento de PHP 5.x e o suporte para o Zend e Symfony Framework.
  • Tudo. Esta é uma opção de download completa, que contém todos os runtimes e tecnologias disponíveis para o NetBeans IDE.

Personalizando Sua Instalação

Se você tiver feito download da opção Tudo, poderá escolher exatamente as ferramentas e os runtimes a serem instalados. Clique em Personalizar na página de Boas-Vindas do instalador e selecione e as funcionalidades que deseja utilizar. Para obter mais informações, consulte Instalando o Software.

Se você deseja adicionar funcionalidade a sua configuração do NetBeans IDE, utilize o Gerenciador de Plug-ins do NetBeans. Por exemplo, presuma que você tenha iniciado pelo download e instalação do pacote do PHP. Em seguida, você pode decidir se deseja experimentar a funcionalidade C/C++. Para fazer isso, acesse o Gerenciador de Plug-ins do NetBeans no IDE (selecione Ferramentas > Plug-ins) e adicione o pacote C/C++ à instalação existente.

Há diversas observações importantes sobre a instalação e a configuração de alguns recursos do NetBeans IDE:

  • O Apache Tomcat 7.0.27 está incluído nas opções de download Java EE e Tudo, mas não está instalado por default nessas opções. Para instalar o Apache Tomcat, selecione a opção adequada abaixo:
    • no download do Java EE, inicie o instalador e marque a caixa de seleção Apache Tomcat 7.0.27 na página de Boas-Vindas;
    • no download de Tudo, inicie o instalador e selecione Apache Tomcat 7.0.27 na caixa de diálogo Personalizar Instalação.
  • Para ativar a compatibilidade com o JavaFX 2.1.1 no NetBeans IDE release 7.2, consulte o documento Configurando o NetBeans IDE com JavaFX 2.1.
  • Para OS X, você pode personalizar a instalação do NetBeans IDE dos downloads Tudo e Java EE. A instalação do IDE a partir dos downloads de Java SE, C/C++ e PHP não pode ser personalizada.
  • Também é possível fazer download de um arquivo zip independente de plataforma e executar o arquivo executável do NetBeans IDE. Observe que o GlassFish Server Open Source e o Apache Tomcat estão disponíveis somente com instaladores específicos para cada plataforma.
  • Para tornar modelos de projeto de amostra inacessíveis na instalação do NetBeans IDE, consulte a página Perguntas Frequentes sobre Ocultar Projetos de Amostra.

Iniciando o Download

  1. Acesse http://download.netbeans.org/netbeans/7.2/
  2. Na parte superior direita da página, selecione a linguagem e a plataforma na lista drop-down. Também é possível optar por fazer download e utilizar somente o arquivo zip independente da plataforma.
  3. Clique no botão Download para selecionar a opção de download que deseja instalar.
  4. Salve o arquivo do instalador no sistema.

Instalando o Software

Siga as instruções nesta seção para instalar o IDE no sistema. Estas instruções de instalação se aplicam a todas as plataformas compatíveis. Para obter uma lista das plataformas compatíveis e dos requisitos do sistema, consulte as notas da release.

Microsoft Windows, Solaris OS e Linux

Para instalar o software:

  1. Após concluir o download, execute o instalador.
    • Para Windows, o arquivo executável do instalador tem a extensão .exe. Clique duas vezes no arquivo do instalador para executá-lo.
    • Nas plataformas Solaris e Linux, o arquivo do instalador tem a extensão .sh. Para essas plataformas, é preciso executar os arquivos do instalador executáveis com o seguinte comando: chmod +x <installer-file-name>
  2. Se você tiver feito download do pacote Tudo ou Java EE, poderá personalizar a sua instalação. Execute as seguintes etapas na página de Boas-Vindas do assistente de instalação:
    1. Clique em Personalizar.
    2. Na caixa de diálogo Personalizar Instalação, faça suas seleções.
    3. Clique em OK.
  3. Clique em Próximo na página de Boas-Vindas do assistente de instalação.
  4. Na página Contrato de Licença, leia o contrato de licença, clique na caixa de seleção para aceitar e clique em Próximo.
  5. Na página Contrato de Licença do JUnit, se optar por instalar o JUnit, clique na opção apropriada e clique em Próximo.
  6. Na página de instalação do NetBeans IDE, proceda da seguinte maneira:
    1. Aceite o diretório default de instalação do NetBeans IDE ou especifique outro diretório.

      Observação: o diretório de instalação deverá estar vazio e o perfil de usuário que estiver sendo utilizado para executar o instalador deverá ter permissões de leitura/gravação nesse diretório.

    2. Aceite a instalação default do JDK para utilizar com o NetBeans IDE ou selecione outra instalação na lista drop-down. Se o assistente de instalação não tiver encontrado uma instalação JDK compatível para ser utilizada com o NetBeans IDE, é porque o JDK não está instalado no local default. Nesse caso, especifique o caminho do JDK instalado e clique em Próximo ou cancele a instalação atual. Após a instalação da versão necessária do JDK, você poderá reiniciar a instalação.
  7. Se a página de instalação do GlassFish Server Open Source Edition 3.1.2.2 for aberta, aceite o diretório de instalação default ou especifique outro local de instalação.
  8. Se estiver instalando o Apache Tomcat, em sua página de instalação, aceite o diretório de instalação default ou especifique outro local de instalação. Clique em Próximo.
  9. Na página Resumo, verifique se a lista de componentes a serem instalados está correta e se você tem espaço suficiente no sistema para a instalação.
  10. Clique em Instalar para iniciar a instalação.
  11. Na página Configuração Concluída, forneça os dados de utilização anônimos, se desejado, e clique em Finalizar.

Observação: se você tiver problemas após concluir instalação do software com sucesso, consulte Solução de Problemas para obter descrições e soluções para problemas não-resolvidos que poderão afetar o processo de instalação.

OS X

Para instalar o software:

  1. Após concluir o download, execute o instalador. O arquivo do instalador tem a extensão .dmg.
  2. Clique no ícone de pacote no painel exibido. A extensão do pacote é .mpkg . O assistente de instalação é iniciado.
  3. Clique em Continuar quando for exibida a caixa de diálogo "Este pacote executará um programa para determinar se o software pode ser instalado".
  4. Na página de Introdução do assistente de instalação, clique em Continuar.
  5. Leia o contrato de licença e clique em Continuar. Clique em Aceitar na janela pop-up para aceitar a licença.
  6. Na página Selecionar um Destino, selecione o drive e clique em Continuar.
  7. Se você tiver feito download do pacote Tudo ou Java EE, poderá personalizar a sua instalação. Pressione o botão Personalizar no lado esquerdo inferior do painel do assistente de instalação, no último painel do assistente de instalação. A árvore de produtos é exibida.
  8. Selecione os produtos que deseja instalar.
  9. Para iniciar a instalação, digite o nome do administrador e a senha no sistema e clique em OK.

Pacote Independente de Plataforma

Para executar o software:

  1. Após fazer download do arquivo ZIP independente de plataforma, extraia-o em qualquer pasta do sistema.
  2. Execute o arquivo executável localizado no diretório netbeans/bin.
  3. Aceite o Contrato de Licença.
  4. Na página Contrato de Licença do JUnit, se optar por instalar o JUnit, clique na opção apropriada e clique em Próximo. A instalação do NetBeans IDE se inicia.

Observação: se não for possível encontrar uma instalação do JDK compatível, talvez seja necessário adicionar manualmente o caminho ao diretório de instalação do JDK, adotando o seguinte procedimento:

  1. Abra o arquivo netbeans.conf localizado no diretório netbeans/etc em um editor de texto.
  2. Insira a localização da instalação do JDK compatível para a opção netbeans_jdkhome. O local default no Windows é C:\Program Files\Java\jdk1.7.0_03.
  3. Salve o arquivo netbeans.conf e execute o arquivo executável no diretório netbeans/bin.

Observação: o GlassFish Server Open Source Edition e o Apache Tomcat somente estão disponíveis com instaladores específicos de cada plataforma.

Observação: para usar o GlassFish Server Open Source Edition e/ou Apache Tomcat já instalados na máquina, no NetBeans IDE, clique em Servidores no menu Ferramentas. Dessa forma, o Assistente de Adição de Instância de Servidor será exibido, o que vai ajudá-lo a registrar uma instância do servidor Web necessário com o NetBeans IDE.

Desinstalando o Software

Siga as instruções nesta seção para desinstalar o IDE do seu sistema. Estas instruções de desinstalação se aplicam a todas as plataformas compatíveis. Para obter uma lista das plataformas compatíveis, consulte as notas da release.

Microsoft Windows

Se necessário, é possível desinstalar o NetBeans IDE, o GlassFish Server Open Source 3.1.2.2 e o Apache Tomcat utilizando os procedimentos a seguir.

Para desinstalar o IDE:

  1. Faça shutdown do IDE.
  2. No Painel de Controle, selecione Adicionar ou Remover Programas, selecione NetBeans IDE 7.2 e clique em Remover.
  3. Para desinstalar o GlassFish, o Tomcat e o IDE ao mesmo tempo, na página Resumo, assegure-se de que as versões listadas são as corretas e de que as caixas apropriadas estão marcadas.
  4. Marque a caixa de seleção de diretório de usuário do NetBeans se quiser removê-lo com o IDE e clique em Desinstalar.
  5. Na página Resumo, clique em Desinstalar.
  6. Quando a desinstalação estiver concluída, clique em Finalizar.

Para desinstalar o GlassFish Server Open Source Edition 3.1.2.2:

  1. Faça shutdown do IDE.
  2. No Painel de Controle, selecione Adicionar ou Remover Programas. Em seguida, selecione GlassFish Server Open Source Edition 3.1.2.2 e clique em Remover.

Para desinstalar o Apache Tomcat:

  1. Faça shutdown do IDE.
  2. No Painel de Controle, selecione Adicionar ou Remover Programas. Em seguida, selecione Apache Tomcat 7.0.27 e clique em Remover.

Solaris OS, Linux

Se necessário, é possível desinstalar o NetBeans IDE, o GlassFish Server Open Source 3.1.2.2 e o Apache Tomcat utilizando os procedimentos a seguir.

Para desinstalar o IDE:

  1. Faça shutdown do IDE.
  2. No diretório de instalação do IDE, execute o arquivo uninstall.sh.
  3. Para desinstalar o GlassFish, o Tomcat e o IDE ao mesmo tempo, na página Resumo, assegure-se de que as versões listadas são as corretas e de que as caixas apropriadas estão marcadas.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação estiver concluída, clique em Finalizar.

Para desinstalar o GlassFish Server Open Source Edition 3.1.2.2:

  1. Faça shutdown do IDE.
  2. Localize o diretório de instalação do servidor de aplicações.

    Observação: o local de instalação default será diferente, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório de instalação default do servidor de aplicações para várias plataformas.

    Plataforma Instalando Como... Instalação do GlassFish 3.1.2.2
    Windows usuário com direitos de administrador %ProgramFiles%\glassfish-3.1.2.2
    Windows usuário %USERPROFILE%\glassfish-3.1.2.2
    Solaris OS  raiz /opt/glassfish-3.1.2.2
    Solaris OS, Linux usuário ~/glassfish-3.1.2.2
    Linux raiz /usr/local/glassfish-3.1.2.2
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação estiver concluída, clique em Finalizar.

Para desinstalar o Apache Tomcat:

  1. Faça shutdown do IDE.
  2. Localize o diretório de instalação do Apache Tomcat.

    Observação: o local de instalação default varia, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório de instalação default do servidor de aplicações para diferentes plataformas.

    Plataforma Instalando Como... Instalação do Apache Tomcat 7.0.27
    Windows usuário com direitos de administrador %ProgramFiles%\Apache Software Foundation\Apache Tomcat 7.0.27
    Windows usuário %USERPROFILE%\Apache Software Foundation\Apache Tomcat 7.0.27
    Solaris OS  raiz /opt/apache-tomcat-7.0.27
    Solaris OS, Linux usuário ~/apache-tomcat-7.0.27
    Linux raiz /usr/local/apache-tomcat-7.0.27
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação estiver concluída, clique em Finalizar

OS X

Para desinstalar o IDE e o servidor de aplicações:

  1. Faça shutdown do software.
  2. Arraste as pastas ou arquivos de programas para a Lixeira. Observe que o local default do IDE e do servidor de aplicações é /Applications/NetBeans/.

Pacote Independente de Plataforma

Para desinstalar os arquivos do IDE, siga as etapas descritas na seção Limpeza Após Instalações Incompletas deste documento.

Suporte a Várias Instalações e Upgrade

Várias instalações do NetBeans IDE 5.x, 6.x e 7.x podem coexistir com o NetBeans IDE 7.2 no mesmo sistema. Você não precisa desinstalar as versões anteriores para instalar ou executar o NetBeans IDE 7.2.

Se você já tiver um NetBeans IDE instalado, terá a opção de importar ou não as definições de usuário do diretório existente ao executar o NetBeans 7.2 pela primeira vez.

Observação: upgrades incrementais de versão do IDE não podem ser executados neste momento. Para fazer upgrade para a versão mais recente, é preciso fazer download do instalador mais recente.

Solução de Problemas

Sobre o Diretório de Trabalho do Instalador

O instalador armazena dados sobre as sessões de instalação no diretório de trabalho do instalador. Exemplos desses dados são a lista de componentes instalados no sistema, as mensagens do log de instalação e os dados de registro.

O local default do diretório de trabalho do instalador é %USERPROFILE%\.nbi no Windows e ~/.nbi em outras plataformas. O subdiretório de log contém os arquivos de log de instalação.

Limpeza após Instalações Incompletas

Após instalações incompletas ou malsucedidas, talvez seja necessário remover as instalações anteriores do sistema.

Para Microsoft Windows, Solaris OS e Linux:

  1. Delete o diretório de trabalho do instalador.
  2. Delete os diretórios de instalação do IDE e do servidor de aplicações GlassFish.
  3. Na plataforma Windows, delete todas as entradas do registro menos a nbi* de todas as chaves de registro: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Para OS X:

  • Arraste as pastas ou arquivos de programas para a Lixeira.

Desativando Alguns Produtos de Software Antes da Instalação

É melhor desativar o firewall, o antivírus e as aplicações de monitoramento antes de iniciar a instalação. Algumas dessas aplicações podem impedir que componentes do IDE sejam configurados corretamente durante o processo de instalação.

Por exemplo, se você tiver a aplicação Logitech Process Monitor (LVPrcSrv.exe) no sistema, desative-a antes da instalação.
