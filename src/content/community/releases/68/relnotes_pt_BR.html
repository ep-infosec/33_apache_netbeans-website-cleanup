Notas de versão do NetBeans IDE 6.8

Última atualização: July 10, 2013

O NetBeans IDE é um ambiente de desenvolvimento integrado, modular e baseado em padrões, escrito na linguagem de programação Java. O projeto do NetBeans consiste em um IDE de código-fonte aberto com recursos completos escrito na linguagem de programação Java e em uma plataforma rica cliente do aplicativo , que pode ser utilizada como uma estrutura genérica para construir qualquer tipo de aplicativo. Para obter mais informações sobre como instalar o software no sistema, consulte as Instruções de instalação do NetBeans IDE 6.8 . Para saber mais sobre os novos recursos incluídos nesta versão do IDE, consulte a página Informações da versão 6.8 do NetBeans IDE.

O que há de novo na 6.8

O NetBeans IDE 6.8 é uma atualização significativa do NetBeans IDE 6.7.1 e inclui os seguintes aprimoramentos:

  • Suporte ao Java EE 6
  • Suporte ao JSF 2.0 com muitos recursos novos
  • Suporte ao PHP 5.3 e ao framework Symfony
  • Integração aprimorada dos recursos Kenai e suporte a rastreamento de bug no Jira
  • As melhorias adicionais estão listadas na página O novo e notável NetBeans IDE 6.8.

Para saber mais sobre esta versão, consulte a página Informações da versão 6.8 do NetBeans IDE.

Informações de suporte a vários idiomas

O NetBeans IDE está disponível nos seguintes idiomas suportados pela Sun: português do Brasil (BR), japonês (JP) e chinês simplificado (ZH).

Compatibilidade

Ao ser executado pela primeira vez depois da instalação, o IDE oferece a opção de importar algumas das configurações usadas na instalação anterior. O IDE é aberto usando as configurações padrão caso você opte por não importar as configurações da instalação existente. Observe que o IDE reconhece as instalações anteriores somente quando o diretório de usuário estiver na localização padrão. Ele não reconhece as instalações que usam --userdir para especificar um diretório padrão. Se desejar importar as configurações de uma localização padrão que o instalador não reconhece, adote o seguinte procedimento:

  1. Inicie o IDE, vá a Ferramentas > Opções no menu principal.
  2. Clique no botão Importar na parte inferior da caixa de diálogo Opções.
  3. Na caixa de diálogo Selecionar opções de importação, clique no botão Procurar e vá até o diretório de usuário que contém as configurações que deseja importar. Para obter mais informações sobre a localização do userdir do NetBeans IDE, consulte a FAQ O que é o userdir e onde posso encontrá-lo?.

Nota:se você editar as Propriedades do Projeto no NetBeans IDE 6.8, de um projeto criado no NetBeans IDE 4.1 ou anterior, não será mais possível trabalhar com esse projeto em versões anteriores do NetBeans IDE.

Problemas conhecidos: instalador

Os problemas do instalador não-resolvidos desta versão são os seguintes:

  • Problema Nº. 6720021: o instalador não funciona ao usar o OpenSolaris com qualquer versão do JDK 5.

    Descrição: o instalador não funciona ao usar o OpenSolaris com qualquer versão do JDK 5.

    Solução alternativa: instale o JDK 6 ou execute o comando "export AWT_TOOLKIT=XToolkit" antes de executar o instalador usando o JDK 5.

  • Problema nº 93568: no Solaris e no Linux, a instalação pára quando a conexão de rede é interrompida.

    Descrição: nas plataformas Solaris e Linux, se a conexão de rede é interrompida, a instalação pára quando o instalador está buscando um JDK compatível nos diretórios de rede.

    Solução alternativa: restabeleça a conexão de rede e prossiga com a instalação.

  • Problema nº 117172: uma janela em branco ao executar o instalador no Ubuntu.

    Descrição: o instalador é exibido como uma caixa de diálogo em branco/cinza ao ser executado no Ubuntu com Beryl/Compiz/Compiz Fusion instalado.

    Solução alternativa: execute o comando export AWT_TOOLKIT=MToolkit antes de executar o instalador.

  • Problema nº 123300. No Fedora Core 8, o instalador interrompe a exibição da interface e volta ao console.

    Descrição: no Fedora Core 8, a exibição da interface de instalação é interrompida e o sistema volta ao console.

    Solução alternativa: consulte https://netbeans.org/servlets/ReadMsg?list=nbusers&msgNo=98765.

  • O assistente de instalação do NetBeans IDE 6.8 não inicia corretamente (trava). Depois que o usuário interrompe o processo de instalação manualmente, o programa de instalação informa que outra instância de instalação está em execução.

    Descrição: No Windows, quando o Java6 é utilizado como padrão e grandes arquivos compactados são armazenados na área de trabalho, o assistente de instalação não inicia devidamente (trava). Depois que o usuário interrompe o processo de instalação manualmente, o programa de instalação informa que outra instância de instalação está em execução.

    Solução Alternativa: Antes da instalação, remova todos os grandes arquivos compactados da área de trabalho.

  • Problema nº. 177872: no Mac OS X, o complemento e as ferramentas de atualização do GlassFish v3 podem não funcionar corretamente

    Descrição: no Mac OS X, o complemento com base no pkg e as ferramentas de atualização não funcionam apropriadamente sobre contas não de raiz se o GlassFish v3 foi ou não instalado.

    Solução: use o comando chown -R para alterar a propriedade da instalação do GlassFish e defina-o para o usuário não da raiz que o necessita. Por exemplo: sudo chown -R <newuser>:admin /Applications/NetBeans/glassfish-v3-, onde newuser é o usuário não da raiz. Após este comando ter sido utilizado, o usuário pode executar diretamente as ferramentas bin/updatetool e bin/pkg.

Problemas conhecidos: IDE do núcleo

Os problemas não-solucionados desta versão são os seguintes:

  • Dificuldades para iniciar o IDE (Solaris x86 e Dual-Core Opteron).

    Descrição: em certas configurações do Solaris x86 e do Dual Core Opteron, você poderá encontrar problemas para iniciar o IDE devido a um bug do JDK ao carregar as bibliotecas GTK look&feel.

    Solução alternativa: se você enfrentar esse comportamento, especifique a seguinte opção de linha de comando:-laf javax.swing.plaf.metal.MetalLookAndFeel

  • Os caracteres não são exibidos adequadamente em alguns locais asiáticos.

    Descrição: ao se executar o IDE em alguns locais asiáticos no Ubuntu e em algumas outras distribuições de Linux, o texto da tela do IDE pode se tornar ilegível. Em determinadas distribuições do Linux, os caracteres multibytes podem ser exibidos em forma de retângulo devido a problemas de configuração de fonte do JDK.

    Solução alternativa: consulte as FAQ para obter ajuda na resolução deste problema: http://wiki.netbeans.org/FaqI18nLinuxAsianFonts

  • Problema Nº. 6720021: dificuldade em iniciar o IDE no OpenSolaris com qualquer versão do JDK 5.

    Descrição: o IDE pode não ser iniciado no OpenSolaris ao usar qualquer versão do JDK 5.

    Solução alternativa: use o JDK 6 ou execute o comando export AWT_TOOLKIT=XToolkit antes de iniciar o IDE.

  • Problema nº 64840: comunicação de rede interrompida no Linux com IPv6 ativado.

    Descrição: comunicação de rede interrompida no Linux com IPv6 ativado, impedindo a conexão do IDE com servidores empacotados, atualização automática etc.

    Solução alternativa: se isso acontecer, execute o IDE usando a chave -J-Djava.net.preferIPv4Stack=true'. Como alternativa, desative o IPv6 (em /etc/modules.conf ou /etc/modprobe.conf).

    Para obter mais informações, consulte:Problema nº 5087907.

  • Problema Nº. 167401: o IDE congela eventualmente em sistemas MacOS X que usam o JDK 1.6.0, atualização 13

    Descrição: o IDE pode congelar eventualmente ao navegar pela barra do menu principal devido ao erro nº. 6993269 do JDK da Apple.

    Solução alternativa: passe para outra versão do JDK ou modifique as opções de início padrão do arquivo netbeans.conf para -J-Dapple.laf.useScreenMenuBar=false.

  • Problema nº 110074: o editor congela periodicamente no Solaris.

    Descrição: ao executar o IDE no Solaris com JDK 6, 6u1 ou 6u2, os segmentos podem ficar bloqueados por um longo período de tempo e fazer com que o editor não responda.

    Solução alternativa: migre sua instalação de JDK 6 para a Atualização 3 ou mais recente.

  • Problema nº 88776: aplicativo com perfil traçado pode falhar quando a criação de perfil é realizada no modo de conexão dinâmica.

    Descrição:ao executar no JDK 1.6, a criação de perfil em um aplicativo com modo de conexão dinâmica pode levar o aplicativo a falhar.

    Solução alternativa: inicie o aplicativo para o qual você deseja criar um perfil com o switch -Xshare:off.

    Consulte também: Problema nº 113847.

  • Problema Nº. 119617: O cursor desaparece no editor e não é possível utilizar o teclado.

    Descrição: As vezes, depois de usar uma dica na janela do editor, o ponto de inserção no editor desaparece e o teclado não funciona. Isto foi observado com maior freqüência no Mac OS X 10.4, mas houveram alguns relatos em algumas versões do Ubuntu.

    Solução alternativa:Use o mouse para abrir e fechar a caixa de diálogo (por exemplo, uma janela de propriedades). Isto faz com que o foco volte para o editor e o ponto de inserção e o teclado funcionará normalmente.

  • Problema Nº. 130357: I18N - Mac OS não gera o javadoc se o nome ou caminho do projeto conter multibyte.

    Descrição: A tarefa javadoc, durante a criação de um arquivo de parâmetros temporários, é escrita na codificação padrão do OS que é a MacRoman. Esta parece ser uma situação do Mac OS, onde as partes BSD e Mach (system & kernel calls) aceitam parâmetros em UTF-8, mas o OS e HFS se divulgam como MacRoman.

    Solução alternativa: Evite o uso de multibyte no nome do projeto ou no caminho do projeto.

  • Problema Nº. 152916: dificuldade em iniciar o IDE no Mac OS X 10.5.5 de 64 bits com o JDK 1.6_07

    Descrição: em determinadas configurações do Mac OS X 10.5.5 de 64 bits com o JDK 1.6_07 definido como plataforma Java padrão do sistema, pode haver problemas em iniciar o IDE a partir do localizador e da linha de comando.

    Solução alternativa: altere o sistema padrão para JDK 1.5.

  • Problema nº 150416: não é possível criar o DB de exemplo sem o InnoDB

    Descrição: o banco de dados de exemplo do NetBeans usa chaves externas. O mecanismo MyISAM não oferece suporte a chaves externas. Portanto, os bancos de dados de exemplo não podem ser usados sem o mecanismo de armazenamento InnoDB.

    Solução alternativa: instale o mecanismo de armazenamento InnoDB transacional.

  • Problema Nº. 158421: dificuldades em usar o IDE com o G1 Garbage Collector

    Descrição: se estiver usando o G1 Garbage Collector (no JDK 6, atualização 14 ou mais recente e no JDK 7), é possível que exceções sejam lançadas ao usar o IDE.

    Solução alternativa: não use o G1 Garbage Collector (remova -XX:+UnlockExperimentalVMOptions -XX:+UseG1GC do netbeans.conf ou da linha de comando).

Problemas conhecidos: Web e Java EE

Os problemas não-resolvidos da Web e Java EE para esta versão são os seguintes:

  • Problema Nº. 178108: as classes de metamodelo das entidades JPA gerenciadas não são geradas automaticamente
    Descrição: as classes de metamodelo das entidades gerenciadas correspondentes não são geradas. A criação de consultas seguras que usam a API Criteria e a classe metamodelo não funcionará.
  • Não existe suporte para Ajax Sample Components
    Descrição: esta versão não oferece suporte aos componentes Blue Prints Ajax Sample porque estão baseados em uma versão antiga do Dojo, que não é mais compatível com as versões mais recentes. Componentes Woodstock incluídos no NetBeans requerem versões mais recentes dos APIs Dojo para renderização no cliente. Para a maioria dos componentes Ajax BP popularmente utilizados (suportados no Netbeans 5.5), existe uma funcionalidade equivalente disponível no Projeto Woodstock, conforme listado abaixo.
    • Preenchimento automático - Este blog (http://blogs.sun.com/dmitry/entry/creating_autocomplete_entry_field_with ) descreve como alcançar uma funcionalidade parecida com os componentes da biblioteca do Projeto Woodstock.
    • Calendário suspenso - o Projeto Woodstock inclui um componente de Calendário.
    • Selecione o Valor de campo - Use auto completar em seu lugar, que possui uma funcionalidade parecida mas uma UI diferente.
    • Barra de progresso - Projeto Woodstock inclui um componente de barra de progresso.
    • Mapas Google - Planejado para o futuro.
    • Componente de pontuação - Planejado para o futuro.
    • Editor de Rich Text - Não suportado

    Descrição: a instrução SELECT * INTO table2 FROM table1 atualiza somente as primeiras 20 linhas da tabela 2.

Problemas conhecidos: Java ME

O NetBeans permite que você desenvolva aplicativos baseados em Java ME para dispositivos MIDP, CLDC e CDC.

Os problemas do Java ME não solucionados nesta versão são os seguintes:

  • Linux Solaris e outros sistemas operacionais não são suportados pelo Java ME SDK 3.0.

    Descrição: os pacotes NetBeans 6.8 Mobility e Java ME SDK 3.0, incluídos nessa versão, somente suporta o sistema operacional Windows.

Os usuários do Mac OS podem baixar o Java Platform Micro Edition Software Development Kit 3.0 para MacOS EA para desenvolver aplicativos móveis.     Descrição: o Java ME SDK e outros emuladores não se iniciam no Windows se o Java.exe não estiver no caminho.

    Solução alternativa: instale o JDK usando as configurações padrão ou, se estiver realizando uma instalação personalizada, certifique-se de selecionar o JRE público para a instalação. Se o JDK já estiver instalado, você pode adicionar o arquivo Java.exe às configurações de caminho do sistema.

  • Problema nº. 171338: os usuários não são capazes de criar projetos Rails

    Descrição: os usuários não são capazes de criar um projeto quando espaços estão presentes no caminho do diretório da plataforma JRuby empacotada.

    Solução: atualize manualmente o RubyGems a partir da linha de comando: nb_install_dir_ruby2/jruby-1.4/bin/jruby -S gem update --system

Problemas conhecidos: C/C++

O NetBeans C/C++ oferece suporte aos aplicativos C e C++ nos Sistemas Operacionais Microsoft Windows, Macintosh OS X, Linux e SolarisTM. Os desenvolvedores de C/C++ podem trazer os aplicativos existentes para o sistema de projeto NetBeans ou desenvolver novos aplicativos e bibliotecas. O suporte para C/C++ do NetBeans abrange o ciclo de desenvolvimento completo: projetos e modelos, um editor com reconhecimento de linguagem avançado, construções baseadas em makefile e depuração.

Para obter informações sobre a instalação e a configuração dos compiladores e das ferramentas para suporte do NetBeans C/C++, consulte Instalando e configurando o suporte para C/C++.

Os problemas C/C++ não-solucionados nesta versão são os seguintes:

  • Problema Nº. 149053: nas plataformas Windows, o caminho das ferramentas make não pode conter espaços

    Descrição: nas plataformas Windows, se o caminho das ferramentas make contiver espaços incorporados, o caminho é truncado e as ferramentas não são encontrados.

    Solução: não use espaços no caminho das ferramentas make.

  • Problema Nº. 166811: o GDB falha no Open Solaris ao iniciar o depurador de arquivos C++

    Descrição: ao depurar os projetos C++ no Open Solaris, o GDB falha devido a uma falha na segmentação.

    Solução alternativa: exclua os pacotes gcc-432 conforme descrito na solução alternativa deste bug.

  • Problema Nº. 148204: nas plataformas Windows, a entrada do usuário não funciona na janela Saída durante a depuração

    Descrição: nas plataformas Windows, quando estiver depurando um aplicativo e o tipo de console do projeto estiver definido como a janela Saída, não é possível digitar a entrada na janela Saída. Esse problema não afeta o desenvolvimento que usa um host remoto.

    Solução alternativa: defina o tipo de console como Terminal externo clicando com o botão direito do mouse no nó do projeto na janela Projetos, selecionando o nó Executar na caixa de diálogo Propriedades do projeto, selecionando Terminal externo na lista suspensa Tipo de console e clicando em OK.

  • Problema Nº. 121776: ocasionalmente, o código Sun Studio compilado tem problemas para ser depurado com gdb

    Descrição: nos sistemas operacionais Solaris e Linux, o código compilado com os compiladores Sun Studio nem sempre pode ser depurado corretamente com o depurador gdb. A maioria dos problemas conhecidos está corrigida no software Sun Studio 12, mas existem muitos outros. Se você tiver problemas para depurar programas compilados com compiladores Sun Studio 12, a primeira coisa a fazer é aplicar as correções do produto Sun Studio 12 mais recentes. Se o problema persistir, registre o defeito em Issuezilla e torne essa tarefa dependente da solução.

    À medida que os problemas forem encontrados, abriremos novos problemas e atualizaremos esta tarefa de acordo com eles, fornecendo algumas idéias do estado atual da depuração de programas compilados com compiladores Sun Studio 12 usando NetBeans e o depurador gdb.

    Esses problemas não afetam a depuração do código compilado com compiladores Sun Studio 12 usando o IDE Sun Studio 12 e o depurador dbx.

  • Problema Nº. 121775: é necessário o Xcode mais recente para depurar o Tiger (Mac OS X 10.4)

    Descrição: ao depurar no Macintosh com Tiger (Mac OS X 10.4), a versão padrão dos compiladores gcc e g++ gera o código com um bug na saída da depuração. A Apple lançou um pacote Xcode atualizado (versão 2.5) com uma correção para esse problema. As versões corretas de gcc, g++ e gdb são: gcc: 4.0.1 - g++ Apple compilação 5367: 4.0.1 - gdb Apple compilação 5367: 6.3.50 - Apple compilação 573 (10/20/2006)

    Para fazer o download dessa versão de Xcode, consulte http://developer.apple.com/tools/download.

    Esse problema não existe no Leopard (Mac OS X 10.5).

    Os sintomas do problema são que os executáveis compilados com o flag -gdwarf (o padrão para projetos de aplicativo em C/C++) têm informações de linhas incorretas e os pontos de interrupção talvez devam ser ignorados.
