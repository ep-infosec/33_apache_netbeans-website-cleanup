Instruções de instalação do NetBeans IDE 7.1.2

Última atualização: October 31, 2013

Este documento descreve como instalar o NetBeans IDE 7.1.2 em seu sistema. Consulte as Notas de Versão do NetBeans IDE 7.1.2 para obter informações sobre os sistemas operacionais suportados e sobre as configurações de hardware para o IDE. Para saber mais sobre os novos recursos incluídos nesta versão do IDE, consulte a página Informações sobre a versão do NetBeans IDE 7.1.2.

Software necessário

É necessário ter o Java SE Development Kit (JDK) 6, atualização 26, ou posterior para instalar o NetBeans IDE. A versão 7.1.2 do IDE não pode ser instalada usando JDK 5.0. O download da atualização mais recente de JDK 6 pode ser feito em http://www.oracle.com/technetwork/java/javase/downloads.

Os conjuntos PHP e C/C++ NetBeans somente requerem que o Java Runtime Environment (JRE) 6 seja instalado e executado. No entanto, se tiver a intenção de utilizar qualquer recurso Java, o JDK 6 ou JDK 7 será necessário.

O JavaFX 2.1 SDK é necessário para usar os recursos de JavaFX 2.1 no NetBeans IDE 7.1.2. O download do JavaFX 2.1 SDK pode ser feito em http://www.oracle.com/technetwork/java/javafx/downloads/index.html.

Opções de download do instalador

Na Página de download do NetBeans IDE, você pode adquirir um dos diversos instaladores, cada um contendo o IDE básico e as ferramentas adicionais.

Os seguintes instaladores estão disponíveis:

  • Java SE. Oferece suporte a todos os recursos de desenvolvimento padrão Java SE, bem como à plataforma de desenvolvimento NetBeans RCP e a JavaFX 2.1 SDK.
  • Java EE. Fornece ferramentas para o desenvolvimento de aplicativos Java SE e Java EE, assim como o suporte para a plataforma de desenvolvimento NetBeans RCP e JavaFX 2.1 SDK. Esta opção de download inclui também o GlassFish Server Open Source Edition 3.1.2, e o software Apache Tomcat 7.0.22.
  • C/C++. Oferece suporte para o desenvolvimento nas linguagens C, C++, Fortran e Assembly.
  • PHP. Fornece ferramentas para o desenvolvimento PHP 5.x e o suporte para o Zend e Symfony Framework.
  • Tudo. Trata-se de uma opção de download completo que contém todos os executáveis e tecnologias disponíveis para o NetBeans IDE.

Personalizando sua instalação

Se você fez o download da opção Tudo, pode escolher exatamente as ferramentas e os tempos de execução para a instalação. Clique em Personalizar na página de boas-vindas do instalador e selecione os recursos que deseja utilizar. Para obter mais informações, consulte Instalando o software.

Se você deseja adicionar funcionalidade a sua configuração do NetBeans IDE, utilize o NetBeans Plugin Manager. Por exemplo, suponhamos que você iniciou baixando e instalando o pacote PHP. Em seguida, você decidiu que deseja experimentar a funcionalidade C/C++. Para fazer isso, acesse o NetBeans Plugin Manager no IDE (selecione Ferramentas > Plug-ins) e adicione o pacote C/C++ à instalação existente.

Há diversas notas importantes sobre a instalação e a configuração de alguns recursos do NetBeans IDE:

  • O Apache Tomcat 7.0.22 está incluído nas opções de download Java EE e Tudo, mas não está instalado por padrão nessas opções. Para instalar o Apache Tomcat, selecione a opção apropriada abaixo:
    • a partir do download de Java EE, inicie o instalador e marque a caixa de seleção de Apache Tomcat 7.0.22 na página de boas-vindas;
    • a partir do download de Tudo, inicie a instalação e selecione Apache Tomcat 7.0.22 na caixa de diálogo Instalação personalizada.
  • Para o desenvolvimento de C/C++ (aplicável aos downloads C/C++ e Tudo), além disso é necessário instalar os compiladores e as ferramentas. Para obter mais informações, consulte o documento Configurando o NetBeans IDE para C/C++/Fortran.
  • Para ativar o suporte a JavaFX 2.1 no NetBeans IDE 7.1.2, consulte o documento Configurando o NetBeans IDE com JavaFX 2.
  • Para Mac OS X, não é possível personalizar a instalação do NetBeans IDE dos downloads de Tudo. A instalação do IDE a partir dos donloads de Java SE, Java EE, C/C++ e PHP não pode ser personalizada.
  • Também é possível fazer o download de um arquivo zip independente da plataforma e executar o arquivo executável do NetBeans IDE. Observe que o GlassFish Server Open Source e o Apache Tomcat estão disponíveis somente com instaladores específicos da plataforma.
  • Para tornar modelos de projeto de amostra inacessíveis na instalação do NetBeans IDE, consulte a página Perguntas frequentes sobre ocultar projetos de amostra.
  • Se fizer o download e instalar um patch para o Netbeans IDE 7.1, o IDE atualizado será rotulado como NetBeans 7.1 (não 7.1.2). Para atualizar o Java ME SDK 3.0 e o Glassfish 3.1.1, execute o instalador do Netbeans IDE 7.1.2.

Iniciando o download

  1. Vá para http://download.netbeans.org/netbeans/7.1.2
  2. Na parte superior direita da página, selecione a linguagem e a plataforma na lista suspensa. Também é possível optar por fazer o download e utilizar somente o arquivo zip independente da plataforma.
  3. Clique no botão Download para selecionar a opção de download que deseja instalar.
  4. Salve o arquivo do instalador no sistema.

Instalação do software

Siga as instruções nesta seção para instalar o IDE no sistema. Estas instruções de instalação aplicam-se a todas as plataformas compatíveis. Para obter uma lista de plataformas com suporte e requisitos de sistema, consulte as notas de versão.

Microsoft Windows, Solaris OS e Linux

Para desinstalar o software:

  1. após concluir o download, execute o instalador.
    • Para Windows, o arquivo executável do instalador tem a extensão .exe. Clique duas vezes no arquivo do instalador para executá-lo.
    • Nas plataformas Solaris e Linux, o arquivo do instalador tem a extensão .sh. Nessas plataformas, você precisa executar os arquivos do instalador com o seguinte comando: chmod +x <nome do arquivo do instalador>
  2. Se você baixou o pacote Java ou Tudo, você pode personalizar a instalação. Execute as seguintes etapas na página de boas-vindas do assistente de instalação:
    1. clique em Personalizar.
    2. Na caixa de diálogo Personalizar a instalação, faça suas seleções.
    3. Clique em OK.
  3. Clique em Avançar na página de boas-vindas do assistente de instalação.
  4. Na página Contrato de Licença, leia o contrato de licença, clique na caixa de seleção para aceitar e clique em Avançar.
  5. Na página Contrato de licença do JUnit, decida se deseja instalar o JUnit e clique na opção apropriada, em seguida clique em Próximo.
  6. Na página de instalação do IDE NetBeans, proceda da seguinte maneira:
    1. aceite o diretório padrão de instalação do IDE NetBeans ou especifique outro diretório.

      Observação: O diretório de instalação deverá estar vazio e o perfil de usuário que estiver sendo utilizado para executar o instalador deverá ter permissões de leitura/gravação para esse diretório.

    2. Aceite a instalação padrão do JDK para utilizar com o NetBeans IDE ou selecione uma instalação diferente na lista suspensa. Se o assistente de instalação não encontrou uma instalação JDK compatível para ser utilizada com o NetBeans IDE, o JDK não está instalado no local padrão. Nesse caso, especifique o caminho do JDK instalado e clique em Avançar ou cancele a instalação atual. Após a instalação da versão necessária do JDK, você pode reiniciar a instalação.
  7. Se a página de instalação do GlassFish Server Open Source Edition 3.1.2 for aberta, aceite o diretório padrão de instalação ou especifique outro local de instalação.
  8. Se estiver instalando o Apache Tomcat, em sua página de instalação, aceite o diretório padrão de instalação ou especifique outro local de instalação. Clique em Próximo.
  9. Na página Resumo, verifique se a lista de componentes a serem instalados está correta e se você tem espaço suficiente no sistema para a instalação.
  10. Clique em Instalar para iniciar a instalação.
  11. Na página Configuração concluída, forneça os dados de utilização anônimos, se desejar, e clique em Finalizar.

Observação:Se tiver problemas para concluir com sucesso a instalação do software, consulte Soluções de problemas para obter descrições e soluções sugeridas para problemas não solucionados que poderiam afetar o processo de instalação.

OS X

Para desinstalar o software:

  1. após concluir o download, execute o instalador. O arquivo do instalador tem a extensão .dmg.
  2. No painel que se abre, clique no ícone de pacote. A extensão do pacote é .mpkg. O assistente de instalação é iniciado.
  3. Na página de boas-vindas do assistente de instalação, clique em Continuar.
  4. Leia o contrato de licença e clique em Continuar. Clique em Aceitar na janela suspensa para aceitar a licença.
  5. Na página Selecionar um destino, selecione o drive e clique em Continuar.
  6. Se você baixou o pacote Java ou Tudo, você pode personalizar a instalação. Pressione o botão Personalizar no lado esquerdo inferior da janela do assistente de instalação, na última janela do assistente de instalação. A árvore de produtos é exibida.
  7. Selecione os produtos que você deseja instalar.
  8. Para iniciar a instalação, digite o nome do administrador e a senha no sistema e clique em OK.

Pacote independente de plataforma

Para executar o software:

  1. após o download do arquivo ZIP independente da plataforma, extraia-o em qualquer pasta do sistema.
  2. Execute o arquivo executável localizado no diretório netbeans/bin.
  3. Aceite o Contrato de Licença.
  4. Na página Contrato de licença do JUnit, decida se deseja instalar o JUnit e clique na opção apropriada, em seguida clique em Próximo. A instalação do NetBeans IDE é iniciada.

Observação: Se não for possível encontrar a instalação do JDK compatível, talvez seja necessário adicionar manualmente o caminho ao diretório de instalação do JDK, adotando o seguinte procedimento:

  1. Abra o arquivo netbeans.conf localizado no diretório netbeans/etc em um editor de texto.
  2. Insira a localização da instalação do JDK compatível para a opção netbeans_jdkhome. A localização padrão no Windows é C:\Program Files\Java\jdk1.7.0_03.
  3. Salve o arquivo netbeans.conf e execute o arquivo executável no diretório netbeans/bin.

Observação: O GlassFish Server Open Source Edition e o Apache Tomcat somente estão disponíveis com instaladores específicos da plataforma.

Observação: Para usar o GlassFish Server Open Source Edition e/ou Apache Tomcat que já estão instalados no computador, no NetBeans IDE, clique em Servers no menu Tools. É exibido o assistente Adicionar instância de servidor, que ajuda a registrar uma instância do servidor Web necessário com o NetBeans IDE.

Desinstalação do software

Siga as instruções nesta seção para desinstalar o IDE de seu sistema. Estas instruções de desinstalação aplicam-se a todas as plataformas compatíveis. Para obter uma lista das plataformas compatíveis, consulte as notas de versão.

Microsoft Windows

Se necessário, você pode desinstalar o NetBeans IDE, o GlassFish Server Open Source Edition 3.1.1 e o Apache Tomcat utilizando os procedimentos a seguir.

Para desinstalar o NetBeans IDE:

  1. Feche o IDE.
  2. No Painel de controle, selecione Adicionar ou remover programas e, em seguida, selecione NetBeans IDE 7.1.2 e clique em Remover.
  3. Para desinstalar o GlassFish, o Tomcat e o IDE ao mesmo tempo, na página Resumo assegure-se de que as versões listadas são as corretas e as caixas apropriadas estão marcadas.
  4. Marque a caixa de seleção do diretório de usuário do NetBeans, se desejar removê-lo junto com o IDE, e clique em Desinstalar.
  5. Na página Resumo, clique em Desinstalar.
  6. Quando a desinstalação for concluída, clique em Concluir.

Para desinstalar o GlassFish Server Open Source Edition 3.1.2:

  1. Feche o IDE.
  2. No Painel de controle, selecione Adicionar ou remover programas e, em seguida, selecione GlassFish Server Open Source Edition 3.1.2 e clique em Remover.

Para desinstalar o Apache Tomcat:

  1. Feche o IDE.
  2. No Painel de controle, selecione Adicionar ou remover programas e então selecione Apache Tomcat 7.0.22 e clique em Remover.

Solaris OS, Linux

Se necessário, você pode desinstalar o NetBeans IDE, o GlassFish Server Open Source Edition 3.1.1 e o Apache Tomcat utilizando os procedimentos a seguir.

Para desinstalar o NetBeans IDE:

  1. Feche o IDE.
  2. No diretório de instalação do IDE, execute o arquivo uninstall.sh.
  3. Para desinstalar o GlassFish, o Tomcat e o IDE ao mesmo tempo, na página Resumo assegure-se de que as versões listadas são as corretas e as caixas apropriadas estão marcadas.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação for concluída, clique em Concluir.

Para desinstalar o GlassFish Server Open Source Edition 3.1.2:

  1. Feche o IDE.
  2. Localize o diretório de instalação do servidor de aplicações.

    Observação: O local padrão de instalação será diferente, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório padrão de instalação do servidor de aplicativos para várias plataformas.

    Plataforma Instalando como... Instalação do GlassFish 3.1.2
    Windows usuário com direitos de administrador %ProgramFiles%\glassfish-3.1.2
    Windows usuário %USERPROFILE%\glassfish-3.1.2
    Solaris OS  raiz /opt/glassfish-3.1.2
    Solaris OS, Linux usuário ~/glassfish-3.1.2
    Linux raiz /usr/local/glassfish-3.1.2
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação for concluída, clique em Concluir.

Para desinstalar o Apache Tomcat:

  1. Feche o IDE.
  2. Localize o diretório de instalação do Apache Tomcat.

    Observação: O local padrão de instalação varia, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório padrão de instalação do servidor de aplicativos para diferentes plataformas.

    Plataforma Instalando como... Instalação do Apache Tomcat 7.0.22
    Windows usuário com direitos de administrador %ProgramFiles%\Apache Software Foundation\Apache Tomcat 7.0.22
    Windows usuário %USERPROFILE%\Apache Software Foundation\Apache Tomcat 7.0.22
    Solaris OS  raiz /opt/apache-tomcat-7.0.22
    Solaris OS, Linux usuário ~/apache-tomcat-7.0.22
    Linux raiz /usr/local/apache-tomcat-7.0.22
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação for concluída, clique em Concluir

OS X

Para desinstalar o IDE e o servidor de aplicações:

  1. Feche o software.
  2. Arraste as pastas ou arquivos de programas para a Lixeira. Observe que o local padrão do IDE e do servidor de aplicações é /Applications/NetBeans/.

Pacote independente de plataforma

Para desinstalar os arquivos do IDE, siga as etapas descritas na seção Limpeza após instalações incompletas deste documento.

Atualização e suporte a várias instalações

Várias instalações do NetBeans IDE 5.x, 6.x e 7.x podem coexistir com o NetBeans IDE 7.1.2 no mesmo sistema. Você não precisa desinstalar as versões anteriores para instalar ou executar o NetBeans IDE 7.1.2.

Se já tiver um NetBeans IDE instalado, terá a opção de importar ou não as configurações de usuário do diretório existente ao executar o NetBeans 7.1.2 pela primeira vez.

Observação: Atualizações de versão incremental do IDE não podem ser executadas neste momento. Para atualizar para a versão mais recente, é preciso efetuar o download do instalador mais recente.

Sobre o diretório de trabalho do instalador

O instalador armazena dados sobre as sessões de instalação no diretório de trabalho do instalador. Exemplos desses dados são a lista de componentes instalados no sistema, as mensagens do registro de instalação e os dados de registro.

O local padrão do diretório de trabalho do instalador é %USERPROFILE%\.nbi no Windows e ~/.nbiem outras plataformas. O subdiretório do registro contém os arquivos de registro de instalação.

Limpeza após instalações incompletas

Após instalações incompletas ou malsucedidas, talvez seja necessário limpar o sistema das instalações anteriores.

Para Microsoft Windows, Solaris OS e Linux

  1. exclua o diretório de trabalho do instalador.
  2. Exclua os diretórios de instalação do IDE e do servidor de aplicações GlassFish.
  3. Na plataforma Windows, exclua todas as entradas de registro nbi* nas seguintes chaves de registro: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Para OS X:

  • Arraste as pastas ou arquivos de programas para a Lixeira.

Desabilitando alguns produtos de software antes da instalação

É melhor desabilitar o firewall, o antivírus e os aplicativos de monitoramento antes de iniciar a instalação. Alguns desses aplicativos podem evitar que os componentes do IDE sejam configurados corretamente durante o processo de instalação.

Por exemplo, se você possuir o aplicativo Logitech Process Monitor (LVPrcSrv.exe) no sistema, desative-o antes da instalação.
