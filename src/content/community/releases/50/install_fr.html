Installation EDI NetBeans (Standalone Installer)

Ci-après, les instruction pour l'installation de l'EDI NetBeans sur toutes les plateformes supportées.

MS Windows

Sur les machines Microsoft Windows, la méthode préférée pour installer l'EDI NetBeans est d'utiliser l'installateur.

Pour installer l'EDI NetBeans 5.0:

  1. Une fois que vous avez téléchargé le fichier d'installation, double-cliquez sur son icône pour lancer l'installation.

  2. Dans l'assistant d'installation, acceptez les Conditions de Licence, et cliquez ensuite sur Next.

  3. Spécifiez un répertoire vide dans lequel installer l'EDI NetBeans.

    Note: Cet EDI NetBeans ne va pas déranger les paramètres de vos autres instalations de NetBeans, car l'EDI crée automatiquement un autre répertoire utilisateur lorsqu'il est lancé (${HOME}/.netbeans/5.0).


  4. Choisissez le JDK sous lequel vous désirez que l'EDI s'exécute parmis les choix proposés, et cliquez sur Next.

  5. Vérifiez que l'emplacement de l'installation est bien correct et que vous avez l'espace adéquat sur votre système pour l'installation.

  6. Cliquez sur Next pour commencer l'installation.

Note: Si vous rencontrez des problèmes pour installer correctement le logiciel, voyez la partieTroubleshooting ci-dessous pour une description completes de problèmes connus et leurs workarounds, qui pourraient affecter le processus d'installation.


Système d'Exploitation SolarisTM (Solaris OS)

Sur les machines tournant sous le système d'exploitation Solaris, l'installateur binaire est la méthode préférée pour l'installation de l'EDI NetBeans.

Pour installer l'EDI NetBeans 5.0:

  1. Naviguez jusqu'au répertoire qui contient l'installateur.

  2. Si nécessaire, modifier les permissions du fichier d'installation pour rendre le binaire exécutable en tapant depuis la console:
    $ chmod +x votre_binaire_exécutable

    Note:Vérifiez bien que vous avez remplacé your_binary_executable avec le nom du fichier correct du binaire que vous venez de télécharger.


  3. Lancez l'installation en introduisant sur la console:
    $ ./votre_binaire_exécutable

    Note: A nouveau, vérifiez-bien que vous remplacez your_binary_executable par le nom de fichier correct du binaire que vous avez téléchargé.


  4. Dans l'assistant d'installation, acceptez les Conditions de Licence, et cliquez ensuite sur Next.

  5. Spécifiez un répertoire vide dans lequel installer l'EDI NetBeans, et cliquez ensuite sur Next.

  6. Choisissez le JDK sous lequel vous désirez que l'EDI s'exécute parmis les choix proposés, et cliquez sur Next.

    L'installateur recherche après les JDK installées sur votre système et vous invite à spécifier lequel il doit utiliser. Vous pouvez également spécifier quel JDK l'EDI devrait utiliser depuis la ligne de commande. Par exemple :

    $ ./votre_binaire_exécutable -is:javahome chemin_vers_votre_jdk

  7. Vérifiez que l'emplacement de l'installation est bien correct et que vous avez l'espace adéquat sur votre système pour l'installation.

  8. Cliquez sur Next pour commencer l'installation.

Note: Si vous rencontrez des problèmes pour installer correctement le logiciel, voyez la partieTroubleshooting ci-dessous pour une description completes de problèmes connus et leurs workarounds, qui pourraient affecter le processus d'installation.

Enregistrement et Démarrage du Serveur d'Application

Si vous avez utilisé l'installateur seul de l'EDI NetBeans 5.0, vous devez enregistrer le Serveur d'Application avant de pouvoir l'utiliser.

Enregistrement du Serveur d'Application Sun Java System Application Server

Pour enregister le Sun Java System Application Server:

  1. Allez dans Tools > Server Manager.

  2. Dans le Server Manager, cliquez sur Add Server.

  3. Dans la boite de dialogue, sélectionnez Sun Java System Application Server 8 et cliquez sur Next.

  4. Introduisez un nom pour l'instance du serveur. Ce nom est utilisé pour identifier le serveur dans l'EDI.

  5. Introduisez l'emplacement, nom d'utilisateur ert mot de passe. Vous ne devez pas introduire l'emplacement d'installation et le domaine si vous enregistrer une instance distante du serveur. Vous devez enregister une installation locale du serveur d'application avant que vous ne puissiez enregistrer les instances distantes.

  6. Cliquez sur Finish.

Note: Si vous avez utilisé l'installateur de l'offre conjointe, le Serveur d'Application Sun Java System Application Server est installé et configuré automatiquement. Il n'y a donc plus besoin de faire la procédure d'enregistrement décrite ci-dessus.


Démarrer le Serveur d'Application Sun Java System Application Server

Pour démarrer le Serveur d'Application:

  1. Dans la fenêtre Runtime, cliquez-droit sur le noeud localhost nouvellement créé et choisissez Start/Stop Server depuis le menu contextuel.

  2. Dans la boîte de dialogue Server Status, cliquez sur Start Server.

  3. Cliquez sur Close pour fermer la boîte de dialogue.

Note: L'offre conjointe de l'EDI NetBeans et Sun Java System Application Server crée l'utilisateur admin et le mot de pase adminadmin par défaut. Les utilisateurs doivent utiliser ces paramètres pour se loguer via la console web..

Distributions sous forme d'Archives

Si vous utilisez un système d'exploitation pour lequel il n'y a pas d'installateur prévu, vous pouvez toujours utiliser l'EDI NetBeans en téléchargeant et en installant les distributions d'archives. Vous pouvez également installer les distibutions d'archives de l'EDI si vous désirez tester les versions de développement.

Pour installer les distributions d'archive de l'EDI NetBeans :

  1. Naviguez jusqu'au répertoire qui contient le fichier d'archive.

  2. Décompressez l'archive en utilisant les utilitaires appropriés pour votre système.

Pour lancer les distributions d'archive de l'EDI NetBeans:

  1. Naviguez jusqu'au sous-répertoire bin de votre installation de l'EDI NetBeans.

  2. Exécutez le lanceur approprié à votre système.

Les lanceurs suivants sont inclus dans les distributions d'archive:

  • netbean.exe - pour Windows
  • nb.exe - pour Windows (depuis la console )
  • netbeans - sous UNIX, Linux
  • NetBeans - sous Mac OS X
  • netbeans.cmd - sous OS/2

S'il n'y a pas de lanceurs spécifiques pour votre système d'exploitation, vous pourriez être amené à en créer un en modifiant le script approprié. Si votre machine supporte le JDK 1.4.2 ou plus, vous devriez, cependant, être capable d'exécuter l'EDI.

Note: Si vous créez un lanceur, vous pouvez le donner en contribution au projet NetBeans. Pour plus d'informations, voyez Contribuez à netbeans.org.

Désinstallation

Si cela s'avère nécessaire, vous pouvez désinstaller l'offre groupée. Cependant, vous devez désinstaller l' EDI NetBeans et le Serveur d'Application Sun Java System Application Server séparemment.

Désinstallation de l'offre groupée EDI NetBeans 5.0 et Sun Java System Application Server Platform Edition 8 depuis Windows

Pour désinstaller l'EDI NetBeans 5.0:

  1. Depuis le menu démarrer de votre système, sélectionnez Panneau de Configuration.

  2. Double-cliquez sur l'icône Ajout/Suppression de programmes.

  3. Sélectionnez NetBeans depuis la liste et cliquez sur Remove.

Pour désinstaller Sun Java System Application Server Platform Edition 8:

  1. Depuis le menu démarrer de votre système, sélectionnez Panneau de Configuration.

  2. Double-cliquez sur l'icône Ajout/Suppression de programmes.

  3. Sélectionnez Sun Java System Application Server Platform Edition 8 dans la liste et cliquez sur Remove.

Désinstallation de l'offre groupée EDI NetBeans 5.0 et Sun Java System Application Server Platform Edition 8 depuis Solaris (Aussi bien les plateformes SPARC que x86)

Pour désinstaller l'EDI NetBeans 5.0:

  1. Naviguez jusqu'au sous-répertoire _uninst de votre répertoire d'installation de l'EDI NetBeans.

  2. Tapez ./uninstaller pour exécuter le désinstallateur.

Pour désinstaller Sun Java System Application Server Platform Edition 8:

  1. Naviguez jusqu'au répertoire d'instaler de Sun Java System Application Server Platform Edition 8.

  2. Dans la console, tapez ./uninstall pour exécuter la désinstallation.

Désinstallation de l'offre groupée EDI NetBeans 5.0 et Sun Java System Application Server Platform Edition 8 depuis Linux

Pour désinstaller l'EDI NetBeans 5.0:

  1. Naviguez jusqu'au sous-répertoire _uninst de votre répertoire d'installation de l'EDI NetBeans.

  2. Dans la console, tapez ./uninstall pour exécuter la désinstallation.

Pour désinstaller Sun Java System Application Server Platform Edition 8:

  1. Naviguez jusqu'au répertoire d'instaler de Sun Java System Application Server Platform Edition 8.

  2. Dans la console, tapez ./uninstall pour exécuter la désinstallation.

Désinstallation de l'offre groupée EDI NetBeans 5.0 et Sun Java System Application Server Platform Edition 8 depuis Macintosh OS X

Pour désinstaller l'EDI NetBeans 5.0:

  1. Naviguez jusqu'au répertoire d'installation de l'EDI NetBeans. Par défaut, NetBeans est installé dans le répertoire Applications.

  2. Cliquez-droit sur l'icône de NetBeans et choisissez Show Package Contents depuis le menu contextuel.

  3. Dans la fenêtre qui apparait, naviguez jusqu'au sous-répertoire _uninst.

  4. Dans la console, tapez ./uninstall pour exécuter la désinstallation.

Pour désinstaller Sun Java System Application Server Platform Edition 8:

  1. Naviguez jusqu'au répertoire d'installation de Sun Java System Application Server Platform Edition 8. Par défaut, le Serveur d'Application est installe dans le sous-répertoire SUNWappserver de votre home directory.

  2. Dans la console, tapez ./uninstall pour exécuter la désinstallation.

Problèmes

Si vous rencontrez des problèmes d'installation et/ou d'exécution de ces logiciels, voyez la liste des problèmes mentionnés ci-dessous pour trouver des workarounds. Pour une liste complete de problèmes connus, recommandation système, et autres informations utiles, voyez les Notes de Version.

Ci-dessuos sont repris les problèmes non résolus pour cette version, qui pourrait affecter ou déranger le processus d'installation:

  • Problème #26965

    Description :

    Lorsqu'on utilise les installations de JDK qui incluent une version de sax.jarincompatible, l'installateur de l'EDI se plante avec le message suivant: "The wizard cannot continue because of the following error: could not load wizard specified in /wizard.inf (104)"

    Solution au Problème:

    Supprimez temporairement le fichier jre/lib/endorsed/sax.jar du répertoire du JDK durant l'installation de l'EDI.


  • Problème #52856 (Windows XP uniquement)

    Description :

    L'installation de Sun Java System Application Server échoue. Notez que ce problème se pose aussi bien pour l'offre groupée de l'EDI NetBeans 5.0 et Sun Java System Application Server Platform Edition 8 Bundle ainsi que pour l'installation du Serveur d'Application seule. Cela se produit lorsqu'une installation précédente du Serveur d'Application a échoué et des tentatives successives de désinstallation échouaient également(en fait, certaines entrées de la base de registre ou des fichiers de configuration n'ont pas été supprimés).

    Solution au Problème:

    Si cela se produit, effacez toutes les installations précédentes de Sun Java System Application Server. Ensuite, supprimer le répertoire <USERHOME>\Local Settings\temp\ de l'installateur et le répertoire \Application Data\Sun\tmp\ du Serveur d'Application (notez que les fichiers peuvent également se situer dans <WINDIR>\Sun\). Si une tentative précédent de désinstallation du Serveur d'Application avait échouée, supprimez également le fichier <SYSTEMDIR>/productregistry. Ensuite, créez un nouveau répertoire dans lequel installer le logiciel et assurez-vous que le répertoire est bien vide, et accessible en écriture. Finallement, réinstaller le Serveur d'Application dans le nouvel emplacement de votre système.


  • Problème #65482 (Linux uniquement)

    Description :

    Impossible d'installer NetBeans sur certaines distributions Linux Mandrake/Mandriva. Durant l'initialisation, l'installateur affiche le message d'erreur suivant: 

    Initialisation Assistant InstallShield ........
    ./netbeans-5_0-bin-linux.bin: line 189: bc: command not found
    ./netbeans-5_0-bin-linux.bin: line 404: [: : integer expression expected

    Solution au Problème:

    L'installateur de NetBeans utiliser l'utilitaire bc de GNU, qui ne semble pas être installé par défaut sur les distributions Mandriva. Si vous rencontrez ce problème, installez l'utilitaire bc de GNU manuellement, avant d'installer NetBeans. Si vous avez une connection Internet active, vous pouvez exécutez urpmi bc depuis la console pour accomplir cela. Il est également possible de télécharger le RPM approprié et de l'installer en utilisant la commande rpm -i <bc_package>.rpm.
