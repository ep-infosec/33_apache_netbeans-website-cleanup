NetBeansTM IDE 5.0 설치 지침

마지막 업데이트: 2006년 1월 27일

다음 지침에서는 NetBeans IDE 5.0(독립형 설치 프로그램)과 NetBeans IDE 5.0 및 Sun Java Application Server Platform Edition 8.2 번들 모두의 설치에 대해 설명합니다.

다음 설치에 대해 자세히 설명합니다.

  • NetBeans IDE 설치(독립형 설치 프로그램)
  • NetBeans IDE 및 Sun Java System Application Server 번들 설치
  • IDE 시작
  • 응용 프로그램 서버 등록 및 시작
  • 아카이브 배포
  • 제거
  • 문제 해결

주: 시스템 요구 사항이나 다른 시작 매개 변수로 IDE를 시작하는 경우 등의 다른 정보를 보려면 릴리스 노트를 참조하십시오.

NetBeans IDE 설치(독립형 설치 프로그램)

다음은 지원되는 모든 플랫폼에서 독립형 NetBeans IDE를 설치하는 경우에 적용되는 지침입니다.

Windows

Microsoft Windows 시스템에서 자주 사용되는 NetBeans IDE 설치 방법은 자동 압축 해제 설치 프로그램을 사용하는 것입니다.

NetBeans IDE 5.0을 설치하려면

  1. 설치 프로그램 파일을 다운로드하고 나면 설치 프로그램의 아이콘을 두 번 클릭하여 설치 프로그램을 시작합니다.

  2. 설치 마법사에서 사용권 계약에 응답한 후 다음을 클릭합니다.

  3. NetBeans IDE를 설치할 빈 디렉토리를 지정합니다.

    주: 이 NetBeans IDE에서는 IDE가 시작될 때 새 사용자 디렉토리(${HOME}/.netbeans/5.0)를 자동으로 생성하기 때문에 다른 NetBeans 설치에 방해가 되지 않습니다.


  4. 적절한 항목 목록에서 IDE가 사용할 JDK를 선택한 후 다음을 클릭합니다.

  5. 설치 위치가 정확하며 설치에 필요한 충분한 공간이 시스템에 있는지 확인합니다.

  6. 다음을 클릭하여 설치를 시작합니다.

주: 소프트웨어 설치를 성공적으로 완료하는 데 문제가 발생한 경우 아래의 문제 해결 절에서 자세한 설명과 설치 프로세스에 영향을 줄 수 있는 해결되지 않은 문제에 대한 해결 방법을 참조하십시오.


SolarisTM 운영 체제(Solaris OS)

Solaris 운영 체제를 실행하는 시스템에서 자주 사용되는 NetBeans IDE 설치 방법은 이진 설치 프로그램입니다.

NetBeans IDE 5.0을 설치하려면

  1. 설치 프로그램이 포함된 디렉토리로 이동합니다.

  2. 필요한 경우 명령 프롬프트에서 다음을 입력하여 설치 프로그램 파일의 권한을 변경하고 이진 파일을 실행할 수 있게 만듭니다.
    $ chmod +x your_binary_executable

    주: your_binary_executable은 실제로 다운로드한 이진 파일 이름으로 바꿔야 합니다.


  3. 명령 프롬프트에서 다음을 입력하여 설치 프로그램을 시작합니다.
    $ ./your_binary_executable

    주: 여기서도 your_binary_executable은 다운로드한 이진 파일의 실제 파일 이름으로 바꿔야 합니다.


  4. 설치 마법사에서 사용권 계약에 응답한 후 다음을 클릭합니다.

  5. NetBeans IDE를 설치할 빈 디렉토리를 지정한 후 다음을 클릭합니다.

  6. 목록에 표시된 적절한 항목 목록에서 IDE가 사용할 JDK를 선택한 후 다음을 클릭합니다.

    설치 프로그램에서 시스템에 설치된 JDK를 검색한 후 사용할 JDK를 지정하라는 내용의 메시지를 표시합니다. IDE에서 사용하는 JDK를 명령줄에서 지정할 수도 있습니다. 예를 들면 다음과 같습니다.

    $ ./your_binary_executable -is:javahome path_to_your_jdk

  7. 설치 위치가 정확하며 설치에 필요한 충분한 공간이 시스템에 있는지 확인합니다.

  8. 다음을 클릭하여 설치를 시작합니다.

주: 소프트웨어 설치를 성공적으로 완료하는 데 문제가 발생한 경우 아래의 문제 해결 절에서 자세한 설명과 설치 프로세스에 영향을 줄 수 있는 해결되지 않은 문제에 대한 해결 방법을 참조하십시오.

Macintosh OS X

Macintosh OS X에서 NetBeans는 고유 응용 프로그램 번들 패키지로 제공됩니다. NetBeans IDE 설치에 자주 사용되는 방법은 tarball(.tar.gz) 파일을 사용하는 것입니다.

NetBeans IDE 5.0을 설치하려면

IDE 시작

다음은 NetBeans IDE 시작에 관한 지침입니다. 설치 프로그램이 없는 운영 체제를 사용하는 경우에도 아카이브 배포를 다운로드하여 설치하면 NetBeans IDE를 사용할 수 있습니다. 자세한 내용은 아래의 아카이브 배포 절을 참조하십시오.

Windows

Microsoft Windows 시스템에서 IDE를 시작하려면 다음 중 하나를 수행합니다.

  • 데스크탑에서 NetBeans IDE 아이콘을 두 번 클릭합니다.

  • 시작 메뉴에서 NetBeans IDE 5.0 > NetBeans IDE를 선택합니다.

SolarisTM 운영 체제(Solaris OS)

Solaris 시스템에서 NetBeans IDE를 시작하려면

  1. 설치 위치의 bin 하위 디렉토리로 이동합니다.

  2. ./netbeans를 입력하여 시작 프로그램 스크립트를 실행합니다.

Linux

Linux 시스템에서 NetBeans IDE를 시작하려면

  1. 설치 위치의 bin 하위 디렉토리로 이동합니다.

  2. ./netbeans를 입력하여 시작 프로그램 스크립트를 실행합니다.

Macintosh OS X

Mac OS X 시스템에서 NetBeans IDE를 시작하려면

  • NetBeans IDE 응용 프로그램 아이콘을 두 번 클릭합니다.

위


응용 프로그램 서버 등록 및 시작

독립형 NetBeans IDE 5.0 설치 프로그램을 사용한 경우에는 응용 프로그램 서버를 사용하기 전에 등록해 두어야 합니다.

Sun Java System Application Server 등록

Sun Java System Application Server를 등록하려면

  1. 도구 > 서버 관리자로 이동합니다.

  2. 서버 관리자에서 서버 추가를 클릭합니다.

  3. 대화 상자에서 Sun Java System Application Server 8.2를 선택하고 다음을 클릭합니다.

  4. 서버 인스턴스의 이름을 입력합니다. 이 이름이 IDE에서 서버를 식별하는 데 사용됩니다.

  5. 위치, 사용자 이름 및 암호를 입력합니다. 서버의 원격 인스턴스를 등록하는 경우에는 설치 위치와 도메인을 입력할 필요가 없습니다. 원격 인스턴스를 등록하려면 먼저 로컬 설치된 응용 프로그램 서버를 등록해야 합니다.

  6. 마침을 클릭합니다.

주: 번들 설치 프로그램을 사용하는 경우에는 Sun Java System Application Server가 자동으로 설치 및 구성되기 때문에 여기에 설명된 등록 절차를 수행할 필요가 없습니다.


Sun Java System Application Server 시작

응용 프로그램 서버를 시작하려면

  1. 런타임 창에서 새로 생성된 localhost 노드를 마우스 오른쪽 버튼으로 클릭하고 컨텍스트 메뉴에서 서버 시작/중지를 선택합니다.

  2. 서버 상태 대화 상자에서 서버 시작을 클릭합니다.

  3. 닫기를 클릭하여 대화 상자를 닫습니다.

주: NetBeans IDE 및 Sun Java System Application Server 번들에서는 기본적으로 admin 사용자를 비밀번호 adminadmin으로 생성합니다. 사용자는 이 설정을 사용하여 웹 콘솔을 통해 로그인해야 합니다.


위


아카이브 배포

설치 프로그램이 없는 운영 체제를 사용하는 경우에도 아카이브 배포를 다운로드하여 설치하면 NetBeans IDE를 사용할 수 있습니다. 개발 작성 상태를 테스트하려는 경우에도 IDE의 아카이브 배포를 설치할 수 있습니다.

아카이브 NetBeans IDE 배포를 설치하려면

  1. 아카이브 파일이 포함된 디렉토리로 이동합니다.

  2. 시스템의 적절한 유틸리티를 사용하여 아카이브 압축을 해제합니다.

아카이브 NetBeans IDE 배포를 시작하려면

  1. NetBeans IDE 설치 위치의 bin 하위 디렉토리로 이동합니다.

  2. 시스템의 적절한 시작 프로그램을 실행합니다.

아카이브 배포에는 다음 시작 프로그램이 포함되어 있습니다.

  • netbean.exe - Windows
  • nb.exe - Windows(콘솔 창에서)
  • netbeans - UNIX, Linux
  • NetBeans - Mac OS X
  • netbeans.cmd - OS/2

운영 체제에 맞는 시작 프로그램이 없는 경우에는 적절한 스크립트를 수정해서 생성해야 할 수도 있습니다. 시스템에서 JDK 1.4.2 이상이 지원되는 경우에는 IDE를 실행할 수 있습니다.

주: 시작 프로그램을 생성한 경우 해당 프로그램을 NetBeans 프로젝트에 제공할 수 있습니다. 자세한 내용은 netbeans.org에 제공을 참조하십시오.


위


제거

필요한 경우 번들 소프트웨어를 제거할 수 있지만 NetBeans IDE와 Sun Java System Application Server는 별도로 제거해야 합니다.

Windows에서 NetBeans IDE 5.0 및 Sun Java System Application Server Platform Edition 8.2 번들 제거

NetBeans IDE 5.0을 제거하려면

  1. 시스템의 시작 메뉴에서 제어판을 선택합니다.

  2. 프로그램 추가/제거 아이콘을 두 번 클릭합니다.

  3. 목록에서 NetBeans를 선택하고 제거를 클릭합니다.

Sun Java System Application Server Platform Edition 8.2를 제거하려면

  1. 시스템의 시작 메뉴에서 제어판을 엽니다.

  2. 프로그램 추가/제거 아이콘을 두 번 클릭합니다.

  3. 목록에서 Sun Java System Application Server Platform Edition 8.2를 선택하고 제거를 클릭합니다.

Solaris(SPARC 및 x86 Platform Edition)에서 NetBeans IDE 5.0 및 Sun Java System Application Server Platform Edition 8.2 번들 제거

NetBeans IDE 5.0을 제거하려면

  1. NetBeans IDE 설치 디렉토리의 _uninst 하위 디렉토리로 이동합니다.

  2. ./uninstaller를 입력하여 InstallShield 설치 제거 프로그램을 실행합니다.

Sun Java System Application Server Platform Edition 8.2를 제거하려면

  1. Sun Java System Application Server Platform Edition 8.2 설치 디렉토리로 이동합니다.

  2. 터미널 창에서 ./uninstall을 입력하여 설치 제거 프로그램을 실행합니다.

Linux에서 NetBeans IDE 5.0 및 Sun Java System Application Server Platform Edition 8.2 번들 제거

NetBeans IDE 5.0을 제거하려면

  1. NetBeans IDE 설치 디렉토리의 _uninst 하위 디렉토리로 이동합니다.

  2. 터미널 창에서 ./uninstaller를 입력하여 InstallShield 설치 제거 프로그램을 실행합니다.

Sun Java System Application Server Platform Edition 8.2를 제거하려면

  1. Sun Java System Application Server Platform Edition 8.2 설치 디렉토리로 이동합니다.

  2. 터미널 창에서 ./uninstall을 입력하여 설치 제거 프로그램을 실행합니다.

Macintosh OS X에서 NetBeans IDE 5.0 및 Sun Java System Application Server Platform Edition 8.2 번들 제거

NetBeans IDE 5.0을 제거하려면

  1. NetBeans IDE의 설치 위치로 이동합니다. 기본적으로 NetBeans는 Applications 디렉토리에 설치됩니다.

  2. NetBeans 아이콘을 마우스 오른쪽 버튼으로 클릭하고 팝업 메뉴에서 Show Package Contents를 선택합니다.

  3. 표시되는 창에서 _uninst 하위 디렉토리로 이동합니다.

  4. 터미널 창에서 ./uninstaller를 입력하여 InstallShield 설치 제거 프로그램을 실행합니다.

Sun Java System Application Server Platform Edition 8.2를 제거하려면

  1. Sun Java System Application Server Platform Edition 8.2 설치 디렉토리로 이동합니다. 기본적으로 응용 프로그램 서버는 홈 디렉토리의 SUNWappserver 하위 디렉토리에 설치됩니다.

  2. 터미널 창에서 ./uninstall을 입력하여 설치 제거 프로그램을 실행합니다.

문제 해결

소프트웨어 설치 또는 실행 중에 문제가 발생한 경우에는 아래의 문제 목록에서 해결 방법을 찾아 보십시오. 알려진 문제의 전체 목록, 시스템 요구 사항, 기타 유용한 정보를 보려면 릴리스 노트를 참조하십시오.

다음은 이 릴리스에서 해결되지 않았으며 설치 프로세스에 영향을 주거나 프로세스 중단을 일으킬 수 있는 문제입니다.

  • 일반 문제

    설명:

    Sun Java System Application Server Platform Edition 8.2에는 JDK 6.0 Pre-release 버전과 관련된 심각한 문제 몇 가지가 있기 때문에 NetBeans IDE 및 Sun Java System Application Server 번들 설치 프로그램을 JDK 6.0에 설치할 수 없습니다. 하지만 NetBeans IDE 독립형 설치 프로그램은 JDK 6.0에 설치할 수 있습니다.


  • 문제 #72601

    설명:

    Sun Java System Application Server의 위치를 지정하고 다음을 클릭한 후에 마법사 페이지에 아무 것도 표시되지 않고 설치를 계속할 수도 없습니다.

    해결 방법:

    마법사에서 취소를 누릅니다. 설치 프로그램에서 방금 생성된 시스템 상의 NetBeans IDE 설치 디렉토리로 이동한 후 해당 디렉토리를 삭제합니다. 그리고 설치 프로그램을 다시 실행한 다음 NetBeans IDE 설치 디렉토리 안에 있지 않은 응용 프로그램 서버 설치 디렉토리 위치를 지정합니다.


  • 문제 #26965

    설명:

    호환되지 않는 버전의 sax.jar가 포함된 JDK 설치를 사용하면 다음과 같은 메시지가 표시되며 IDE 설치 프로그램이 중단됩니다. "다음의 오류로 인해 마법사를 계속 진행할 수 없습니다.적절한 항목 목록에서 IDE가 사용할 JDK를 선택한 후 다음을 클릭합니다. /wizard.inf 에서 지정한 마법사를 로드할 수 없습니다.(104)"

    해결 방법:

    IDE를 설치하는 동안 jre/lib/endorsed/sax.jar을 JDK 디렉토리에서 일시적으로 제거합니다.


  • 문제 #52856(Windows XP만 해당)

    설명:

    Sun Java System Application Server 설치가 수행되지 않습니다. 이 문제는 NetBeans IDE 5.0 및 Sun Java System Application Server Platform Edition 8.2 번들 설치 프로그램을 사용하는 경우와 독립형 응용 프로그램 서버 설치 프로그램을 사용하는 경우 모두에 발생하는 것으로 확인되었습니다. 이 문제는 이전에 응용 프로그램 서버 설치가 제대로 수행되지 않았고 그 후에 제거에 성공하지 못한(즉 레지스트리 항목 또는 구성 파일 중 일부가 삭제되지 않은) 경우에 발생합니다.

    해결 방법:

    이 문제가 발생하면 이전에 설치된 Sun Java System Application Server를 모두 제거합니다. 그리고 설치 프로그램의 <USERHOME>\Local Settings\temp\ 디렉토리와 응용 프로그램 서버의 \Application Data\Sun\tmp\ 디렉토리를 삭제합니다. <WINDIR>\Sun\에도 파일이 있을 수 있습니다. 이전에 응용 프로그램 서버가 제대로 제거되지 않은 경우에는 <SYSTEMDIR>/productregistry 파일도 삭제합니다. 다음으로는 소프트웨어를 설치할 새 디렉토리를 생성한 후 디렉토리가 비어 있으며 쓰기 가능한지 확인합니다. 마지막으로 시스템의 새 위치에 응용 프로그램 서버를 다시 설치합니다.


  • 문제 #65482(Linux만 해당)

    설명:

    일부 Mandrake/Mandriva Linux 배포에서 NetBeans를 설치할 수 없습니다. 설치 중에 설치 프로그램에서 다음 메시지를 표시합니다. 

    Initializing InstallShield Wizard........
    ./netbeans-5_0-bin-linux.bin: line 189: bc: command not found
    ./netbeans-5_0-bin-linux.bin: line 404: [: : integer expression expected

    해결 방법:

    NetBeans 설치 프로그램에서는 GNU의 bc 유틸리티를 사용하는데 Mandriva에서는 이 유틸리티가 기본적으로 설치되지 않습니다. 이 문제가 발생한 경우에는 NetBeans를 설치하기 전에 수동으로 GNU의 bc 유틸리티를 설치하십시오. 활성 상태인 인터넷 연결이 있는 경우에는 명령줄에서 urpmi bc를 실행하면 됩니다. rpm -i <bc_package>.rpm 명령을 사용하여 적절한 RPM을 다운로드하고 설치할 수도 있습니다.

  • 문제 #67530

    설명:

    설치 프로그램을 시작한 후에 "Welcome to the Install Wizard for ERROR: null"이라는 메시지가 표시됩니다. 여러 설치 프로그램을 동시에 실행하거나 다른 프로그램이 종료되지 않은 상태에서 설치 프로그램을 실행한 경우에 이 문제가 발생할 수 있습니다.

    해결 방법:

    \InstallShield\Universal\Common 디렉토리의 내용을 삭제한 후 설치 프로그램을 다시 시작합니다.

    Windows 시스템에서 이 디렉토리는 C:\Program Files\Common files\InstallShield\Universal\Common에 있습니다.

    Solaris 및 Linux 시스템에서 이 디렉토리는 /$HOME/InstallShield/Universal/Common에 있습니다.

  • 문제 #52012(Windows XP 시스템 터키어 로켈만 해당)

    설명:

    터키어 로켈을 사용하도록 설정된 시스템에서 설치 프로그램을 성공적으로 마칠 수 없습니다.

    해결 방법:

    설치 프로그램을 취소합니다. 시스템에서 영어(미국) 로켈로 전환한 후에 설치 프로그램을 다시 시작합니다.

    Windows XP 시스템에서는 시작 | 제어판을 선택한 후에 국가 및 언어 옵션을 두 번 클릭하면 시스템 로켈을 전환할 수 있습니다. 그 다음에 대화 상자의 표준 및 서식 섹션에 있는 드롭다운 목록에서 로켈을 선택합니다.
