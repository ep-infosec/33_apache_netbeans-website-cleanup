Следующие инструкции относятся к установке как среды NetBeans 5.0 (отдельная программа установки), так и набора среды NetBeans 5.0 и сервера приложений Sun Java Application Server Platform Edition 8.2.

Установка среды NetBeans (отдельная программа установки)

Ниже следуют инструкции по установке интегрированной среды разработки NetBeans на всех поддерживаемых платформах.

Windows

На компьютеры с ОС Microsoft Windows интегрированную среду NetBeans рекомендуется устанавливать с помощью самораспаковывающейся программы установки.

Для установки интегрированной среды NetBeans 5.0:

  1. После загрузки программы установки дважды щёлкните по значку программы установки для её запуска .

  2. В мастере установки прочитайте лицензионное соглашение, и если вы его принимаете, нажмите Далее.

  3. Укажите пустую папку, в которую вы хотите установить среду NetBeans.

    Примечание: Эта среда NetBeans не нарушит настроек других установок NetBeans, т.к. интегрированная среда автоматически создаёт новый каталог настроек пользователя при запуске в ${HOME}/.netbeans/5.0.


  4. Выберите набор разработчика Java (JDK), который будет использовать интегрированная среда, из списка и нажмите Далее.

  5. Убедитесь, что папка установки указана верно, и что у вас хватает свободного места на диске для установки.

  6. Нажмите Далее чтобы начать установку.

Примечание: Если вы столкнётесь с ошибками при установке, перейдите к разделу Устранение ошибок, в котором дано полное описание и предложены способы решения ошибок установки.


ОС SolarisTM

На машины с ОС Solaris интегрированную среду NetBeans рекомендуется устанавливать с помощью бинарного установщика.

Для установки интегрированной среды NetBeans 5.0:

  1. Перейдите к каталогу с программой становки.

  2. Если необходимо, измените права доступа файла установщика, чтобы сделать его выполнимым. Для этого в командной строке введите:
    $ chmod +x ваш_бинарный_установщик

    Примечание: Замените ваш_бинарный_установщик на имя загруженного вами бинарного файла.


  3. Запустите установку введя в командной строке:
    $ ./ваш_бинарный_установщик

    Примечание: Здесь тоже не забудьте заменить ваш_бинарный_установщик на имя загруженного вами бинарного файла.


  4. В мастере установки прочитайте лицензионное соглашение, и если вы его принимаете, нажмите Далее.

  5. Укажите пустую папку, в которую вы хотите установить среду NetBeans, и нажмите Далее.

  6. Выберите набор разработчика Java (JDK), который будет использовать интегрированная среда, из списка и нажмите Далее.

    Программа установки находит установленные в вашей системе JDK и запрашивает, какую она должна использовать. Вы также можете указать используемую JDK из командной строки. Например:

    $ ./ваш_бинарный_установщик -is:javahome путь_к_вашей_jdk

  7. Убедитесь, что папка установки указана верно, и что у вас хватает свободного места на диске для установки.

  8. Нажмите Далее чтобы начать установку.

Примечание: Если вы столкнётесь с ошибками при установке, перейдите к разделу Устранение ошибок, в котором дано полное описание и предложены способы решения ошибок установки.

Для установки набора из интегрированной среды NetBeans 5.0 и сервера приложений Sun Java System 8.2:

  1. Перейдите к каталогу с программой становки.

Запуск интегрированной среды

Ниже следуют инструкции по запуску интегрированной среды разработки NetBeans. Если вы используете операционную систему, для которой нет программы установки, вы можете загрузить и установить среду из архива. Дополнительная информация находится в разделе Установка из архива ниже.

Windows

Для запуска интегрированной среды на машинах с Microsoft Windows сделайте одно из нижеперечисленного:

  • Дважды щелкните на значке NetBeans 5.0 на вашем рабочем столе.

  • В меню Пуск выберите Все программы > NetBeans 5.0 > NetBeans IDE.

ОС SolarisTM

Для запуска среды разработки NetBeans на системах Solaris:

  1. Перейдите к подпапке bin вашей установки.

  2. Запустите скрипт запуска введя ./netbeans.

Linux

Для запуска среды разработки NetBeans на системах Linux:

  1. Перейдите к подпапке bin вашей установки.

  2. Запустите скрипт запуска введя ./netbeans.

Mac OS X

Для запуска среды разработки NetBeans на системах Mac OS X:

  • Дважды щелкните на значке NetBeans 5.0.

Регистрация и запуск сервера приложений

Если вы установили среду NetBeans 5.0 отдельно, необходимо зарегистрировать сервер приложений перед его использованием.

Регистрация сервера приложений Sun Java System

Для того чтобы зарегистрировать сервер приложений Sun Java System:

  1. Вызовите Сервис > Менеджер серверов.

  2. В менеджере серверов нажмите Добавить сервер.

  3. В появившемся диалоге выберите сервер приложений Sun Java System 8.2 и нажмите Далее.

  4. Введите имя для экземпляра сервера. Это имя используется для идентификации сервера в интегрированной среде.

  5. Введите расположение, имя пользователя и пароль. Вам не нужно вводить расположение установки и домен если вы регистрируете удалённый экземпляр сервера. Вы должны зарегистрировать хотя бы один локальный экземпляр сервера приложений Sun Java System перед тем, как вы сможете регистрировать удалённые экземпляры.

  6. Нажмите Завершить.

Примечание: Если вы установили пакет среды NetBeans 5.0 и сервера приложений Sun Java System, локальный сервер приложений был установлен и зарегистрирован автоматически, поэтому описанная выше процедура регистрации не нужна.


Запуск сервера приложений Sun Java System

Для запуск сервера приложений:

  1. В окне Окружение щёлкните правой кнопкой мыши по недавно созданному узлу сервера приложений (он будет иметь указанное вами имя) и выберите команду Запустить в контекстном меню.

Примечание: Набор из среды NetBeans и сервера приложений Sun Java System создаёт пользователя admin с паролем adminadmin. Пользователи должны использовать эти настройки для входа через веб-консоль.

Установка из архива

Если вы используете операционную систему, для которой нет программы установки, вы можете загрузить и установить среду из архива. Вы также можете установить среду из архива если хотите потестировать текущие сборки.

Для установки интегрированной среды NetBeans из архива:

  1. Передите к папке с архивом.

  2. Распакуйте архив с помощью соответствующих утилит вашей системы.

Для запуска установленной из архива среды NetBeans:

  1. Перейдите к подпапке bin вашей установки среды NetBeans.

  2. Выполните соответствующий вашей системе загрузчик

В архивы включены такие загрузчики:

  • netbeans.exe - Windows
  • nb.exe - Windows (из консольного окна)
  • netbeans - UNIX, Linux
  • NetBeans - Mac OS X
  • netbeans.cmd - OS/2

Если для вашей системы нет загрузчика, вам придётся создать его, изменив подходящий скрипт. Если ваша машина поддерживает JDK 1.4.2 или выше, то указанного набора загрузчиков должно хватить, т.к. для других платформ набора разработчика JDK нужных версий не существует.

Примечание: Если вы создадите загрузчик, можете предложить его проекту NetBeans. Дополнительная информация находится на странице Вклад в netbeans.org.

Удаление

При необходимости вы можете удалить установленное программное обеспечение, однако вы должны удалять среду NetBeans и сервер приложений Sun Java System Application по отдельности.

Удаление набора среды NetBeans 5.0 и сервера приложений Sun Java System Platform Edition 8.2 в Windows

Для удаления интегрированной среды NetBeans:

  1. В меню Пуск выберите Панель управления.

  2. Дважды щелкните на значке Добавление/удаление программ.

  3. Выберите NetBeans из списка и нажмите Удалить.

Для удаления сервера приложений Sun Java System Platform Edition 8.2:

  1. В меню Пуск выберите Панель управления.

  2. Дважды щелкните на значке Добавление/удаление программ.

Устранение ошибок

Если у вас возникли проблемы при установке или выполнении программного обеспечения, просмотрите список проблем и прочитайте способы решения ошибок установки. Полный список извечтных проблем, системные требования и другая полезная информация находятся в Замечаниях о выпуске.

Ниже перечислены нерешённые на момент выхода данной версии проблемы, которые могут повлиять на работоспособность или нарушить процесс установки:

  • Общие проблемы

    Описание:

    Мастер установки набора среды NetBeans и сервера приложений Sun Java System не позволяет установить набор на JDK 6.0, т.к. сервер приложений Sun Java System Platform Edition 8.2 работает с ошибками на предварительных версиях JDK 6.0. Вы, однако, можете установить интегрированную среду NetBeans 5.0 на JDK 6.0 отдельно.


  • Проблема #72601

    Описание:

    После того, как вы указали расположение сервера приложений Sun Java System и нажали Далее, страница мастера становится пустой и продолжить установку невозможно.

    Способ решения:

    Нажмите Отмена в мастере. Перейдите к папке, которую вы указали как место для установки среды NetBeans, и которую только что создал мастер установки, и удалите её. Затем запустите мастер ещё раз и в этот раз укажите расположение сервера приложений вне папки установки среды NetBeans.


  • Проблема #26965

    Описание:

    При использовании пакетов разработчика JDK с несовместимой версией sax.jar мастер установки среды падает с сообщением "The wizard cannot continue because of the following error: could not load wizard specified in /wizard.inf (104)" (мастер не может продолжить из-за следующей ошибки: невозможно загрузить мастер из /wizard.inf (104))

    Способ решения:

    Временно удалите jre/lib/endorsed/sax.jar из папки установки JDK на время установки среды.


  • Проблема #52856 (только на Windows XP)

    Описание:

    Невозможно установить сервер приложений Sun Java System. Данная проблема относится как к установке набора из интегрированной среды NetBeans 5.0 и сервера приложений Sun Java System Platform Edition 8.2, так и к установке сервера приложений отдельно. Она возникает если предыдущая установка сервера приложений была завершена неудачно, и все попытки удаления также не прошли (к примеру, не были удалены некоторые значения реестра или файлы настройки).

    Способ решения:

    В данном случае удалите все предыдущие установки сервера приложений Sun Java System. Затем очистите папки <USERHOME>\Local Settings\Temp\ и <USERHOME>\Application Data\Sun\tmp\ (она также может находиться в <WINDIR>\Sun\). Если предыдущие попытки удаления закончились неудачно, также удалите файл <SYSTEMDIR>/productregistry. Затем создайте папку, в которую вы хотите установить сервер приложений, и убедитесь, что эта папка пуста и доступна на запись. И наконец, переустановите сервер приложений в новое место.


  • Проблема #65482 (только в Linux)

    Описание:

    Невозможно установить NetBeans на некоторых дистрибутивах Mandrake/Mandriva Linux. Во время инициализации мастер установки выводит следующее сообщение:

    Initializing InstallShield Wizard........
    ./netbeans-5_0-bin-linux.bin: line 189: bc: command not found
    ./netbeans-5_0-bin-linux.bin: line 404: [: : integer expression expected

    Способ решения:

    Мастер установки NetBeans использует утилиту GNU bc, которая, похоже, по умолчанию в Mandriva не ставится. Если у вас возникла данная продлема, установите утилиту GNU bc до установки NetBeans. Если у вас есть соединение с Интернет, вы можете для этого запустить urpmi bc из командной строки. Вы также можете загрузить соответствующий пакет RPM и установить его с помощтю команды rpm -i <пакет_bc>.rpm.

  • Проблема #67530

    Описание:

    После запуска мастера установки вы видите сообщение "Добро пожаловать в мастер установки интегрированной среды ERROR: null" (или "Welcome to the Install Wizard for ERROR: null" в нелокализованной версии). Это может быть вызвано запуском нескольких программ установки одновлеменно или запуском мастера до того, как предыдущий мастер завершил свою работу.

    Способ решения:

    Удалите содержимое папки \InstallShield\Universal\Common и перезапустите программу установки.

    На системах Windows она находится в C:\Program Files\Common files\InstallShield\Universal\Common.

    В системах Solaris и Linux эта папка находится в /$HOME/InstallShield/Universal/Common.

  • Проблема #52012 (только турецкая локаль только на системах Windows XP)

    Описание:

    Мастер установки завершается с ошибкой на системах, использующих турецкую локаль.

    Способ решения:

    Отмените установку. На вашей системе переключитесь на локаль English (United States) и перезапустите программу установки.

    На системах Windows XP переключение локали осуществляется в Панели управления с помощью апплета Язык и Региональные стандарты. Затем выберите локаль из выпадающего списка в разделе Языковые стандарты и форматы.
