Introduction

Bienvenue à l'EDI NetBeansTM 5.0. L'EDI NetBeans est un Environnemt de Développement Intégré (EDI, ou IDE en anglais) modulaire, et conçu sur les standards, écrit en langage de programmation JavaTM. Le projet NetBeans est constitué de:

  • Un EDI open source écrit dans le langage de programmation Java.
  • Une plateforme d'application, qui peut être utilisée comme framework générique pour concevoir tous les types d'applications.

Pour avoir plus d'informations concernant les nouvelles fonctionnalités de développement dans l'EDI NetBeans, allez voir le document Quoi de Neuf dans l'EDI NetBeans 5.0.

Si vous recherchez des informations quant à l'installation de l'EDI, voyez les Instructions d'Installation de NetBeans.

Recommandation Système

Systèmes d'Exploitation Supportés

L'EDI NetBeans tourne sour les systèmes d'exploitation qui supporte la JavaTM VM. Ci-dessous, une liste des plateformens sur lesquelles l'EDI NetBeans a été testé.

  • Microsoft Windows XP Professionel SP2
  • Système d'exploitation Solaris (SPARC® et x86 Platform Editions) versions 8, 9, et 10
  • Red Hat Fedora Core 3
  • Mac OS X 10.4

L'EDI NetBeans est également connu pour fonctionner sur les plateformes suivantes:

  • Java Desktop System 2
  • Microsoft Windows 2000 Professionel SP3
  • De Nombreuses autres distributions Linux
  • Open VMS 7.2-1 ou plus
  • D'autres plateforms UNIX®, comme HP-UX

    Note: le Sun Java System Application Server peut ne pas tourner sur certaines plateformes.

Logiciels Requis

L'EDI NetBeans tourne sour le J2SE JDK 5.0 (JavaTM 2 JDK, Standard Edition), qui est constitué du Java Runtime Environment plus des outils de développement pour compiler, déboguer, et exécuter des applications écrites dans le langage JavaTM. L'EDI NetBeans 5.0 tourne également sur la version 1.4.2 du J2SE SDK, mais il n'a été testé que sous le JDK 5.0.

Note: Si vous désirez exécuter l'EDI sur une version précédente du SDK, ou si vous préférez installer l'EDI manuellement, vous pouvez télécharger les versions archivées de l'EDI sur http://java.sun.com/products/archive/index.html.

Vous pouvez télécharger le JDK pour votre plateforme depuis l'un des sites repris ci-dessous:

  • Solaris: http://java.sun.com/j2se/1.5.0/
  • Windows: http://java.sun.com/j2se/1.5.0/
  • Linux: http://java.sun.com/j2se/1.5.0/
  • Mac OS X: http://www.apple.com/support/downloads/javaupdate142.html (Mac OS X 10.3.4 or later). Les mises à jour Mac OS sont également disponibles depuis le mécanisme de mise à jour de logiciels d'Apple.
  • Open VMS: http://h18012.www1.hp.com/java/download/index.html
  • OS/2: http://www.goldencode.com/company/software/j2se/index.html ou http://www.innotek.de/products/javaos2/javaos2general_e.html.
    Si vous obtenez un NullPointerException lors du démarrage de l'EDI sur OS/2, essayez de changer le nom de votre répertoire JAVA_HOME/jre en lettres capitales.

Compatibilité

Lorsque vous exécutez pour la première fois l'EDI NetBeans 5.0, vous pouvez importer un sous-ensemble des paramètres de configurations utilisés dans l'EDI NetBeans 4.1. Si vous ne choisissez pas d'importer les paramètres depuis une version précédente, l'EDI s'ouvre en utilisant les paramètres par défaut. Les paramètres spécifiques aux projets ne sont pas importable depuis la version 3.6.
Note: L'EDI ne reconnait que les installations précédentes où le répertoire utilisateur réside dans l'emplacement par défaut. Il ne reconnait pas les installations où vous avez spécifié un répertoire utilisateur en utilisant le paramètre de la ligne de commande --userdir. Si vous désirez importer des paramètres depuis un EDI que l'installateur ne peut reconnaitre, téléchargez une version archive de l'EDI au lieu de l'installateur.

Bien qu'il soit possible d'importer les paramètres depuis une installation précédente de l'EDI, il n'est pas possible d'importer les paramètres depuis l'EDI NetBeans 5.0 dans une version précédent de l'EDI.

Note:Si vous éditez la fenêtre Project Properties pour un projet qui fut créé dans l'EDI NetBeans 4.1 ou précédent, le projet ne fonctionnera plus dans les versions précédentes de l'EDI.

Problèmes Connus


Note: L'offre conjointe de l'EDI NetBeans et Sun Java System Application Server crée l'utilisateur admin et le mot de pase adminadmin par défaut. Les utilisateurs doivent utiliser ces paramètres pour se loguer via la console web..


Les problèmes non-résolus pour cette version sont repris ci-dessous:

  • Problème #26965

    Description :

    Lorsqu'on utilise les installations de JDK qui incluent une version de sax.jarincompatible, l'installateur de l'EDI se plante avec le message suivant: "The wizard cannot continue because of the following error: could not load wizard specified in /wizard.inf (104)"

    Solution au Problème:

    Supprimez temporairement le fichier jre/lib/endorsed/sax.jar du répertoire du JDK durant l'installation de l'EDI.


  • Problème #32463 (Macintosh OS X uniquement)

    Description :

    le comportement du clic-droit de la souris ne s'émule pas sur plusieurs fichiers.

    Solution au Problème:

    Maintenez enfoncée la touche Control tout en sélectionnant plusieurs fichiers à la fois. Attention que le menu contextuel va apparaitre après chaque clic de souris et pourrait rendre impossible des sélections successives.


  • Problème #47645

    Description :

    Ne peut pas tourner une cible avec une tâche qui a failonerror=true depuis l'EDI bien que le même script fonctionne en dehors de l'EDI sans aucun problème.

    Solution au Problème:

    Si cela se produit, il est recommendé d'exécuter le script depuis la ligne de commande. Une autre possibilité est de réécrire le script.


  • Problème #48288

    Description :

    Les champs qui sont générés automatiquement par l'Éditeur Visuel et qui se trouvent dans les blocs protégés (stoqués dans les fichiers .form) ne sont pas modifiés lors d'un refactoring.

    Solution au Problème:

    Si cela se produit, renommer premièrement les champs du GUI en cliquant-droit sur le noeud correspondant dans la fenêtre Inspector et choisissez Rename. Ensuite, renommez les occurences en dehors des blocs protégés en cliquant droite sur le noeud qui représente le champ et en choisissant Refactoring > Rename.


  • Problème #51880 (Windows XP)

    Description :

    Les JSP des projets free-form ne peuvent être débogués lors de l'attachement du débogueur à un processus.

    Solution au Problème:

    Utilisez une target ant debug comme décrit dans Écriture d'une Target Debug pour un Projet Web Free-Form à la place.


  • Problème #52856 (Windows XP uniquement)

    Description :

    L'installation de Sun Java System Application Server échoue. Notez que ce problème se pose aussi bien pour l'offre groupée de l'EDI NetBeans 5.0 et Sun Java System Application Server Platform Edition 8 Bundle ainsi que pour l'installation du Serveur d'Application seule. Cela se produit lorsqu'une installation précédente du Serveur d'Application a échoué et des tentatives successives de désinstallation échouaient également(en fait, certaines entrées de la base de registre ou des fichiers de configuration n'ont pas été supprimés).

    Solution au Problème:

    Si cela se produit, effacez toutes les installations précédentes de Sun Java System Application Server. Ensuite, supprimer le répertoire <USERHOME>\Local Settings\temp\ de l'installateur et le répertoire \Application Data\Sun\tmp\ du Serveur d'Application (notez que les fichiers peuvent également se situer dans <WINDIR>\Sun\). Si une tentative précédent de désinstallation du Serveur d'Application avait échouée, supprimez également le fichier <SYSTEMDIR>/productregistry. Ensuite, créez un nouveau répertoire dans lequel installer le logiciel et assurez-vous que le répertoire est bien vide, et accessible en écriture. Finallement, réinstaller le Serveur d'Application dans le nouvel emplacement de votre système.


  • Problème #54965 (Windows XP uniquement)

    Description :

    When the WSDL operation qualifies for wrapper style and the output part's element has more than one sub element, the output element does not unwrap. This causes a conflict with JSR-109.

    Solution au Problème:

    Si cela se produit, cliquez-droit sur le noeud du projet et choisissez Properties. Sur le panneau Web Service Clients de la fenêtre Project Properties, sélectionnez "donotunwrap" et désactivez "strict".


  • Problème #56885

    Description :

    Le débogueur ne s'arrête pas au début du Web Service après avoir pressé F7

    Solution au Problème:

    Il n'y a actuellemnt aucune solution connue à ce problème.


  • Problème #56910

    Description :

    Sun Java System Application Server is restarted in some cases after deployment. This occurs when a module that is deployed to the application server has errors in its sources, such as when a remote method does not throw a RemoteException.

    Solution au Problème:

    If this occurs, run the integrated J2EE verifier to obtain information about why the archive can not be loaded and then edit the source file to correct any errors.


  • Problème #57282 (Linux uniquement )

    Description :

    Sun Java System Application Server is not installed on 64bit JDK on Linux/AMD64. The Application Server installer but does not run on 64bit JDK and therefore the Application Server cannot be installed using a 64bit JDK. This does not mean, however, that the Application Server itself will not RUN on 64bit JDK.

    Solution au Problème:

    If this occurs, run the Application Server bundle installer on a 32bit JDK. After installation you should be able to run both the IDE and Application Server on a 64bit JDK.


  • Problème #57697 (Windows XP uniquement)

    Description :

    Sun Java System Application Server installation fails because the bundle installer cannot find the JDK.

    Solution au Problème:

    If this happens, start the console application (cmd.exe), navigate to the directory where the installer is located, and type the installer name plus the -is:javahome <JAVA HOME DIR> switch. Create a new directory within which to install the software and ensure that the directory is empty and writable. If necessary, you can also use the -is:log <log file> parameter to generate an installer log.

    Notes:

    • If a previous uninstallation attempt of the Application Server was not successful, delete the <SYSTEMDIR>/productregistry file.
    • If one of the values specified in the silent installer statefile is invalid (i.e. it has a null value, zero length value, or its port is in use), add the appropriate values to the statefile.
    • Voir Problème #52856 ci-dessus pour des informations supplémentaires.

  • Problème #60235 (Mac OSX uniquement )

    Description :

    Editor focus occasionally lost when switching tabs in Mac OSX.

    Solution au Problème:

    If this happens, open any other window using the Window menu. If this does not remedy the problem, restart the IDE.


  • Problème #63084 (Linux uniquement)

    Description :

    NetBeans freezes while debugging an application on the Weblogic9 server started with the Bea JDK (JRockKit).

    Solution au Problème:

    If you experience this behavior, start Weblogic with the Sun JDK that is bundled with WebLogic. To do this, set the system property JAVA_VENDOR to value Sun. Note that the performance of the server will also benefit from this adjustment.


  • Problème #64771 (Linux uniquement)

    Description :

    When starting NetBeans with the SELinux (Security-enhanced) Linux option enabled, startup fails.

    Solution au Problème:

    If this happens, disable SELinux temporarily by typing #setenforce 0 from the command line. Note that this requires root priviledges. Then add selinux=0 to your kernel command line to disable SELinux at boot up.


  • Problème #64840 (Linux uniquement )

    Description :

    Network communication broken on Linux with IPv6 enabled, preventing the IDE from connecting to bundled servers, Auto Update, etc.

    Solution au Problème:

    If this happens, run the IDE using the -J-Djava.net.preferIPv4Stack=true' switch. Alternatively, disable IPv6 (in /etc/modules.conf or /etc/modprobe.conf).

    Pour plus d'informations, allez voir : Problème #5087907.


  • Problème #70779

    Description :

    Exemples d'application Web disponible uniquement avec le jeu de caractères Anglais.

    Solution au Problème:

    Si cela se produit, forcez le caractère d'encodage dans le fichier sur UTF-8 en rajoutant la ligne suivante:

    <%@page pageEncoding="UTF-8"%>
