Instructions d'installation de NetBeans IDE 6.9

Dernière mise à jour : 8 juin 2010

Ce document décrit les procédures d'installation de NetBeans IDE 6.8 sur votre système. Veuillez consulter les notes de version de NetBeans IDE 6.9 pour obtenir de plus amples informations sur les systèmes d''exploitation et les configurations matérielles pris en charge par l''environnement de développement intégré (IDE) Pour en savoir plus sur les nouvelles fonctionnalités incluses dans cette version de l''EDI, reportez-vous à la page Information de version pour NetBeans IDE 6.9

Logiciels requis

La version Java SE Development Kit (JDK) 6.0 Update 13 ou version ultérieure est requis pour installer l'EDI NetBeans. La version 6.9 de l'IDE ne peut pas être installée avec JDK 5.0. Vous devez télécharger la dernière version du JDK 6.0 sur http://java.sun.com/javase/downloads

Les kits PHP, Ruby et C/C++ de Netbeans seuls nécessitent l''installation du runtime Java JRE 6. Cependant, si vous voulez utiliser n'importe quelle fonctionnalité de Java, vous devrez avoir la version JDK 6.0 installée.

Options de téléchargement du programme d'installation

Sur la page de téléchargement de NetBeans IDE, vous pouvez choisir entre plusieurs programmes d'installation disponibles, chacun contenant l'EDI de base et des outils supplémentaires.

Les programmes d'installation suivants sont disponibles :

  • Java SE. Intègre toutes les fonctions standards de développement Java SE comme support pour la plateforme de développement Netbeans RCP.
  • JavaFX. Prend en charge le développement d'applications riches multiplate-forme JavaFX.
  • Java. Fournit aussi bien les outils pour le développement d''applications Java SE, Java EE et Java ME que le support de la plateforme de développement Netbeans RCP. Cette option de téléchargement contient GlassFish Server Open Source Edition 3.0.1 et Apache Tomcat 6.0.26.
  • Ruby. Fournit des outils de développement Ruby et prend en charge Rails et JRuby. Il contient aussi GlassFish Server Open Source Edition 3.0.1
  • C/C++. Prend en charge le développement dans les langages C, C++, Qt, Fortran et Assembly.
  • PHP. Fournit des outils de développement PHP 5.x, Symfony et Zend Framework.
  • Tout. Cette option de téléchargement est la plus complète. Elle contient toutes les exécutions et tous les packages disponibles pour NetBeans IDE.

Personnalisation de votre installation

Si vous téléchargez Java ou utilisez l'option de téléchargement "Tout", vous pouvez choisir exactement quels outils et exécutions installer. Cliquez sur Personnaliser sur la page d'accueil du programme d'installation et sélectionnez les fonctionnalités que vous souhaitez utiliser. Pour plus d'informations, reportez-vous à la section Installation du logiciel.

Si vous souhaitez ajouter des fonctionnalités à la configuration de NetBeans IDE, utilisez le Gestionnaire de plug-ins NetBeans. Par exemple, si vous commencez par télécharger et installer l'élément Ruby, puis décidez d'essayer la fonctionnalité C/C++ : ouvrez le Gestionnaire de plug-ins NetBeans dans l'EDI (choisissez Outils > Plug-ins) et ajoutez l'élément C/C++ à votre installation existante.

Il est important de noter les éléments suivants relatifs à l'installation et la configuration de certaines fonctionnalités NetBeans IDE :

  1. Apache Tomcat 6.0.26 est inclus dans les options de téléchargement "Java" et "Tout" mais n'est pas installé par défaut à partir de ces options. Pour installer Apache Tomcat à partir de l'option de téléchargement "Java" ou "Tout", lancer l'installeur et sélectionner Apache Tomcat 6.0.26 dans la boîte de dialogue de personnalisation d'installation.
  2. Pour le développement C/C++ (applicable pour les options de téléchargement C/C++ et Tout), vous devez en outre installer des compilateurs et des outils. Pour plus d'informations, reportez-vous à la section Configuring the NetBeans IDE for C/C++/Fortran.
  3. Pour Mac OS X, vous pouvez personnaliser l'installation de NetBeans IDE à partir des options de téléchargement Java, Ruby et Tout. L'installation de l'EDI à partir des options de téléchargement JavaSE, PHP et C/C++ ne peut pas être personnalisée.
  4. Vous pouvez aussi télécharger un fichier zip de la plateforme indépendante et exécuter le fichier exécutable de Netbeans IDE. Notez que GlassFish Server Open Source Edition et Apache Tomcat sont seulement disponibles avec les installeurs spécifiques à la plateforme.

Lancement du téléchargement

  1. Aller à https://netbeans.org/downloads/
  2. Dans la partie supérieure droite de la page, sélectionnez la langue et la plate-forme dans la liste déroulante. Vous pouvez également choisir de télécharger et d'utiliser le fichier zip indépendant de la plate-forme.
  3. Cliquez sur le bouton de téléchargement correspondant à l'option que vous souhaitez installer.
  4. Enregistrez le fichier du programme d'installation sur votre système.

Installation du logiciel

Suivez les instructions de cette section pour installer l'EDI sur votre système. Ces instructions d'installation s'appliquent à toutes les plates-formes prises en charge. Pour consulter la liste des plates-formes prises en charge, reportez-vous aux Notes de version.

Microsoft Windows, SE Solaris et Linux

Pour installer le logiciel :

  1. Une fois le téléchargement terminé, exécutez le programme d'installation.
    • Sous Windows, le fichier exécutable du programme d'installation porte l'extension .exe. Double-cliquez sur le fichier du programme d'installation pour l'exécuter.
    • Pour les plates-formes Solaris et Linux, le fichier du programme d'installation porte l'extension .sh. Pour ces plates-formes, vous devez rendre les fichiers du programme d'installation exécutables à l'aide de la commande suivante : chmod +x <nom-fichier-programme-installation>
  2. Pour choisir les outils et exécutions à installer, exécutez les étapes suivantes à partir de la page d'accueil de l'assistant d'installation :
    1. Cliquez sur Personnaliser.
    2. Dans la boîte de dialogue Personnaliser l'installation, effectuez vos sélections.
    3. Cliquez sur OK.
  3. Sur la page d'accuiel de l'assistant d'installation, cliquez sur Suivant.
  4. A la page d''accord de licence, lire l'accord de licence, cliquer sur la case à cocher d'acceptation et cliquer sur Suivant.
  5. Sur la page d'installation de NetBeans IDE, procédez comme suit :
    1. Acceptez le répertoire d'installation par défaut pour NetBeans IDE ou spécifiez un autre répertoire.

      Note: Le répertoire d''installation doit être vide et l''utilisateur doit avoir les droits de lecture/écriture pour ce répertoire.

    2. Acceptez le JDK par défaut à utiliser avec NetBeans IDE ou sélectionnez une autre installation dans la liste déroulante. Si l'assistant d'installation n'a pas trouvé d'installation JDK compatible à utiliser avec NetBeans IDE, votre JDK n'est pas installé dans l'emplacement par défaut. Dans ce cas, spécifier le chemin d'un JDK installé et cliquer Suivant ou annuler l'installation en cours. Après l'installation de la version JDK requise vous pouvez redémarrer l'installation.
  6. Si la page d''installation GlassFish Server Open Source Edition 3.0.1 s''ouvre, accepter le répertoire d''installation par défaut ou spécifier un autre emplacement.
  7. Si vous installez Apache Tomcat, acceptez le répertoire d'installation par défaut ou indiquez un autre emplacement d'installation sur la page d'installation.
  8. Sur la page récapitulative, vérifiez que la liste des composants à installer est correcte et que l'espace disponible sur votre système est suffisant pour l'installation.
  9. Cliquez sur Installer pour commencer l'installation.

Remarque : si vous rencontrez des problèmes lors de l'installation du logiciel, reportez-vous à la section Dépannage pour consulter les descriptions et solutions de contournement suggérées relatifs aux problèmes non résolus susceptibles d'affecter la procédure d'installation.

Mac OS X

Pour installer le logiciel :

  1. Une fois le téléchargement terminé, exécutez le programme d'installation. Le fichier du programme d'installation porte l'extension .dmg.
  2. Cliquez sur l'icône du package dans les fenêtres qui s'ouvrent. Le package porte l'extension .mpkg. L'assistant d'installation démarre.
  3. Sur la page d'accueil de l'assistant d'installation, cliquez sur Continuer.
  4. Lisez le contrat de licence et cliquez sur Continuer. Cliquez sur Accepter dans la fenêtre contextuelle pour accepter la licence.
  5. Sur la page de sélection de la destination, choisissez le lecteur et cliquez sur Continuer.
  6. Si vous avez choisi l'option de téléchargement Tout, Java ou Ruby, vous pouvez personnaliser votre installation. Sur la dernière fenêtre de l'assistant d'installation, cliquez sur le bouton Personnaliser en bas à gauche. L'arborescence des produits s'affiche.
  7. Sélectionnez les produits à installer.
  8. Saisissez le nom et le mot de passe administrateur de votre système et cliquez sur OK pour commencer l'installation.

Package indépendant de la plate-forme

Pour exécuter le logiciel :

  1. Une fois le téléchargement du fichier ZIP indépendant de la plate-forme terminé, extrayez-le dans n'importe quel dossier de votre système.
  2. Exécutez le fichier exécutable situé dans le répertoire netbeans/bin.
  3. Acceptez le contrat de licence. NetBeans IDE se lance.

Note:Si une installation de JDK compatible n''existe pas, vous devriez ajouter manuellement un chemin vers le répertoire d''installation JDK en faisant ceci:

  1. Ouvrir le fichier netbeans.conf situé dans le répertoire netbeans/etcdans un éditeur de texte.
  2. Entrer l''emplacement de l''installation de JDK compatible pour l''option netbeans_jdkhome. L''emplacement par défaut dans Windows est C:\Program Files\Java\jdk1.6.0.
  3. Enregistrer le fichiernetbeans.conf et exécuter le fichier dans le répertoirenetbeans/bin.

Note: GlassFish Server Open Source Edition et Apache Tomcat sont disponible seulement avec les installeurs spécifiques à la plateforme.

Désinstallation du logiciel

Suivre les instructions dans cette section pour désinstaller l''EDI de votre système. Ces instructions de désinstallation s'appliquent à toutes les plateformes supportées. Pour consulter la liste des plates-formes prises en charge, reportez-vous aux Notes de version.

Microsoft Windows

Si nécessaire, vous pouvez désinstaller l'EDI Netbeans,lassFish Server Open Source Edition 3.0.1 et Apache Tomcat en utilisant les procédures suivantes.

Pour désinstaller l'EDI :

  1. Arrêtez l'EDI.
  2. A partir du panneau de configuration, sélectionner Ajout/Suppression de programmes puis sélectionner Netbeans IDE 6.9 et cliquer Supprimer.

Pour désinstaller GlassFish Server Open Source Edition 3.0.1:

  1. Arrêtez l'EDI.
  2. A partir du panneau de configuration, sélectionner Ajout/Suppression de programmes puis sélectionner GlassFish Server Open Source Edition 3.0.1 et cliquer Supprimer.

Pour désinstaller Apache Tomcat :

  1. Arrêtez l'EDI.
  2. A partir du panneau de configuration, sélectionner Ajout/Suppression de programmes puis sélectionner Apache Tomcat 6.0.26 et cliquer Supprimer.

SE Solaris, Linux

Si nécessaire, vous pouvez désinstaller l'EDI Netbeans,lassFish Server Open Source Edition 3.0.1 et Apache Tomcat en utilisant les procédures suivantes.

Pour désinstaller l'EDI :

  1. Arrêtez l'EDI.
  2. Dans le répertoire d'installation de l'EDI, exécutez le fichier uninstall.sh.
  3. Pour désinstaller GlassFish et Tomcat ensemble avec l'EDI, à la page Récapitulatif s'assurer que les versions correctes sont repertoriées et cocher les cases appropriées.
  4. Sur la page récapitulative, cliquez sur Désinstaller.
  5. Une fois la désinstallation terminée, cliquez sur Terminer.

Pour désinstaller GlassFish Server Open Source Edition 3.0.1:

  1. Arrêtez l'EDI.
  2. Recherchez le répertoire d'installation du serveur d'application.

    Note: L''emplacement de l''installation par défaut est différent en fonction du type d''installation et de la plateforme. Le tableau suivant présente le répertoire d'installation du serveur d'application par défaut pour différentes plates-formes.

    Plate-forme Installation en tant que... Installation de GlassFish 3.0.1
    Windows Utilisateur avec droits d'administrateur %ProgramFiles%\glassfish-3.0.1
    Windows Utilisateur %USERPROFILE%\glassfish-3.0.1
    SE Solaris racine /opt/glassfish-3.0.1
    SE Solaris, Linux Utilisateur ~/glassfish-3.0.1
    Linux racine /usr/local/glassfish-3.0.1
  3. Dans le répertoire d'installation du serveur d'application, exécutez le fichier uninstall.sh.
  4. Sur la page récapitulative, cliquez sur Désinstaller.
  5. Une fois la désinstallation terminée, cliquez sur Terminer.

Pour désinstaller Apache Tomcat :

  1. Arrêtez l'EDI.
  2. Recherchez le répertoire d'installation d'Apache Tomcat et exécutez le fichier uninstall.sh.
  3. Sur la page récapitulative, cliquez sur Désinstaller.
  4. Une fois la désinstallation terminée, cliquez sur Terminer.

Mac OS X

Pour désinstaller l'IDE et le serveur d'application:

  1. Arrêtez le logiciel.
  2. Faites glisser les dossiers ou fichiers du programme vers la corbeille. Notez que l'emplacement par défaut de l'EDI et du serveur d'application est /Applications/NetBeans/.

Package indépendant de la plate-forme

Pour désinstaller les fichiers de l'EDI, suivez les étapes décrites à la section Nettoyage après une installation incomplète de ce document.

Cohabitation de plusieurs version de NetBeans IDE

Plusieurs installations de NetBeans IDE 5.x et 6.x peuvent coexister sur le même système avec NetBeans IDE 6.9. Vous n'avez pas à désinstaller les versions antérieures pour installer ou exécuter NetBeans IDE 6.9.

Si vous avez une installation existante de NetBeans IDE, vous pouvez choisir ou non d'importer les paramètres utilisateur à partir du répertoire utilisateur existant lorsque vous exécutez NetBeans IDE 6.9 pour la première fois.

Note: La mise à jour de version incrémentale de l'IDE ne peut être effectuée à ce moment. Pour mettre à niveau vers la version la plus récente, vous devez télécharger le dernier programme d'installation.

Dépannage

À propos du répertoire de travail du programme d'installation

Le programme d'installation stocke les données concernant les sessions d'installation dans son répertoire de travail. Ces données sont par exemple la liste des composants installés sur votre système, les messages du journal d'installation et les données du registre.

L'emplacement par défaut du répertoire de travail du programme d'installation est %USERPROFILE%\.nbi sous Windows et ~/.nbi sur d'autres plates-formes. Le sous-répertoire du journal contient les fichiers journaux d'installation.

Nettoyage après une installation incomplète

Après des installations incomplètes ou ayant échoué, il peut être nécessaire de supprimer les installations précédentes de votre système.

Sous Microsoft Windows, le SE Solaris et Linux :

  1. Supprimez le répertoire de travail du programme d'installation.
  2. Supprimez les répertoires d'installation de l'EDI et du serveur d'application GlassFish.
  3. Sous la plate-forme Windows, supprimez toutes les entrées du registre nbi* des clés de registre suivantes: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Sous Mac OS X :

  • Faites glisser les dossiers ou fichiers du programme vers la corbeille.

Désactivation de certains logiciels avant l'installation

Il est préférable de désactiver les applications de pare-feu, d'antivirus et de contrôle avant de commencer l'installation. Certaines de ces applications peuvent empêcher la configuration des composants de l'EDI pendant la procédure d'installation.

Par exemple, si l'application Process Monitor de Logitech (LVPrcSrv.exe est en cours d'exécution sur votre système, désactivez-la avant l'installation.
