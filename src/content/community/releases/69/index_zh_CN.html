NetBeans IDE 6.9 发行信息

NetBeans IDE 是一个荣获奖项的集成开发环境，可用于 Windows、Mac、Linux 和 Solaris。NetBeans 项目由开源 IDE 和应用程序平台组成；通过该 IDE 和平台，开发者可以使用 Java 平台以及 JavaFX、PHP、JavaScript 和 Ajax、Ruby 和 Ruby on Rails、Groovy 和 Grails、C/C++ 快速创建 Web、企业、桌面和 Mobile 应用程序。

NetBeans 项目由活跃的开发者社区支持，并且还提供了广泛的文档和培训资源，以及各种供您选用的第三方插件。

发行版概述

NetBeans IDE 6.9 新增了 JavaFX 编写器，它是一种用于以可视方式生成 JavaFX GUI 应用程序的可视布局工具，类似于 Java SE 应用程序的 Swing GUI 生成器。其他重要功能包括用于 NetBeans 平台应用程序的 OSGi 互操作性，支持使用 Maven 开发 OSGi 包，支持 JavaFX SDK 1.3、PHP Zend 框架和 Ruby on Rails 3.0，以及改进了 Java 编辑器、Java 调试器和问题跟踪等功能。

功能集锦

OSGi

  • 可使用 Maven 开发 OSGi 包
  • 捆绑了 Felix 容器
  • 可在 NetBeans RCP 应用程序中使用 OSGi 包

NetBeans 平台

  • OSGi 互操作性（可开发 OSGi 包并将其用在基于平台的应用程序中）
  • 支持 Felix 2.0.3 OSGi，并对 Equinox 提供实验性支持
  • 可将 NetBeans 模块转换为 OSGi 包，并在未修改的 OSGi 容器中运行它们
  • 标记编辑器可用于基于平台的 Ant 和 Maven 项目
  • 基于 NetBeans 平台的应用程序可生成用于最常见操作系统的安装程序
  • 增强了对 Web 服务使用和数据库连接的支持

 

JavaFX

  • JavaFX 编写器可用于设计窗体式 UI 组件，具有多种可选状态，并可访问各种数据源
  • 添加并改进了编辑器提示和重构功能
  • 增强了 JavaFX 形状、颜色、效果和图表的面板
  • 改进了代码格式设置功能

Java EE

  • 支持上下文和依赖关系注入 (Contexts and Dependency Injection, CDI) (JSR-299)
  • 支持 Spring Framework 3.0
  • 捆绑了 GlassFish Server Open Source Edition 3.0.1
  • 可在更改数据库后轻松地重新生成 JPA 条目
  • 为 RCP 应用程序提供了 REST Web 服务支持

Java

  • 可进行 Java 调试器断点分组，并提供了调试器连接参数历史记录
  • 在编辑器中支持标注处理程序，可在“项目属性”中对此进行配置
  • 提供了新的 Applet 和 Web Start (JNLP) 支持
  • 改进了栈跟踪分析器和 URL 中的导航，以及“转至覆盖的方法”/“转至实现的方法”操作

Web 语言：HTML、CSS、JavaScript

  • 为类似 CSS 和 HTML 的语言提供了重构和查找使用实例功能
  • 为 ID 和类选择器属性提供了代码完成和超级链接功能
  • 重构内嵌 CSS 样式

 

Java Card

  • 支持创建 Java Card Connected 3 Servlet、传统和扩展 Applet
  • 向后兼容 Java Card 2.2.x 和 Java Card Classic
  • 调试和多 Java Card Platform/设备创建
  • 可与 Java Card 引用实现配合使用

PHP

  • 支持 PHP Zend 框架
  • 提供了“覆盖”/“实现”和“已覆盖”/“已实现”标注
  • 新的格式化程序具有很多格式设置规则

Ruby

  • 支持 Ruby on Rails 3.0

C/C++

  • 在 C/C++ 项目中集成了单元测试
  • 增强了远程开发，包括远程文件下载和浏览
  • 改进了对 makefile 目标和 Fortran 的支持
  • 改进了在覆盖的方法和类之间进行的超级链接导航

常规

  • 提供了更多 Java 和 Ruby 编辑器提示
  • 在编辑器中支持使用拼写检查器
  • “帮助”菜单项可报告 NetBeans IDE 的错误和运行缓慢状况
  • 支持多个基于 Kenai 的团队开发服务器（例如 kenai.com 和 netbeans.org）。
